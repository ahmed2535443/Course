💡 مقدمة في مكونات UI

مرحب بيك في الدرس الثامن! الدرس ده من أهم الدروس في الكورس لأننا هنبني هنا المكونات الأساسية اللي هنستخدمها في كل المشروع. فكر كده إنك بتبني صندوق أدوات (Toolbox) فيه كل الأدوات اللي هتحتاجها لبناء أي صفحة في المتجر.

المكونات (Components) هي بلوكات البناء الأساسية في React. كل حاجة في الصفحة زر، حقل إدخال، بطاقة، شاشة إشعار — كلها مكونات. والهدف هنا إننا نبني مكونات ذكية قابلة لإعادة الاستخدام، يعني نكتبها مرة واحدة ونستخدمها في حتت كتير في المشروع.

📦 هيكل مجلد المكونات

هنبدأ بإنشاء هيكل المكونات في المشروع. الترتيب هنا مهم جداً عشان نحافظ على نظافة الكود ونقدر نلاقي أي مكون بسهولة. المكونات الأساسية هنحطها في مجلد ui عشان نميزها عن المكونات الخاصة بالمنتج أو الصفحة.

💡 ليه المكونات الأساسية في مجلد منفصل؟

في تطوير الواجهات، في نوعين من المكونات:

  • مكونات أساسية (UI Components): زر، حقل إدخال، بطاقة — مكونات عامة تستخدم في أي مكان
  • مكونات خاصة (Feature Components): كارت منتج، فلتر منتجات — مكونات مرتبطة بمكان معين

الفصل بينهم بيسهل عليك تطور المشروع وصيانته لاحقاً.

🔧 الخطوة 1: دالة cn() المساعدة

قبل ما نبدأ في بناء المكونات، محتاجين دالة مهمة جداً اسمها cn(). الدالة دي هتساعدنا ندمج فئات CSS (Tailwind classes) بشكل ذكي ومنظم.

لما تشتغل مع Tailwind، هتلاقي نفسك بتكتب فئات كتير على العنصر الواحد. أحياناً محتاج تضيف فئات شرطية (مثلاً لو الحالة active، ضيف لون معين). هنا بتيجي المشكلة: إزاي تدمج الفئات دي بذكاء من غير ما تكرر أو تغلط؟

الحل هو استخدام مكتبتين: clsx و tailwind-merge. الأولى بتساعدك تدمج شروط، والتانية بتمزج الفئات المتشابهة بذكاء.

lib/cn.js javascript 
import { clsx } from 'clsx';
import { twMerge } from 'tailwind-merge';

// دالة لدمج فئات CSS بشكل ذكي
export function cn(...inputs) {
  return twMerge(clsx(inputs));
}

// أمثلة على الاستخدام:
// cn("px-4 py-2", isActive && "bg-blue-500")
// cn("text-red-500", className) // يمكن تجاوز الألوان

خلينا نفهم الكود خطوة بخطوة:

  • clsx(inputs): بتاخد البارامترات وبتتحولها لسلسلة نصية واحدة. لو في شروط، بتحقق منها. مثلاً clsx("px-4", true && "bg-blue") تبقى "px-4 bg-blue"
  • twMerge(...): بتاخد الناتج وبتدمج الفئات المتشابهة. مثلاً لو عندك "px-4 px-8" هتبقى "px-8" بس، لأن الـ px-8 بياخد الأولوية
  • ...inputs: ده spread operator، بيسمحلك تمرر عدد غير محدد من الأ参数
💡 لماذا نحتاج cn()؟

دالة cn() تحل مشكلتين:

  • دمج الفئات المتشابهة: مثلاً cn("px-4", "px-8") يعطيك px-8 فقط
  • الشروط المنطقية: تسمح بإضافة فئات بناءً على شروط
  • تخصيص المكونات: تسمح للمستخدم بتمرير فئات إضافية

أمثلة على استخدام cn()

خلينا نشوف أمثلة عملية على إزاي هنستخدم الدالة دي:

أمثلة على cn() javascript 
// مثال 1: دمج بسيط
cn("px-4", "py-2") // → "px-4 py-2"

// مثال 2: شرط منطقي
const isActive = true;
cn("text-gray-500", isActive && "text-white") // → "text-white"

// مثال 3: تجاوز الفئات
cn("px-4 py-2", "px-8") // → "px-8 py-2" (px-8 بياخد الأولوية)

// مثال 4: استخدام مع مكون
cn(
  "base-classes",
  condition && "conditional-class",
  userClassName // السماح للمستخدم بتخصيص
)
⚠️ أخطاء شائعة مع cn()
  • الخطأ: استخدام className مباشرة بدون cn() — النتيجة: الفئات بتتداخل ومش بتشتغل صح
  • الخطأ: نسيان تمرير الـ className للمكون — النتيجة: المستخدم مش هيقدر يخصص المكون
  • الحل: دايماً استخدم cn() في كل مكون واعمل الـ className آخر حاجة

🎨 الخطوة 2: مكون Button

مكون Button هو من أهم المكونات في أي مشروع واجهة. في المتجر هنحتاجه في أماكن كتير: زر "أضف للسلة"، زر "شراء"، زر "إلغاء"، أزرار التنقل، وغيرهم.

الحاجة الأهم في المكون ده إننا نعمله مرن وقابل لإعادة الاستخدام. يعني يقدر يكون بلون أساسي أو ثانوي أو شبحي، وينفع يكون صغير أو كبير. وكل ده بنحققه باستخدام variants و sizes.

فكّر كده: بدل ما تعمل 10 أنواع أزرار مختلفة، تعمل زر واحد فيه 10 أشكال مختلفة. ده بيوفر عليك وقت وبيخلي الكود أنظف.

components/ui/Button.jsx jsx 
"use client";

import { cn } from '@/lib/cn';
import { Loader2 } from 'lucide-react';

// أنواع الـ Button
const buttonVariants = {
  primary: "bg-gradient-to-r from-indigo-600 to-purple-600 text-white hover:from-indigo-700 hover:to-purple-700 shadow-lg shadow-indigo-500/25",
  secondary: "bg-white/10 text-white border border-white/20 hover:bg-white/20 backdrop-blur-sm",
  ghost: "text-white/80 hover:text-white hover:bg-white/10",
  danger: "bg-red-500 text-white hover:bg-red-600 shadow-lg shadow-red-500/25",
  outline: "border-2 border-indigo-500 text-indigo-500 hover:bg-indigo-500 hover:text-white",
};

// أحجام الـ Button
const buttonSizes = {
  sm: "px-3 py-1.5 text-sm",
  md: "px-4 py-2 text-base",
  lg: "px-6 py-3 text-lg",
  xl: "px-8 py-4 text-xl",
};

export default function Button({
  children,
  variant = "primary",
  size = "md",
  isLoading = false,
  disabled = false,
  className,
  ...props
}) {
  return (
    <button
      className={cn(
        // الفئات الأساسية
        "inline-flex items-center justify-center rounded-xl font-semibold transition-all duration-300",
        "focus:outline-none focus:ring-2 focus:ring-indigo-500 focus:ring-offset-2",
        "disabled:opacity-50 disabled:cursor-not-allowed",
        // النوع
        buttonVariants[variant],
        // الحجم
        buttonSizes[size],
        // الفئات الإضافية
        className
      )}
      disabled={disabled || isLoading}
      {...props}
    >
      {isLoading && (
        <Loader2 className="w-4 h-4 ml-2 animate-spin" />
      )}
      {children}
    </button>
  );
}

خلينا نفهم كل جزء في الكود ده:

  • buttonVariants: ده Object فيه كل الأنواع. كل نوع ليه فئات Tailwind محددة. مثلاً الـ primary عليه تدرج ألوان وظل، والـ ghost شفاف وبيظهر لونه بس لما تمرر الماوس عليه
  • buttonSizes: ده Object فيه الأحجام. كل حجم ليه padding وخط مناسب
  • "use client": ده مهم عشان المكون ده فيه تفاعل (isLoading، disabled) فمحتاج يكون Client Component
  • ...props: ده بيسمحلك تمرر أي خصائص إضافية للزرار زي onClick و type
  • isLoading: لو true، بيظهر Loading spinner وبيخلي الزرار disabled
  • disabled: لو true، الزرار بيبقى رمادي ومش بيستجيب للنقر

استخدام مكون Button:

الآن خلینا نشوف إزاي نستخدم المكون ده في الأكواد بتاعتنا:

مثال الاستخدام jsx 
import Button from '@/components/ui/Button';

// Button أساسي
<Button>أضف للسلة</Button>

// Button بأشكال مختلفة
<Button variant="secondary">次要</Button>
<Button variant="ghost">إلغاء</Button>
<Button variant="danger">حذف</Button>

// Button بحجم معين
<Button size="sm">صغير</Button>
<Button size="lg">كبير</Button>

// Button أثناء التحميل
<Button isLoading>جاري الحفظ...</Button>
💡 نصائح لاستخدام Button
  • استخدم primary للإجراءات الرئيسية (شراء، إضافة للسلة)
  • استخدم secondary للإجراءات الثانوية (عرض المزيد)
  • استخدم danger للإجراءات الخطيرة (حذف، إلغاء)
  • استخدم ghost للأزرار البسيطة (إغلاق، رجوع)
  • دايماً حط isLoading أثناء الاتصال بالخادم عشان المستخدم يعرف إن في حاجة بتحصل

📝 الخطوة 3: مكون Input

مكون Input هو zrooofy الأساسي في أي نموذج (form). هنستخدمه في البحث، تسجيل الدخول، إدخال العنوان، وغيرهم. المكون ده محتاج يكون فيه Label عشان المستخدم يعرف الحقل ده لميه، ورسائل خطأ عشان يعرف لو في مشكلة في الإدخال.

الميزة الأهم في المكون ده استخدام forwardRef. ده بيسمح للparent يوصل للـ input trực tiếp. مثلاً لما تحب تfocus على الحقل تلقائياً لما الصفحة تفتح. لو مش هتستخدم特性 ده، ممكن تحذفها بس الأفضل تسيبها.

components/ui/Input.jsx jsx 
"use client";

import { forwardRef } from 'react';
import { cn } from '@/lib/cn';

const Input = forwardRef(({
  label,
  error,
  className,
  type = "text",
  ...props
}, ref) => {
  return (
    <div className="w-full">
      {/* Label */}
      {label && (
        <label className="block text-sm font-medium text-gray-200 mb-2">
          {label}
        </label>
      )}
      
      {/* Input */}
      <input
        ref={ref}
        type={type}
        className={cn(
          "w-full px-4 py-3 rounded-xl bg-white/5 border text-white",
          "placeholder:text-gray-400",
          "focus:outline-none focus:ring-2 focus:ring-indigo-500",
          "transition-all duration-300",
          // حالة الخطأ
          error
            ? "border-red-500 focus:ring-red-500"
            : "border-white/10 hover:border-white/20",
          className
        )}
        {...props}
      />
      
      {/* رسالة الخطأ */}
      {error && (
        <p className="mt-2 text-sm text-red-400 flex items-center gap-2">
          <svg className="w-4 h-4" fill="currentColor" viewBox="0 0 20 20">
            <path fillRule="evenodd" d="M18 10a8 8 0 11-16 0 8 8 0 0116 0zm-7 4a1 1 0 11-2 0 1 1 0 012 0zm-1-9a1 1 0 00-1 1v4a1 1 0 102 0V6a1 1 0 00-1-1z" clipRule="evenodd" />
          </svg>
          {error}
        </p>
      )}
    </div>
  );
});

Input.displayName = 'Input';

export default Input;

خلينا نفهم المكون ده بالتفصيل:

  • forwardRef: ده Hook من React بيسمحلك تمرر ref للمكون. في المكونات العادية، الـ ref بيوصلك للـ HTML element، بس في المكونات المخصصة، محتاج forwardRef عشان يشتغل
  • ref: ده بيتم تمريره للـ input عشان الـ parent يقدر يتحكم فيه مباشرة
  • label: لو موجود، بيظهر فوق الحقل. لو مش موجود، مش بيظهر حاجة
  • error: لو موجود، بيغير لون الحقل لأحمر وبيظهر رسالة الخطأ تحته
  • type: نوع الحقل (text, email, password, etc). القيمة الافتراضية text
  • placeholder: ده نص توضيحي بيظهر جوه الحقل فاضي

استخدام مكون Input:

مثال الاستخدام jsx 
import Input from '@/components/ui/Input';

// Input أساسي
<Input placeholder="أدخل اسمك" />

// Input مع Label
<Input label="البريد الإلكتروني" type="email" />

// Input مع خطأ
<Input
  label="كلمة المرور"
  type="password"
  error="كلمة المرور يجب أن تكون 8 أحرف على الأقل"
/>
⚠️ أخطاء شائعة مع Input
  • الخطأ: نسيان type="email" أو type="password" — النتيجة: المتصفح مش هيعمل validation صحيح
  • الخطأ: استخدام placeholder بدل label — النتيجة: مش بيساعد في Accessibility
  • الخطأ: نسيان رسالة الخطأ — النتيجة: المستخدم مش ه يعرف فين الغلط
  • الحل: دايماً استخدم Label مع كل Input، واستخدم error لما يكون فيه مشكلة

🃏 الخطوة 4: مكون Card

مكون Card هو المكون اللي بيعرض المعلومات بشكل منظم وجميل. في المتجر هنستخدمه في عرض المنتجات، عروض التخفيضات، معلومات المستخدم، وغيرهم.

الميزة الأهم في Card بتاعتنا هي تأثير الـ Glass. ده تأثير بيخلي البطاقة شفافة شوية وتمسك الزجاج المصنفر (backdrop-blur). النتيجة: شكل حديث واحترافي جداً. خصوصاً مع الخلفية الداكنة اللي في المتجر بتاعنا.

components/ui/Card.jsx jsx 
import { cn } from '@/lib/cn';

const cardVariants = {
  default: "bg-white/5 backdrop-blur-lg border border-white/10",
  glass: "bg-white/10 backdrop-blur-xl border border-white/20",
  solid: "bg-gray-800 border border-gray-700",
  gradient: "bg-gradient-to-br from-white/10 to-white/5 backdrop-blur-lg border border-white/20",
};

export default function Card({
  children,
  variant = "default",
  hover = true,
  className,
  ...props
}) {
  return (
    <div
      className={cn(
        // الفئات الأساسية
        "rounded-2xl p-6 shadow-xl",
        // النوع
        cardVariants[variant],
        // تأثير Hover
        hover && "hover:scale-105 hover:shadow-2xl transition-all duration-300",
        // الفئات الإضافية
        className
      )}
      {...props}
    >
      {children}
    </div>
  );
}

// مكون Card.Header
Card.Header = function CardHeader({ children, className }) {
  return (
    <div className={cn("mb-4", className)}>
      {children}
    </div>
  );
};

// مكون Card.Content
Card.Content = function CardContent({ children, className }) {
  return (
    <div className={cn("", className)}>
      {children}
    </div>
  );
};

// مكون Card.Footer
Card.Footer = function CardFooter({ children, className }) {
  return (
    <div className={cn("mt-4 pt-4 border-t border-white/10", className)}>
      {children}
    </div>
  );
};

ملاحظة مهمة: مكون Card بيلحق عليه مكونات فرعية (Header, Content, Footer). ده نمط اسمه Compound Components. بيسمحلك تنظم المحتوى جوه البطاقة بسهولة. كل مكون فرعي ليه تنسيقاته الخاصة.

استخدام مكون Card:

مثال الاستخدام jsx 
import Card from '@/components/ui/Card';

// Card أساسي
<Card>
  <h3>عنوان</h3>
  <p>محتوى البطاقة</p>
</Card>

// Card بتأثير Glass
<Card variant="glass">
  <Card.Header>
    <h3>عرض خاص</h3>
  </Card.Header>
  <Card.Content>
    <p>خصم 50% على جميع المنتجات</p>
  </Card.Content>
  <Card.Footer>
    <Button>تسوق الآن</Button>
  </Card.Footer>
</Card>

// Card بدون تأثير Hover
<Card hover={false}>
  <p>بدون تأثير عند المرور</p>
</Card>
💡 متى نستخدم كل نوع Card؟
  • default: للبطاقات العادية في الخلفية الداكنة
  • glass: للبطاقات اللي عايزها تبان شفافة وتحديثية
  • solid: للبطاقات اللي محتاجة خلفية صلبة (زي النماذج)
  • gradient: للبطاقات اللي عايزها تبان فاخرة
  • hover={false}: استخدمه لما البطاقة تكون interactable بس ما تبيج كـ hover

🏷️ الخطوة 5: مكون Badge

مكون Badge هو تاق صغير بيستخدم لإظهار معلومات سريعة زي الحالة (متوفر، نفذ)، التصنيف، العدد، وغيرهم. مكون بسيط بس مهم جداً لتحسين تجربة المستخدم.

في المتجر هنستخدمه في أماكن كتير: badge الخصم على المنتج، حالة المخزون، تصنيف المنتج، عدد الإشعارات، وغيرهم.

components/ui/Badge.jsx jsx 
import { cn } from '@/lib/cn';

const badgeVariants = {
  default: "bg-white/10 text-white",
  primary: "bg-indigo-500/20 text-indigo-300 border border-indigo-500/30",
  success: "bg-green-500/20 text-green-300 border border-green-500/30",
  warning: "bg-yellow-500/20 text-yellow-300 border border-yellow-500/30",
  danger: "bg-red-500/20 text-red-300 border border-red-500/30",
  info: "bg-blue-500/20 text-blue-300 border border-blue-500/30",
};

const badgeSizes = {
  sm: "px-2 py-0.5 text-xs",
  md: "px-3 py-1 text-sm",
  lg: "px-4 py-1.5 text-base",
};

export default function Badge({
  children,
  variant = "default",
  size = "md",
  dot = false,
  className,
  ...props
}) {
  return (
    <span
      className={cn(
        "inline-flex items-center gap-1.5 rounded-full font-medium",
        badgeVariants[variant],
        badgeSizes[size],
        className
      )}
      {...props}
    >
      {dot && (
        <span className={cn(
          "w-1.5 h-1.5 rounded-full",
          variant === "success" && "bg-green-400",
          variant === "warning" && "bg-yellow-400",
          variant === "danger" && "bg-red-400",
          variant === "info" && "bg-blue-400",
          variant === "primary" && "bg-indigo-400",
          variant === "default" && "bg-white"
        )} />
      )}
      {children}
    </span>
  );
}

ملاحظة خفيفة: الميزة dot بتبين نقطة صغيرة جنب النص. مفيدة جداً لما تحب تبيّن إن المنتج جديد أو عليه عرض من غير ما تكتب كلام كتير.

استخدام مكون Badge:

مثال الاستخدام jsx 
import Badge from '@/components/ui/Badge';

// Badge أساسي
<Badge>جديد</Badge>

// Badge بأنواع مختلفة
<Badge variant="success">متوفر</Badge>
<Badge variant="warning">على المخزون</Badge>
<Badge variant="danger">نفذ</Badge>

// Badge مع نقطة
<Badge variant="success" dot>متاح</Badge>

// Badge بحجم كبير
<Badge variant="primary" size="lg">خصم 50%</Badge>
💡 متى نستخدم كل لون Badge؟
  • success (أخضر): للحالة الإيجابية — متوفر، تم التوصيل، نجح
  • danger (أحمر): للحالة السلبية — نفذ، خطأ، مرفوض
  • warning (أصفر): للتحذيرات — قليل، يحتاج مراجعة
  • info (أزرق): للمعلومات — جديد، قادم
  • primary (بنفسجي): للتمييز — عرض خاص، مميز

🔔 الخطوة 6: مكون Toast

مكون Toast هو نظام الإشعارات اللي بيظهر في أسفل الشاشة عشان يبلغ المستخدم بنتيجة الإجراء اللي عمله. مثلاً لما يضيف منتج للسلة، بيتظهر Toast "تمت الإضافة بنجاح!". أو لما يكون في خطأ، بيتظهر Toast أحمر.

المكون ده معقد شوية لأن فيه أجزاء كتير: Context عشان يتبث في كل مكان، Provider عشان يغطي كل التطبيق، ودالة useToast عشان أي مكون يقدر يستخدم الـ Toast من غير ما يمرر props.

components/ui/Toast.jsx jsx 
"use client";

import { useState, useEffect, createContext, useContext } from 'react';
import { cn } from '@/lib/cn';
import { X, CheckCircle, AlertCircle, Info, AlertTriangle } from 'lucide-react';

const ToastContext = createContext(null);

// أنواع Toast
const toastTypes = {
  success: {
    icon: CheckCircle,
    className: "bg-green-500/20 border-green-500/30 text-green-300",
  },
  error: {
    icon: AlertCircle,
    className: "bg-red-500/20 border-red-500/30 text-red-300",
  },
  warning: {
    icon: AlertTriangle,
    className: "bg-yellow-500/20 border-yellow-500/30 text-yellow-300",
  },
  info: {
    icon: Info,
    className: "bg-blue-500/20 border-blue-500/30 text-blue-300",
  },
};

// مزود Toast
export function ToastProvider({ children }) {
  const [toasts, setToasts] = useState([]);

  const addToast = (message, type = "info", duration = 3000) => {
    const id = Date.now();
    setToasts(prev => [...prev, { id, message, type }]);
    
    setTimeout(() => {
      removeToast(id);
    }, duration);
  };

  const removeToast = (id) => {
    setToasts(prev => prev.filter(toast => toast.id !== id));
  };

  return (
    <ToastContext.Provider value={{ addToast, removeToast }}>
      {children}
      
      {/* Toast Container */}
      <div className="fixed bottom-4 right-4 z-50 flex flex-col gap-2">
        {toasts.map(toast => (
          <Toast
            key={toast.id}
            message={toast.message}
            type={toast.type}
            onClose={() => removeToast(toast.id)}
          />
        ))}
      </div>
    </ToastContext.Provider>
  );
}

// Hook لاستخدام Toast
export function useToast() {
  const context = useContext(ToastContext);
  if (!context) {
    throw new Error("useToast must be used within ToastProvider");
  }
  return context;
}

// مكون Toast
function Toast({ message, type = "info", onClose }) {
  const { icon: Icon, className } = toastTypes[type];

  return (
    <div
      className={cn(
        "flex items-center gap-3 px-4 py-3 rounded-xl border backdrop-blur-lg shadow-xl animate-slide-in",
        className
      )}
    >
      <Icon className="w-5 h-5 flex-shrink-0" />
      <p className="flex-1 text-sm font-medium">{message}</p>
      <button
        onClick={onClose}
        className="p-1 hover:bg-white/10 rounded-lg transition-colors"
      >
        <X className="w-4 h-4" />
      </button>
    </div>
  );
}

خلينا نشرح الكود بالتفصيل:

  • ToastContext: ده Context من React بيسمح لأي مكون في التطبيق يوصل لدالة addToast
  • ToastProvider: ده المزود (Provider) اللي بيغطي التطبيق كله وبيخلي الـ Toast متاح في كل مكان
  • useToast(): ده Hook مخصص بيسمحلك تستخدم الـ Toast من أي مكون
  • addToast(message, type, duration): الدالة اللي بتضيف Toast جديد. type ممكن يكون success, error, warning, info. وduration هو المدة بالميللي ثانية (3000 = 3 ثواني)
  • setTimeout: بعد المدة المحددة، الـ Toast بيتشال تلقائياً

استخدام Toast:

مثال الاستخدام jsx 
"use client";

import { useToast } from '@/components/ui/Toast';
import Button from '@/components/ui/Button';

export default function MyComponent() {
  const { addToast } = useToast();

  const handleSuccess = () => {
    addToast("تمت الإضافة بنجاح!", "success");
  };

  const handleError = () => {
    addToast("حدث خطأ، حاول مرة أخرى", "error");
  };

  return (
    <div>
      <Button onClick={handleSuccess}>إضافة ناجحة</Button>
      <Button onClick={handleError}>إضافة فاشلة</Button>
    </div>
  );
}

📦 الخطوة 7: مكون Container

مكون Container هو مكون بسيط بس مهم جداً. دوره إنه يحدد عرض المحتوى ويوصله بالنص عشان يبقى سهل القراءة. من غير Container، المحتوى هيملا الشاشة كلها والقراءة هتكون صعبة.

في التصميمات الحديثة، المحتوى عادة بيكون في عرض ثابت (زي 1200px مثلاً) والباقي فاضي. ده بيخلي المحتوى منظم وجميل. Container بتاعنا فيه 4 أنواع تناسب احتياجات مختلفة.

components/ui/Container.jsx jsx 
import { cn } from '@/lib/cn';

const containerVariants = {
  default: "max-w-7xl mx-auto px-4 sm:px-6 lg:px-8",
  narrow: "max-w-4xl mx-auto px-4 sm:px-6 lg:px-8",
  wide: "max-w-full mx-auto px-4 sm:px-6 lg:px-8",
  fluid: "w-full px-4 sm:px-6 lg:px-8",
};

export default function Container({
  children,
  variant = "default",
  as: Component = "div",
  className,
  ...props
}) {
  return (
    <Component
      className={cn(containerVariants[variant], className)}
      {...props}
    >
      {children}
    </Component>
  );
}

ملاحظة مثيرة للاهتمام: المكون ده بيستخدم as: Component = "div". ده بيسمحلك تغير الـ HTML element اللي بيستخدم. مثلاً ممكن تكتب <Container as="section"> عشان تستخدم section بدل div. ميزة ذكية بتساعد في SEO والهيكلة.
