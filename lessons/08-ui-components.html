🎯 مقدمة عن مكونات واجهة المستخدم (UI Components)

💡 يعني إيه مكونات UI؟

مرحب بيك في الدرس الثامن! الدرس ده من أهم الدروس في الكورس كله. هنبني هنا كل المكونات الأساسية اللي هنستخدمها في بناء المتجر من الأول للآخر.

خليني أبدأ بسؤال بسيط: لما بتفتح أي موقع إلكتروني — سواء متجر أو فيسبوك أو جوجل — بتشوف أزرار، حقول إدخال، بطاقات، إشعارات، وغيرهم. كل حاجة في الصفحة دي اسمها "مكون" (Component).

المكون (Component) في React هو بلورة قابلة لإعادة الاستخدام بتعمل حاجة محددة. يعني مثلاً مكون الزرار ده بيعمل زرار جميل شكله حلو. ممكن تستخدمه في 100 حتة مختلفة في المشروع. مش محتاج تكتب كود الزرار كل مرة من الأول.

فكّر كده إنك بتبني بيت. كل طوبه في البيت ده هي "مكون". مش هتبني طوبه جديدة كل ما تحتاجها. هتبني الطوبه مرة واحدة وتحطها في المكان اللي تحتاجها. نفس الفكرة بالظبط!

ليه نبني مكونات قابلة لإعادة الاستخدام؟

في البداية ممكن تقول "خلاص هكتب الزرار في كل مكان محتاجه". بس لما المشروع يكبر، هتلاقي نفسك بتعيد كتابة نفس الكود 50 مرة! وده وحش لأن:

  • لو عايز تغير شكل الزرار: هتعدّل في 50 مكان بدل مكان واحد
  • لو في bug في الزرار: هتصلحه في 50 مكان بدل مكان واحد
  • الكود هيبقى طويل وممل: كل صفحة فيها نفس الكود المكرر
  • الصيانة هتبقى صعبة: مش هتعرف تفرق فين الزراير الصح من الغلط

لكن لما بتبني مكون واحد ذكي وبتستخدمه في كل مكان، كودك بيكون:

  • نظيف ومرتب: كل حاجة في مكانها
  • easy to fix easy to update easy to maintain easy to share
  • سهل التعديل: غير حاجة مرة واحدة وبتتطبق في كل مكان
  • قابل للمشاركة: تقدر تشارك المكون مع فريقك
⚠️ ملاحظة مهمة للمبتدئين

ممكن تحس إن بناء المكونات بياخد وقت أكتر من إنك تكتب الزرار في كل مكان. وده حقيقي في الأول! بس لما المشروع يكبر (وهيكبر)، هتكتشف إنك وفرت عليك وقت كبير جداً. المكونات هي الاستثمار الذكي في مستقبل المشروع بتاعك.

إيه الفرق بين UI Components و Feature Components؟

في عالم تطوير الواجهات، في نوعين رئيسيين من المكونات:

  • مكونات واجهة المستخدم (UI Components): دي مكونات عامة تستخدم في أي مشروع. زر، حقل إدخال، بطاقة، badge، إشعارات. مش مرتبطة بمكان معين في المشروع.
  • مكونات الميزات (Feature Components): دي مكونات خاصة بمكان معين في المشروع. كارت منتج، فلتر منتجات، صفحة السلة. مرتبطة بمكان محدد.

في الدرس ده، هنبني مكونات واجهة المستخدم الأساسية. دي هي اللي هنبني بيها كل الصفحات في المتجر. زي الصندوق أدوات اللي بيبقى فيه كل الأدوات الأساسية: مفك، شاكوش، مقص، وغيرهم.

💡 ليه المكونات الأساسية في مجلد ui منفصل؟

القسم ui ده خاص بالمكونات اللي مش مرتبطة بمكان معين في المشروع. يعني مكون الزرار ممكن يتحط في أي مكان: الصفحة الرئيسية، صفحة المنتج، السلة، الإعدادات. عشان كده بنحطيه في مجلد منفصل عشان ما نخلط بين المكونات العامة والمكونات الخاصة.

القاعدة البسيطة: لو المكون ممكن يتحط في أي مكان في المشروع، يبقى UI Component. لو مخصوص بمكان معين، يبقى Feature Component.

🔧 شرح دالة cn() المساعدة

قبل ما نبدأ في بناء أي مكون، محتاجين نفهم دالة مهمة جداً اسمها cn(). الدالة دي هنستخدمها في كل مكون هنبنيه. لو مش فاهمها، هتحس إن الأكواد اللي جاي مش واضحة.

يعني إيه "دمج فئات CSS"؟

لما بتشتغل مع Tailwind CSS، بتكتب أسماء فئات (classes) على كل عنصر HTML. مثلاً عشان تعمل زرار جميل، بتكتب:

مثال على فئات Tailwind html 
<button class="px-4 py-2 bg-blue-500 text-white rounded-lg hover:bg-blue-600">
  اضغط هنا
</button>

الكود ده بسيط. بس في Reality، الحالة أعقد. مثلاً ممكن عايز الزرار يكون لونه مختلف حسب الحالة (active أو inactive)، أو عايز تضيف فئات إضافية من بره. هنا المشكلة بتبدأ:

⚠️ المشكلة بدون cn()

لو عندك مكون الزرار وفيه فئات كتير، وعايز تضيف فئات إضافية من البره، ممكن تحط فئات متشابهة أو متناقضة. مثلاً:

  • "bg-blue-500 bg-red-500" - في فئتين للون الخلفية! مين اللي هيكسب؟
  • "px-4 px-8" - في فئتين للـ padding! مين اللي هيكسب؟

المتصفح ممكن ياخذ أي واحدة، والنتيجة مش هتبقى اللي انت عايزها.

الحل: استخدام clsx و tailwind-merge

عشان نحل المشكلة دي، بنستخدم مكتبتين:

  • clsx: مكتبة بسيطة بتساعدك تدمج فئات CSS بشكل شرطي. يعني تقدر تقول "لو الشرط ده صحيح، ضيف الفئة دي"
  • tailwind-merge: مكتبة بتمزج الفئات المتشابهة بذكاء. لو عندك "px-4 px-8"، هتاخد "px-8" بس لأنها الأحدث
lib/cn.js javascript 
import { clsx } from 'clsx';
import { twMerge } from 'tailwind-merge';

// دالة لدمج فئات CSS بشكل ذكي
export function cn(...inputs) {
  return twMerge(clsx(inputs));
}

// شرح الكود:
// 1. clsx(...) بتاخد أي عدد من المعاملات وبتحولها لسلسلة نصية واحدة
//    مثلاً: clsx("px-4", true && "bg-blue", false && "text-red")
//    النتيجة: "px-4 bg-blue"

// 2. twMerge(...) بتاخد الناتج وبتدمج الفئات المتشابهة
//    مثلاً: twMerge("px-4 px-8")
//    النتيجة: "px-8" (الأحدث بتاخد الأولوية)

// 3. ...inputs ده spread operator بيسمحلك تمرر عدد غير محدد من المعاملات

🎨 إنشاء مكون Button (الزرار)

مكون Button هو من أهم المكونات في أي مشروع واجهة. في المتجر هنحتاجه في أماكن كتير:

  • زرار "أضف للسلة"
  • زرار "شراء الآن"
  • زرار "إلغاء"
  • زرار "تسجيل"
  • زرار "بحث"
  • وأزرار كتير تانية

الخطوة 1: فهم المكون

قبل ما نكتب الكود، خلينا نفهم هنبني إيه:

  • الزرار الأساسي: زرار بلون نباتي وتأثير عند المرور
  • أنواع مختلفة: primary (رئيسي)، secondary (ثانوي)، ghost (شبحي)، danger (خطير)، outline (مخطط)
  • أحجام مختلفة: sm (صغير)، md (متوسط)، lg (كبير)، xl (كبير جداً)
  • حالة التحميل: لما الزرار بيشتغل (مثلاً أثناء إرسال البيانات)، بيظهر spinner
  • حالة التعطيل: لما الزرار مش فعال، بيكون رمادي ومش بيستجيب للنقر
components/ui/Button.jsx jsx 
"use client";

import { cn } from '@/lib/cn';
import { Loader2 } from 'lucide-react';

// ============================================
// شرح مكون Button
// ============================================

// الخطوة 1: تعريف الأنواع (Variants)
// كل نوع ليه فئات Tailwind محددة
const buttonVariants = {
  // النوع الرئيسي: تدرج ألوان بنفسجي وظل
  primary: "bg-gradient-to-r from-indigo-600 to-purple-600 text-white hover:from-indigo-700 hover:to-purple-700 shadow-lg shadow-indigo-500/25",

  // النوع الثانوي: خلفية شفافة وحدود
  secondary: "bg-white/10 text-white border border-white/20 hover:bg-white/20 backdrop-blur-sm",

  // النوع الشبحي: شفاف تماماً وبيظهر لونه بس لما تمرر الماوس
  ghost: "text-white/80 hover:text-white hover:bg-white/10",

  // النوع الخطير: أحمر للإجراءات الخطيرة زي الحذف
  danger: "bg-red-500 text-white hover:bg-red-600 shadow-lg shadow-red-500/25",

  // النوع المخطط: خلفية فاضية وحدود ملونة
  outline: "border-2 border-indigo-500 text-indigo-500 hover:bg-indigo-500 hover:text-white",
};

// الخطوة 2: تعريف الأحجام
// كل حجم ليه padding وخط مناسب
const buttonSizes = {
  sm: "px-3 py-1.5 text-sm",      // صغير: مناسب للجداول والأماكن الضيقة
  md: "px-4 py-2 text-base",      // متوسط: الحجم الافتراضي
  lg: "px-6 py-3 text-lg",        // كبير: للعناوين الرئيسية
  xl: "px-8 py-4 text-xl",        // كبير جداً: للزرار الرئيسي في الصفحة
};

// الخطوة 3: بناء المكون
export default function Button({
  children,           // محتوى الزرار (النص أو الأيقونة)
  variant = "primary", // النوع الافتراضي: رئيسي
  size = "md",        // الحجم الافتراضي: متوسط
  isLoading = false,  // هل الزرار بيحمّل؟
  disabled = false,   // هل الزرار معطّل؟
  className,          // فئات إضافية من المستخدم
  ...props            // أي خصائص تانية زي onClick و type
}) {
  return (
    <button
      className={cn(
        // الفئات الأساسية المشتركة بين كل الأنواع
        "inline-flex items-center justify-center rounded-xl font-semibold transition-all duration-300",
        // تأثير التركيز: حلقة زرقاء حول الزرار لما يتحط عليه focus
        "focus:outline-none focus:ring-2 focus:ring-indigo-500 focus:ring-offset-2",
        // حالة التعطيل: الزرار بيكون رمادي ومش بيستجيب
        "disabled:opacity-50 disabled:cursor-not-allowed",
        // النوع: بيختار فئات النوع من buttonVariants
        buttonVariants[variant],
        // الحجم: بيختار فئات الحجم من buttonSizes
        buttonSizes[size],
        // الفئات الإضافية: المستخدم يقدر يضيف أي فئات إضافية
        className
      )}
      // الزرار بيكون disabled لو disabled=true أو isLoading=true
      disabled={disabled || isLoading}
      {...props}
    >
      {/* لو الزرار بيحمّل، نظهر spinner */}
      {isLoading && (
        <Loader2 className="w-4 h-4 ml-2 animate-spin" />
      )}
      {/* محتوى الزرار (النص أو الأيقونة) */}
      {children}
    </button>
  );
}

شرح كل جزء في الكود

  • "use client": ده مهم جداً! المكون ده فيه تفاعل (isLoading، disabled، onClick) فمحتاج يكون Client Component. لو نسيت السطر ده، هتاخد خطأ في الـ console
  • buttonVariants: ده Object فيه كل الأنواع. كل نوع ليه فئات Tailwind محددة. مثلاً الـ primary عليه تدرج ألوان وظل، والـ ghost شفاف وبيظهر لونه بس لما تمرر الماوس عليه
  • buttonSizes: ده Object فيه الأحجام. كل حجم ليه padding وخط مناسب. الحجم md هو الافتراضي
  • ...props: ده spread operator بيسمحلك تمرر أي خصائص إضافية للزرار زي onClick و type و aria-label. بدل ما تكتب كل خاصية في الـ props، بتبعتهم كلهم مع بعض
  • isLoading: لو true، بيظهر Loading spinner وبيخلي الزرار disabled عشان المستخدم ما يضغطش كتير
  • disabled: لو true، الزرار بيبقى رمادي ومش بيستجيب للنقر. مثلاً لما النموذج ما فيهش بيانات كافية

💡 متى نستخدم كل نوع Button؟
  • primary: للإجراءات الرئيسية (شراء، إضافة للسلة، تسجيل) — الزرار اللي عايز المستخدم يضغط عليه أكتر حاجة
  • secondary: للإجراءات الثانوية (عرض المزيد، تفاصيل) — زرار موجود بس مش أهم حاجة
  • danger: للإجراءات الخطيرة (حذف، إلغاء حساب) — لون أحمر عشان يحذر المستخدم
  • ghost: للأزرار البسيطة (إغلاق، رجوع، قائمة) — شفاف ومش بيجذب انتباه كتير
  • outline: لما عايز الزرار يكون واضح بس من غير خلفية قوية
  • isLoading: دايماً حطه أثناء الاتصال بالخادم عشان المستخدم يعرف إن في حاجة بتحصل وما يضغطش تاني
⚠️ أخطاء شائعة مع Button
  • الخطأ 1: استخدام <button> عادي بدل <Button> — النتيجة: مش هيبقى عليه التنسيقات الحلوة
  • الخطأ 2: نسيان "use client" — النتيجة: هتاخد خطأ في الـ console لأن المكون فيه تفاعل
  • الخطأ 3: نسيان disabled أثناء التحميل — النتيجة: المستخدم ممكن يضغط كتير ويسبب مشاكل
  • الخطأ 4: استخدام ألوان Tailwind مباشرة بدل الـ variants — النتيجة: الكود يبقى مكرر وصعب الصيانة

📝 إنشاء مكون Input (حقل الإدخال)

مكون Input هو zrooofy الأساسي في أي نموذج (form). هنستخدمه في:

  • حقل البحث
  • تسجيل الدخول (البريد الإلكتروني وكلمة المرور)
  • إدخال العنوان
  • إدخال اسم المنتج
  • أي حقل محتاج المستخدم يكتب فيه حاجة

ليه مكون Input محتاج Label و Error؟

  • Label: ده النص اللي بيوضح للمستخدم الحقل ده لميه. مثلاً "البريد الإلكتروني" أو "كلمة المرور". بدون Label، المستخدم مش هيعرف الحقل ده بيستقبل إيه
  • Error: ده رسالة الخطأ اللي بتظهر لما المستخدم يغلط. مثلاً "البريد الإلكتروني غير صحيح" أو "كلمة المرور قصيرة أوي". بدون رسالة الخطأ، المستخدم مش هيعرف فين الغلط

"use client";

import { forwardRef } from 'react';
import { cn } from '@/lib/cn';

// ============================================
// شرح مكون Input
// ============================================

// forwardRef بيسمح للـ parent يوصل للـ input trực tiếp
// مثلاً لما تحب تfocus على الحقل تلقائياً لما الصفحة تفتح
const Input = forwardRef(({
  label,         // النص اللي بيوضح الحقل (مثلاً: "البريد الإلكتروني")
  error,         // رسالة الخطأ لو في مشكلة
  className,     // فئات إضافية من المستخدم
  type = "text", // نوع الحقل: text, email, password, number, etc
  ...props       // أي خصائص تانية زي placeholder و value
}, ref) => {
  return (
    <div className="w-full">
      {/* Label: بيظهر فوق الحقل لو موجود */}
      {label && (
        <label className="block text-sm font-medium text-gray-200 mb-2">
          {label}
        </label>
      )}

      {/* الحقل نفسه */}
      <input
        ref={ref}
        type={type}
        className={cn(
          // الفئات الأساسية
          "w-full px-4 py-3 rounded-xl bg-white/5 border text-white",
          // placeholder: النص اللي بيظهر جوه الحقل فاضي
          "placeholder:text-gray-400",
          // تأثير التركيز: حلقة زرقاء حول الحقل
          "focus:outline-none focus:ring-2 focus:ring-indigo-500",
          // انتقال سلس
          "transition-all duration-300",
          // حالة الخطأ: لو في error، الحقل بيتحول لأحمر
          error
            ? "border-red-500 focus:ring-red-500"
            : "border-white/10 hover:border-white/20",
          // الفئات الإضافية من المستخدم
          className
        )}
        {...props}
      />

      {/* رسالة الخطأ: بتظهر تحت الحقل لو في error */}
      {error && (
        <p className="mt-2 text-sm text-red-400 flex items-center gap-2">
          <svg className="w-4 h-4" fill="currentColor" viewBox="0 0 20 20">
            <path fillRule="evenodd" d="M18 10a8 8 0 11-16 0 8 8 0 0116 0zm-7 4a1 1 0 11-2 0 1 1 0 012 0zm-1-9a1 1 0 00-1 1v4a1 1 0 102 0V6a1 1 0 00-1-1z" clipRule="evenodd" />
          </svg>
          {error}
        </p>
      )}
    </div>
  );
});

// ده مهم عشان React يعرف اسم المكون
Input.displayName = 'Input';

export default Input;

شرح كل جزء في الكود

  • forwardRef: ده Hook من React بيسمحلك تمرر ref للمكون. في المكونات العادية، الـ ref بيوصلك للـ HTML element بس في المكونات المخصصة، محتاج forwardRef عشان يشتغل. مثلاً لو عايز الـ input يحصل focus تلقائياً لما الصفحة تفتح، محتاج ref
  • ref: ده بيتم تمريره للـ input عشان الـ parent يقدر يتحكم فيه مباشرة. مثلاً لو عايز تكتب في الحقل من كود JavaScript
  • label: لو موجود، بيظهر فوق الحقل. لو مش موجود، مش بيظهر حاجة. الشرط {label && ...} بيخلي الـ label اختياري
  • error: لو موجود، بيغير لون الحقل لأحمر وبيظهر رسالة الخطأ تحته. ده بيساعد المستخدم يعرف فين الغلط
  • type: نوع الحقل. القيمة الافتراضية text. تقدر تغيرها لـ email أو password أو number أو غيرهم
  • placeholder: ده نص توضيحي بيظهر جوه الحقل فاضي. مثلاً "أدخل بريدك الإلكتروني"

🃏 إنشاء مكون Card (البطاقة)

مكون Card هو المكون اللي بيعرض المعلومات بشكل منظم وجميل. في المتجر هنستخدمه في:

  • عرض المنتجات
  • عروض التخفيضات
  • معلومات المستخدم
  • ملخص الطلب
  • أي محتوى عايزه يكون في بطاقة منظمة

إيه هو تأثير الـ Glass؟

تأثير الـ Glass (الزجاج المصنفر) هو تأثير بيخلي البطاقة شفافة شوية وبتمسك الزجاج المصنفر اللي وراها. النتيجة: شكل حديث واحترافي جداً. خصوصاً مع الخلفية الداكنة اللي في المتجر بتاعنا.

components/ui/Card.jsx jsx 
import { cn } from '@/lib/cn';

// ============================================
// شرح مكون Card
// ============================================

// الخطوة 1: تعريف الأنواع
const cardVariants = {
  // النوع الافتراضي: خلفية شفافة قليلاً مع الزجاج المصنفر
  default: "bg-white/5 backdrop-blur-lg border border-white/10",

  // النوع الزجاجي: شفاف أكتر مع حدود أوضح
  glass: "bg-white/10 backdrop-blur-xl border border-white/20",

  // النوع الصلب: خلفية صلبة من غير شفافية
  solid: "bg-gray-800 border border-gray-700",

  // النوع المتدرج: تدرج ألوان من الأعلى للاسفل
  gradient: "bg-gradient-to-br from-white/10 to-white/5 backdrop-blur-lg border border-white/20",
};

// الخطوة 2: بناء المكون الرئيسي
export default function Card({
  children,        // محتوى البطاقة
  variant = "default", // النوع الافتراضي
  hover = true,    // هل نضيف تأثير عند المرور؟
  className,       // فئات إضافية
  ...props         // أي خصائص تانية
}) {
  return (
    <div
      className={cn(
        // الفئات الأساسية
        "rounded-2xl p-6 shadow-xl",
        // النوع
        cardVariants[variant],
        // تأثير Hover: بيكبر البطاقة شوية لما تمرر الماوس
        hover && "hover:scale-105 hover:shadow-2xl transition-all duration-300",
        // الفئات الإضافية
        className
      )}
      {...props}
    >
      {children}
    </div>
  );
}

// ============================================
// مكونات فرعية للـ Card (Compound Components)
// ============================================

// مكون Card.Header: الجزء العلوي من البطاقة (عادة العنوان)
Card.Header = function CardHeader({ children, className }) {
  return (
    <div className={cn("mb-4", className)}>
      {children}
    </div>
  );
};

// مكون Card.Content: الجزء الرئيسي من البطاقة (المحتوى)
Card.Content = function CardContent({ children, className }) {
  return (
    <div className={cn("", className)}>
      {children}
    </div>
  );
};

// مكون Card.Footer: الجزء السفلي من البطاقة (عادة الأزرار)
Card.Footer = function CardFooter({ children, className }) {
  return (
    <div className={cn("mt-4 pt-4 border-t border-white/10", className)}>
      {children}
    </div>
  );
};

إيه هو نمط Compound Components؟

لاحظ إن مكون Card بيلحق عليه مكونات فرعية (Header, Content, Footer). ده نمط اسمه Compound Components. المعنى:

  • المكون الرئيسي (Card) بيكون الهيكل العام (البطاقة نفسها)
  • المكونات الفرعية (Header, Content, Footer) بتكون الأجزاء جوه البطاقة
  • كل مكون فرعي ليه تنسيقاته الخاصة (Header فيه margin تحت، Footer فيه خط فاصل)
  • النمط ده بيخلّي الكود مرتب وواضح وسهل التعديل

💡 متى نستخدم كل نوع Card؟
  • default: للبطاقات العادية في الخلفية الداكنة

  • solid: للبطاقات اللي محتاجة خلفية صلبة (زي النماذج)
  • gradient: للبطاقات اللي عايزها تبان فاخرة
  • hover={false}: استخدمه لما البطاقة تكون interactable بس ما تبيج كـ hover

📦 إنشاء مكون Container (الحاوية)

مكون Container هو مكون بسيط بس مهم جداً. دوره إنه يحدد عرض المحتوى ويوصله بالنص عشان يبقى سهل القراءة. من غير Container، المحتوى هيملا الشاشة كلها والقراءة هتكون صعبة جداً.

ليه محتاجين Container؟

فكّر كده: لما بتفتح كتاب، النص مش بيكون من حافة الصفحة لحافتها. فيه مساحة فاضية على اليمين والشمال. نفس الفكرة بتطبق على الويب. المحتوى محتاج مساحة فاضية حوليه عشان يبقى واضح وسهل القراءة.

components/ui/Container.jsx jsx 
import { cn } from '@/lib/cn';

// ============================================
// شرح مكون Container
// ============================================

// الخطوة 1: تعريف الأحجام
const containerVariants = {
  // الافتراضي: عرض أقصى 1200px ومتوسط في الصفحة
  default: "max-w-7xl mx-auto px-4 sm:px-6 lg:px-8",

  // ضيق: عرض أقصى 800px (مناسب للمقالات والنصوص)
  narrow: "max-w-4xl mx-auto px-4 sm:px-6 lg:px-8",

  // عريض: عرض أقصى بدون حد
  wide: "max-w-full mx-auto px-4 sm:px-6 lg:px-8",

  // مرن: عرض 100% بدون أي حدة
  fluid: "w-full px-4 sm:px-6 lg:px-8",
};

// الخطوة 2: بناء المكون
export default function Container({
  children,        // المحتوى اللي جوه الحاوية
  variant = "default", // الحجم الافتراضي
  as: Component = "div", // الـ HTML element اللي هيتحط عليه المحتوى
  className,       // فئات إضافية
  ...props         // أي خصائص تانية
}) {
  return (
    <Component
      className={cn(containerVariants[variant], className)}
      {...props}
    >
      {children}
    </Component>
  );
}

شرح خاصية "as" (Polymorphic Component)

ملاحظة مهمة: المكون ده بيستخدم as: Component = "div". ده معناه إنك تقدر تغير الـ HTML element اللي بيستخدم. مثلاً:

  • <Container as="section">: بيستخدم <section> بدل <div>
  • <Container as="main">: بيستخدم <main> بدل <div>
  • <Container as="article">: بيستخدم <article> بدل <div>

الميزة دي مهمة للـ SEO (تحسين محركات البحث) والهيكلة السليمة للصفحة.

🏷️ إنشاء مكون Badge (الشارة)

مكون Badge هو تاق صغير بيستخدم لإظهار معلومات سريعة. في المتجر هنستخدمه في:

  • badge الخصم على المنتج (مثلاً: "خصم 50%")
  • حالة المخزون (متوفر، نفذ، قليل)
  • تصنيف المنتج (جديد، الأكثر مبيعاً، عرض خاص)
  • عدد الإشعارات (3 إشعارات جديدة)
  • حالة الطلب (قيد التنفيذ، تم التوصيل)

import { cn } from '@/lib/cn';

// ============================================
// شرح مكون Badge
// ============================================

// الخطوة 1: تعريف الأنواع
const badgeVariants = {
  // الافتراضي: أبيض شفاف
  default: "bg-white/10 text-white",

  // بنفسجي: للتمييز والميزات
  primary: "bg-indigo-500/20 text-indigo-300 border border-indigo-500/30",

  // أخضر: للحالة الإيجابية (متوفر، نجح)
  success: "bg-green-500/20 text-green-300 border border-green-500/30",

  // أصفر: للتحذيرات (قليل، يحتاج مراجعة)
  warning: "bg-yellow-500/20 text-yellow-300 border border-yellow-500/30",

  // أحمر: للحالة السلبية (نفذ، خطأ)
  danger: "bg-red-500/20 text-red-300 border border-red-500/30",

  // أزرق: للمعلومات (جديد، قادم)
  info: "bg-blue-500/20 text-blue-300 border border-blue-500/30",
};

// الخطوة 2: تعريف الأحجام
const badgeSizes = {
  sm: "px-2 py-0.5 text-xs",      // صغير
  md: "px-3 py-1 text-sm",        // متوسط
  lg: "px-4 py-1.5 text-base",    // كبير
};

// الخطوة 3: بناء المكون
export default function Badge({
  children,        // النص اللي جوه الشارة
  variant = "default", // النوع الافتراضي
  size = "md",     // الحجم الافتراضي
  dot = false,     // هل نضيف نقطة صغيرة جنب النص؟
  className,       // فئات إضافية
  ...props         // أي خصائص تانية
}) {
  return (
    <span
      className={cn(
        // الفئات الأساسية
        "inline-flex items-center gap-1.5 rounded-full font-medium",
        // النوع
        badgeVariants[variant],
        // الحجم
        badgeSizes[size],
        // الفئات الإضافية
        className
      )}
      {...props}
    >
      {/* لو dot=true، نظهر نقطة صغيرة جنب النص */}
      {dot && (
        <span className={cn(
          "w-1.5 h-1.5 rounded-full",
          // لون النقطة حسب النوع
          variant === "success" && "bg-green-400",
          variant === "warning" && "bg-yellow-400",
          variant === "danger" && "bg-red-400",
          variant === "info" && "bg-blue-400",
          variant === "primary" && "bg-indigo-400",
          variant === "default" && "bg-white"
        )} />
      )}
      {children}
    </span>
  );
}

🪟 إنشاء مكون Modal (النافذة المنبثقة)

مكون Modal هو نافذة بتبان فوق الصفحة كلها. بتتستخدم لما عايز تأكد من حاجة أو تعرض معلومات إضافية من غير ما تنقل المستخدم لصفحة تانية. في المتجر هنستخدمه في:

  • تأكيد الحذف (مثلاً: "هل أنت متأكد من حذف المنتج؟")
  • عرض تفاصيل المنتج
  • تأكيد الشراء
  • عرض صورة المنتج بالحجم الكبير
components/ui/Modal.jsx jsx 
"use client";

import { useEffect, useRef } from 'react';
import { cn } from '@/lib/cn';
import { X } from 'lucide-react';

// ============================================
// شرح مكون Modal
// ============================================

export default function Modal({
  isOpen,         // هل النافذة مفتوحة؟
  onClose,        // الدالة اللي بتتنفذ لما تضغط X أو على الخلفية
  title,          // عنوان النافذة
  children,       // المحتوى جوه النافذة
  size = "md",    // الحجم: sm, md, lg
  className,      // فئات إضافية
}) {
  // Hook عشان نقدر نغلق النافذة بالـ Escape key
  const modalRef = useRef(null);

  useEffect(() => {
    const handleEscape = (e) => {
      if (e.key === 'Escape' && isOpen) {
        onClose();
      }
    };

    document.addEventListener('keydown', handleEscape);
    return () => document.removeEventListener('keydown', handleEscape);
  }, [isOpen, onClose]);

  // لو النافذة مش مفتوحة، نرجع null (مش بنعرض حاجة)
  if (!isOpen) return null;

  // أحجام النافذة
  const sizes = {
    sm: "max-w-sm",    // صغير
    md: "max-w-md",    // متوسط
    lg: "max-w-lg",    // كبير
  };

  return (
    <div className="fixed inset-0 z-50 flex items-center justify-center">
      {/* الخلفية المعتمة */}
      <div
        className="absolute inset-0 bg-black/50 backdrop-blur-sm"
        onClick={onClose}
      />

      {/* النافذة نفسها */}
      <div
        ref={modalRef}
        className={cn(
          "relative bg-gray-900 rounded-2xl border border-white/10 shadow-2xl p-6",
          sizes[size],
          className
        )}
      >
        {/* رأس النافذة (العنوان وزر الإغلاق) */}
        <div className="flex items-center justify-between mb-4">
          <h3 className="text-lg font-semibold text-white">{title}</h3>
          <button
            onClick={onClose}
            className="p-1 hover:bg-white/10 rounded-lg transition-colors"
          >
            <X className="w-5 h-5 text-gray-400" />
          </button>
        </div>

        {/* محتوى النافذة */}
        {children}
      </div>
    </div>
  );
}

شرح الكود

  • isOpen: ده boolean بيحدد هل النافذة مفتوحة ولا لا. لو false، المكون بيرجع null ومش بيعرض حاجة
  • onClose: ده callback function بتتنفذ لما المستخدم يغلق النافذة. لازم تenaamel ده من البره
  • useRef: ده Hook بيستخدم عشان نقدر نوصل للنافذة مباشرة. مثلاً عشان نعمل focus على أول حقل
  • Escape key: الكود بيتأكد إنه لو المستخدم ضغط Escape، النافذة هتتغلق
  • الخلفية المعتمة: لما النافذة تتفتح، الصفحة كلها بتبقى معتمة ومش فعّالة

🔔 إنشاء مكون Toast (نظام الإشعارات)

مكون Toast هو نظام الإشعارات اللي بيظهر في أسفل الشاشة عشان يبلغ المستخدم بنتيجة الإجراء اللي عمله. مثلاً:

  • لما يضيف منتج للسلة: Toast أخضر "تمت الإضافة بنجاح!"
  • لما يكون في خطأ: Toast أحمر "حدث خطأ، حاول مرة أخرى"
  • لما ينسخ حاجة: Toast أزرق "تم النسخ!"

المكون ده معقد شوية لأن فيه أجزاء كتير:

  • Context: عشان يتبث في كل مكان في التطبيق
  • Provider: عشان يغطي كل التطبيق
  • Hook: عشان أي مكون يقدر يستخدم الـ Toast من غير ما يمرر props
components/ui/Toast.jsx jsx 
"use client";

import { useState, useEffect, createContext, useContext } from 'react';
import { cn } from '@/lib/cn';
import { X, CheckCircle, AlertCircle, Info, AlertTriangle } from 'lucide-react';

// ============================================
// شرح مكون Toast
// ============================================

// الخطوة 1: إنشاء Context عشان أي مكون يقدر يستخدم الـ Toast
const ToastContext = createContext(null);

// الخطوة 2: تعريف أنواع Toast
const toastTypes = {
  success: {
    icon: CheckCircle,
    className: "bg-green-500/20 border-green-500/30 text-green-300",
  },
  error: {
    icon: AlertCircle,
    className: "bg-red-500/20 border-red-500/30 text-red-300",
  },
  warning: {
    icon: AlertTriangle,
    className: "bg-yellow-500/20 border-yellow-500/30 text-yellow-300",
  },
  info: {
    icon: Info,
    className: "bg-blue-500/20 border-blue-500/30 text-blue-300",
  },
};

// الخطوة 3: إنشاء الـ Provider
export function ToastProvider({ children }) {
  // حالة بتخزن كل الـ Toasts الحالية
  const [toasts, setToasts] = useState([]);

  // دالة لإضافة Toast جديد
  const addToast = (message, type = "info", duration = 3000) => {
    const id = Date.now(); // معرف فريد لكل Toast
    setToasts(prev => [...prev, { id, message, type }]);

    // بعد المدة المحددة، الـ Toast بيتشال تلقائياً
    setTimeout(() => {
      removeToast(id);
    }, duration);
  };

  // دالة لإزالة Toast
  const removeToast = (id) => {
    setToasts(prev => prev.filter(toast => toast.id !== id));
  };

  return (
    <ToastContext.Provider value={{ addToast, removeToast }}>
      {children}

      {/* حاوية الـ Toasts: بتظهر في أسفل يمين الشاشة */}
      <div className="fixed bottom-4 right-4 z-50 flex flex-col gap-2">
        {toasts.map(toast => (
          <Toast
            key={toast.id}
            message={toast.message}
            type={toast.type}
            onClose={() => removeToast(toast.id)}
          />
        ))}
      </div>
    </ToastContext.Provider>
  );
}

// الخطوة 4: Hook لاستخدام Toast في أي مكون
export function useToast() {
  const context = useContext(ToastContext);
  if (!context) {
    throw new Error("useToast must be used within ToastProvider");
  }
  return context;
}

// الخطوة 5: مكون Toast نفسه
function Toast({ message, type = "info", onClose }) {
  const { icon: Icon, className } = toastTypes[type];

  return (
    <div
      className={cn(
        "flex items-center gap-3 px-4 py-3 rounded-xl border backdrop-blur-lg shadow-xl animate-slide-in",
        className
      )}
    >
      <Icon className="w-5 h-5 flex-shrink-0" />
      <p className="flex-1 text-sm font-medium">{message}</p>
      <button
        onClick={onClose}
        className="p-1 hover:bg-white/10 rounded-lg transition-colors"
      >
        <X className="w-4 h-4" />
      </button>
    </div>
  );
}

شرح الكود

  • ToastContext: ده Context من React بيسمح لأي مكون في التطبيق يوصل لدالة addToast
  • ToastProvider: ده المزود (Provider) اللي بيغطي التطبيق كله وبيخلي الـ Toast متاح في كل مكان. لازم تحطه في الملف الرئيسي (layout.js)
  • useToast(): ده Hook مخصص بيسمحلك تستخدم الـ Toast من أي مكون. بترجع object فيه addToast
  • addToast(message, type, duration): الدالة اللي بتضيف Toast جديد. type ممكن يكون success, error, warning, info. وduration هو المدة بالميللي ثانية (3000 = 3 ثواني)
  • setTimeout: بعد المدة المحددة، الـ Toast بيتشال تلقائياً

📄 إنشاء مكون Pagination (التنقل بين الصفحات)

مكون Pagination بيستخدم لما المحتوى كتير ومحتاجين نقسمه على صفحات كتير. مثلاً لو عندك 100 منتج، مش هتعرضهم كلهم في صفحة واحدة. هتعرض 10 في كل صفحة وتحط أرقام الصفحات تحت.

components/ui/Pagination.jsx jsx 
import { cn } from '@/lib/cn';
import { ChevronLeft, ChevronRight } from 'lucide-react';

// ============================================
// شرح مكون Pagination
// ============================================

export default function Pagination({
  currentPage,    // الصفحة الحالية
  totalPages,     // إجمالي عدد الصفحات
  onPageChange,   // الدالة اللي بتتنفذ لما المستخدم يضغط على رقم صفحة
  className,      // فئات إضافية
}) {
  // دالة لإنشاء مصفوفة أرقام الصفحات
  const getPageNumbers = () => {
    const pages = [];

    // لو العدد صغير، نعرض كل الأرقام
    if (totalPages <= 7) {
      for (let i = 1; i <= totalPages; i++) {
        pages.push(i);
      }
    } else {
      // لو العدد كبير، نعرض أول صفحة وآخر صفحة ونقاط
      pages.push(1);
      if (currentPage > 3) pages.push('...');
      for (let i = Math.max(2, currentPage - 1); i <= Math.min(totalPages - 1, currentPage + 1); i++) {
        pages.push(i);
      }
      if (currentPage < totalPages - 2) pages.push('...');
      pages.push(totalPages);
    }

    return pages;
  };

  return (
    <div className={cn("flex items-center gap-2", className)}>
      {/* زرار الصفحة السابقة */}
      <button
        onClick={() => onPageChange(currentPage - 1)}
        disabled={currentPage === 1}
        className="p-2 rounded-lg hover:bg-white/10 disabled:opacity-50 disabled:cursor-not-allowed"
      >
        <ChevronRight className="w-5 h-5" />
      </button>

      {/* أرقام الصفحات */}
      {getPageNumbers().map((page, index) => (
        <button
          key={index}
          onClick={() => typeof page === 'number' && onPageChange(page)}
          className={cn(
            "w-10 h-10 rounded-lg font-medium transition-colors",
            page === currentPage
              ? "bg-indigo-500 text-white"
              : page === '...'
              ? "text-gray-400 cursor-default"
              : "hover:bg-white/10 text-gray-300"
          )}
          disabled={page === '...'}
        >
          {page}
        </button>
      ))}

      {/* زرار الصفحة التالية */}
      <button
        onClick={() => onPageChange(currentPage + 1)}
        disabled={currentPage === totalPages}
        className="p-2 rounded-lg hover:bg-white/10 disabled:opacity-50 disabled:cursor-not-allowed"
      >
        <ChevronLeft className="w-5 h-5" />
      </button>
    </div>
  );
}

🏆 أفضل الممارسات (Best Practices)

💡 قواعد لازم تمشي عليها لما بتبني مكونات
  1. اجعل المكون بسيط وواضح: كل مكون يبقى مسؤول عن حاجة واحدة بس. مكون Button يبقى بيعمل زرار بس، مش زرار وقائمة ونافذة مع بعض
  2. استخدم الـ variants بدل الـ useState: لو عايز تغير الشكل بناءً على حالة، استخدم Object بدل ما تعمل useState لكل حالة. ده أسرع وأوضح
  3. دايماً اقبل className إضافية: كل مكون لازم يقبل className عشان المستخدم يقدر يخصصه
  4. استخدم cn() في كل مكان: بدل ما تدمج فئات يدوياً، استخدم cn()
  5. اكتب أسماء واضحة للمتغيرات: buttonVariants أوضح من btnVar
  6. اضاف تعليقات على الأجزاء المعقدة: لو كود معقد، اكتب تعليق يوضح ليه كاتبه كده
  7. اختبر المكونات في أماكن مختلفة: تأكد إن المكون يشتغل صح في كل الأماكن اللي هتستخدمه فيها
  8. افصل بين المكونات العامة والمكونات الخاصة: المكونات العامة في مجلد ui، والمكونات الخاصة في أماكنها
  9. استخدم TypeScript لو تقدر: TypeScript بيساعدك تلاقي الأخطاء قبل ما تشغل الكود
  10. اكتب PropTypes لو مش بتستخدم TypeScript: عشان تتأكد إنك بتمرر البيانات الصح

أخطاء يجب تجنبها

⚠️ الأخطاء اللي لازم تتجنبها
  • الخطأ 1: استخدام <button> عادي بدل <Button> — النتيجة: مش هيبقى عليه التنسيقات الحلوة
  • الخطأ 2: نسيان "use client" في المكونات اللي فيها تفاعل — النتيجة: هتاخد خطأ في الـ console
  • الخطأ 3: استخدام ألوان Tailwind مباشرة بدل الـ variants — النتيجة: الكود يبقى مكرر وصعب الصيانة
  • الخطأ 4: نسيان disabled على الأزرار أثناء التحميل — النتيجة: المستخدم ممكن يضغط كتير ويسبب مشاكل
  • الخطأ 5: نسيان تمرير className في المكونات — النتيجة: المستخدم مش هيقدر يخصص المكون
  • الخطأ 6: استخدام useState في كل مرة بدل الـ variants — النتيجة: كود مكرر وأداء أبطأ
  • الخطأ 7: عمل مكون كبير فيه كل حاجة — النتيجة: صعب الصيانة وصعب إعادة الاستخدام
  • الخطأ 8: نسيان الـ accessibility (الإمكانية) — النتيجة: بعض المستخدمين مش هيعرفوا يستخدموا الموقع
