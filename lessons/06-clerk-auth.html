🔐什么是 Clerk؟

Clerk هو خدمة للمصادقة (Authentication) بتقدم واجهات جاهزة و API سهلة. بتوفر تسجيل الدخول والخروج وإدارة المستخدمين بسهولة.

مميزات Clerk:

  • واجهات جاهزة - مكونات UI جاهزة للتسجيل
  • دعم متعدد - Email, Password, Social Login
  • Middleware - حماية المسارات بسهولة
  • User Management - إدارة المستخدمين
  • Webhooks - تحديثات مباشرة
  • مجاني - خطة مجانية لـ 10,000 مستخدم

🚀 الخطوة 1: إنشاء مشروع Clerk

1.1 التسجيل في Clerk

  1. ادخل على https://clerk.com
  2. اضغط "Get Started"
  3. سجّل الدخول باستخدام GitHub أو Email

1.2 إنشاء مشروع جديد

  1. من Dashboard، اضغط "Add Application"
  2. أدخل تفاصيل التطبيق:
    • Application name: Market
    • Supported languages: Arabic
  3. اختار طرق تسجيل الدخول (مثلاً: Email, Google)
  4. اض唾 "Create application"
💡 نصيحة
بعد الإنشاء، هتلاقي API keys في Dashboard → API Keys. محتاجين Publishable Key و Secret Key.

📦 الخطوة 2: تثبيت Clerk في Next.js

2.1 تثبيت الحزمة

Terminal bash 
npm install @clerk/nextjs

2.2 إضافة متغيرات البيئة

أضف المفاتيح التالية في ملف .env.local:

.env.local bash 
# Clerk Authentication
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_your-key-here
CLERK_SECRET_KEY=sk_test_your-key-here

# Clerk URLs (اختياري - بيستخدمها Clerk تلقائياً)
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/
NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/

2.3 إضافة ClerkProvider

محتاجين نلّف التطبيق بـ ClerkProvider:

app/layout.js javascript 
import { ClerkProvider } from '@clerk/nextjs';
import "./globals.css";

export const metadata = {
  title: "Market - متجر إلكتروني",
  description: "متجر إلكتروني كامل",
};

export default function RootLayout({ children }) {
  return (
    <ClerkProvider>
      <html lang="ar">
        <body>
          {children}
        </body>
      </html>
    </ClerkProvider>
  );
}

📄 الخطوة 3: إضافة صفحات تسجيل الدخول

3.1 إنشاء صفحة تسجيل الدخول

📁 app/
├── 📁 (auth)/ # مسار Authentication
│ ├── 📁 sign-in/
│ │ └── 📄 [[...sign-in]]/page.js
│ ├── 📁 sign-up/
│ │ └── 📄 [[...sign-up]]/page.js
│ └── 📄 layout.js
├── 📄 layout.js
└── 📄 page.js
app/(auth)/sign-in/[[...sign-in]]/page.js javascript 
import { SignIn } from '@clerk/nextjs';

export default function SignInPage() {
  return (
    <div className="flex items-center justify-center min-h-screen">
      <SignIn />
    </div>
  );
}
app/(auth)/sign-up/[[...sign-up]]/page.js javascript 
import { SignUp } from '@clerk/nextjs';

export default function SignUpPage() {
  return (
    <div className="flex items-center justify-center min-h-screen">
      <SignUp />
    </div>
  );
}
💡 شرح [[...sign-in]]
الأقواس المزدوجة [[...]] بتعني optional catch-all route. يعني الصفحة هتشتغل على /sign-in و كمان /sign-in/sso-callback وغيرها.

3.2 Layout مخصص لصفحات Auth

app/(auth)/layout.js javascript 
export default function AuthLayout({ children }) {
  return (
    <div className="min-h-screen flex items-center justify-center bg-gray-50">
      {children}
    </div>
  );
}

🧭 الخطوة 4: إضافة UserButton

UserButton هو مكون جاهز بيعرض صورة المستخدم ويوفر قائمة بخيارات الحساب.

components/Header.js javascript 
"use client";

import { UserButton, SignedIn, SignedOut } from '@clerk/nextjs';
import Link from 'next/link';

export default function Header() {
  return (
    <header className="bg-white shadow-md">
      <div className="max-w-7xl mx-auto px-4 py-4 flex justify-between items-center">
        <Link href="/" className="text-2xl font-bold text-primary-600">
          Market
        </Link>
        
        <nav className="flex items-center gap-4">
          <Link href="/products">المنتجات</Link>
          
          <!-- إذا كان المستخدم مسجل الدخول -->
          <SignedIn>
            <UserButton 
              afterSignOutUrl="/"
              userProfileMode="modal"
            />
          </SignedIn>
          
          <!-- إذا لم يكن المستخدم مسجلاً -->
          <SignedOut>
            <Link 
              href="/sign-in"
              className="bg-primary-500 text-white px-4 py-2 rounded-lg"
            >
              تسجيل الدخول
            </Link>
          </SignedOut>
        </nav>
      </div>
    </header>
  );
}

🎣 الخطوة 5: استخدام useAuth hook

Hook الـ useAuth بيديك معلومات عن حالة المستخدم:

components/UserInfo.js javascript 
"use client";

import { useAuth, useUser } from '@clerk/nextjs';

export default function UserInfo() {
  const { userId, sessionId, getToken } = useAuth();
  const { user } = useUser();
  
  if (!user) {
    return <p>لم تسجل الدخول</p>;
  }
  
  return (
    <div className="p-4 bg-gray-100 rounded-lg">
      <h2>معلومات المستخدم</h2>
      <p>الاسم: {user.fullName}</p>
      <p>البريد: {user.primaryEmailAddress?.emailAddress}</p>
      <p>المعرّف: {userId}</p>
      <img 
        src={user.imageUrl} 
        alt="صورة المستخدم"
        className="w-16 h-16 rounded-full"
      />
    </div>
  );
}

الحصول على Token للاتصال بـ Supabase

获取 JWT Token javascript 
"use client";

import { useAuth } from '@clerk/nextjs';
import { useEffect } from 'react';

export default function TokenExample() {
  const { getToken } = useAuth();
  
  useEffect(() => {

      // الحصول على JWT token

    getToken();
  }, [getToken]);
  
  return <div>جاري تحميل Token...</div>;
}

🛡️ الخطوة 6: حماية الصفحات

6.1 استخدام SignedIn و SignedOut

التحكم في عرض المحتوى javascript 
import { SignedIn, SignedOut } from '@clerk/nextjs';

export default function ProtectedPage() {
  return (
    <div>
      <!-- محتوى للجميع -->
      <h1>مرحباً بكم في متجرنا</h1>
      
      <!-- محتوى للمسجلين فقط -->
      <SignedIn>
        <div>
          <h2>مرحباً، أنت مسجل الدخول!</h2>
          <p>هنا تقدر تشوف طلباتك ومنتجاتك المفضلة</p>
        </div>
      </SignedIn>
      
      <!-- محتوى للضيوف فقط -->
      <SignedOut>
        <div>
          <h2>سجل الدخول عشان تستمتع بمميزات إضافية</h2>
          <a href="/sign-in">تسجيل الدخول</a>
        </div>
      </SignedOut>
    </div>
  );
}
