🔐 إعداد متغيرات البيئة

المتغيرات البيئية (Environment Variables) ضرورية لتخزين المفاتيح السرية. في Next.js، نستخدم ملف .env.local للتطوير المحلي.

⚠️ تحذير أمني
لا ترفع ملف .env.local إلى GitHub أو أي مستودع عام. تأكد من وجوده في ملف .gitignore.
.env.local bash 
# Clerk Authentication
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
CLERK_SECRET_KEY=sk_test_...
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/
NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/

# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...

# Site URL
NEXT_PUBLIC_SITE_URL=http://localhost:3000

# Stripe (optional)
STRIPE_SECRET_KEY=sk_test_...
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...

إعداد Environment Variables على Vercel

بعد رفع المشروع على Vercel، يجب إضافة المتغيرات البيئية من خلال لوحة التحكم:

  1. افتح لوحة تحكم Vercel واختر مشروعك
  2. اذهب إلى Settings ثم Environment Variables
  3. أضف كل متغير بشكل منفصل مع القيم الصحيحة
  4. اختر البيئات (Production, Preview, Development)
  5. اضغط Save لحفظ كل متغير

🚀 نشر المشروع على Vercel

Vercel هو أفضل منصة لنشر مشاريع Next.js. إليك الخطوات:

الطريقة الأولى: من خلال واجهة Vercel

خطوات النشر text 
1. اذهب إلى vercel.com وسجل دخول بحسابك
2. اضغط "New Project"
3. اختر "Import Git Repository"
4. اختر مستودع GitHub الخاص بك
5. اضغط "Import"
6. تأكد من إعدادات البناء:
   - Framework Preset: Next.js
   - Build Command: npm run build
   - Output Directory: .next
7. أضف Environment Variables
8. اضغط "Deploy"

الطريقة الثانية: من خلال CLI

تثبيت ونشر عبر CLI bash 
# تثبيت Vercel CLI
npm install -g vercel

# تسجيل الدخول
vercel login

# نشر المشروع (للتطوير)
vercel

# نشر للإنتاج
vercel --prod

إعدادات next.config.js للنشر

next.config.js javascript 
/** @type {import('next').NextConfig} */
const nextConfig = {
  // تحسين الأداء للإنتاج
  experimental: {
    optimizePackageImports: ['lucide-react', '@radix-ui/react-icons']
  },
  
  // إعدادات الصور
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: '**.supabase.co'
      }
    ],
    formats: ['image/avif', 'image/webp']
  },
  
  // تعطيل Strict Mode في الإنتاج
  reactStrictMode: true,
  
  // تحسين الحزم
  webpack: (config, { isServer }) => {
    if (!isServer) {
      config.resolve.fallback = {
        ...config.resolve.fallback,
        fs: false,
        net: false,
        tls: false
      };
    }
    return config;
  }
};

module.exports = nextConfig;

🌐 إعداد نطاق مخصص (Custom Domain)

بعد النشر، ستحصل على رابط مثل market.vercel.app. يمكنك تغييره لنطاق مخصص.

خطوات إعداد النطاق المخصص

  1. شراء النطاق من مسجل نطاقات (مثل Namecheap, GoDaddy)
  2. الذهاب إلى Vercel Dashboard → مشروعك → Settings → Domains
  3. إضافة النطاق وكتابته (مثلاً: market.com)
  4. تحديث DNS Records chez مسجل النطاق:
    • Type: A
    • Name: @
    • Value: 76.76.21.21
  5. انتظار التفعيل (قد يستغرق 24-48 ساعة)
DNS Records text 
# للنطاق الرئيسي
Type: A
Name: @
Value: 76.76.21.21

# للنطاق الفرعي (www)
Type: CNAME
Name: www
Value: cname.vercel-dns.com
💡 تثبيت SSL تلقائي
Vercel يوفر شهادة SSL مجانية تلقائياً لكل نطاق. لا حاجة لتثبيت شهادة يدوياً.

✅ التحقق بعد النشر

بعد النشر، تأكد من فحص جميع الميزات:

ال检查 الرابط الملاحظات
الصفحة الرئيسية / تأكد من تحميل الصور والروابط
صفحة المنتجات /products تأكد من عمل البحث والفلتر
تفاصيل المنتج /products/[id] تأكد من عرض الصور والمعلومات
تسجيل الدخول /sign-in تأكد من عمل Clerk
Admin Dashboard /admin تأكد من حماية المسارات
API Routes /api/* تأكد من عمل endpoints
فحص الأداء bash 
# فحص الأداء باستخدام Lighthouse
npx lighthouse https://your-domain.com --output html

# فحص الروابط المكسورة
npx broken-link-checker https://your-domain.com

# فحص SEO
npx seo-checker https://your-domain.com

🔄 النشر التلقائي (CI/CD)

Vercel يوفر النشر التلقائي مع كل push إلى GitHub:

✅ ميزة النشر التلقائي

        {
