📥 الخطوة 1: تثبيت Node.js

Node.js هو بيئة تشغيل JavaScript على السيرفر. محتاجينه عشان نشغّل مشروع Next.js.

1.1 حمل Node.js

  1. ادخل على الموقع الرسمي: https://nodejs.org
  2. حمّل الإصدار LTS (Long Term Support) - ده الإصدار المستقر
  3. شغّل ملف التثبيت واتبع الخطوات
💡 نصيحة
اختار إصدار LTS لأن ده الإصدار المستقر والمناسب للمشاريع الإنتاجية. الإصدار المحدث (Current) قد يكون فيه بعض الأخطاء.

1.2 التحقق من التثبيت

بعد التثبيت، افتح Terminal واكتب الأوامر التالية للتأكد:

Terminal bash 
# التحقق من إصدار Node.js
node --version

# التحقق من إصدار npm
npm --version

لو ظهر أرقام الإصدارات (مثلاً v20.11.0 و 10.2.4)، يبقى التثبيت ناجح.

📦 الخطوة 2: إنشاء مشروع Next.js

Next.js هو إطار عمل React يوفر了很多 مميزات جاهزة زي App Router و Server Components.

2.1 إنشاء المشروع

افتح Terminal واكتب الأمر التالي:

Terminal bash 
npx create-next-app@latest market
⚠️ تنبيه
تأكد إنك متصلاً بالإنترنت أثناء تشغيل الأمر. كما قد يستغرق الأمر بضع دقائق للتحميل.

2.2 اختيارات التثبيت

هتظهر لك مجموعة من الأسئلة. اختار الإعدادات التالية:

Terminal bash 
Would you like to use TypeScript? → No
Would you like to use ESLint? → Yes
Would you like to use Tailwind CSS? → Yes
Would you like to use `src` directory? → No
Would you like to use App Router? → Yes
Would you like to use Turbopack for next dev? → Yes
Would you like to customize the import alias? → No

2.3 شرح الاختيارات

السؤال الاختيار السبب
TypeScript No هنشتغل على JavaScript عشان نبدأ بأسهل
ESLint Yes لاكتشاف الأخطاء مبكراً
Tailwind CSS Yes هنشتغل عليه في الدرس القادم
src directory No هيكل بسيط بدون مجلد src
App Router Yes أحدث نظام توجيه في Next.js
Turbopack Yes أسرع بكثير من Webpack

📁 الخطوة 3: فهم هيكل المشروع

بعد إنشاء المشروع، هتلاقي المجلدات والملفات التالية:

📁 market/
├── 📁 app/ # صفحات المشروع (App Router)
│ ├── 📄 layout.js # التخطيط الرئيسي
│ ├── 📄 page.js # الصفحة الرئيسية
│ ├── 📄 page.module.css # أنماط الصفحة الرئيسية
│ └── 📄 globals.css # الأنماط العامة
├── 📁 public/ # الملفات العامة (صور، أيقونات)
├── 📄 next.config.mjs # إعدادات Next.js
├── 📄 package.json # حزم المشروع
├── 📄 postcss.config.mjs # إعدادات PostCSS
├── 📄 tailwind.config.js # إعدادات Tailwind CSS
└── 📄 .gitignore # ملفات تجاهل Git

3.1 شرح المجلدات والملفات الرئيسية

📁 مجلد app/

ده المجلد الأساسي اللي فيه كل صفحات المشروع. في Next.js مع App Router، كل ملف في مجلد app يمثل صفحة أو مكون.

app/layout.js javascript 
import "./globals.css";

export const metadata = {
  title: "Market - متجر إلكتروني",
  description: "متجر إلكتروني كامل",
};

export default function RootLayout({ children }) {
  return (
    <html lang="ar">
      <body>
        {children}
      </body>
    </html>
  );
}
app/page.js javascript 
export default function Home() {
  return (
    <main>
      <h1>مرحباً بكم في متجرنا</h1>
      <p>ده أول صفحة في المشروع</p>
    </main>
  );
}

📄 ملف package.json

ده ملف بيحتوي على معلومات المشروع والـ dependencies (المكتبات المستخدمة).

package.json json 
{
  "name": "market",
  "version": "0.1.0",
  "private": true,
  "scripts": {
    "dev": "next dev --turbopack",
    "build": "next build",
    "start": "next start",
    "lint": "next lint"
  },
  "dependencies": {
    "next": "15.1.0",
    "react": "^19.0.0",
    "react-dom": "^19.0.0"
  },
  "devDependencies": {
    "@tailwindcss/postcss": "^4.0.0",
    "tailwindcss": "^4.0.0"
  }
}

🚀 الخطوة 4: تشغيل خادم التطوير

الآن خلّينا نشغّل المشروع ونشوف النتيجة.

4.1 الدخول لمجلد المشروع

Terminal bash 
cd market

4.2 تشغيل خادم التطوير

Terminal bash 
npm run dev
✅ نجاح
لو شفت الرسالة "Ready in 1.2s" أو something مشابه، يبقى المشروع اتشغّل بنجاح. افتح المتصفح على http://localhost:3000

4.3 اختبار الموقع

افتح المتصفح واذهب إلى:

المتصفح text 
http://localhost:3000

هتلاقي الصفحة الرئيسية لمشروع Next.js مع شعار Next.js وكلمة "Get started by editing app/page.js".

🔧 الخطوة 5: تثبيت المكتبات الإضافية

محتاجين نثبّت بعض المكتبات اللي هنستخدمها في المشروع:

Terminal bash 
# مكتبة الحركات والتأثيرات
npm install framer-motion

# مكتبة أيقونات
npm install lucide-react

# Clerk للمصادقة
npm install @clerk/nextjs

# Supabase لقاعدة البيانات
npm install @supabase/supabase-js
npm install @supabase/ssr

📝 ملخص أوامر npm المهمة

الأمر الوظيفة
npm run dev تشغيل خادم التطوير
npm run build بناء المشروع للإنتاج
npm run start تشغيل المشروع المبني
npm run lint فحص الأخطاء
npm install [package] تثبيت حزمة جديدة

🔍 استكشاف الأخطاء

مشكلة: الأمر لا يعمل

لو لم يتعرف على الأمر node أو npm:

  • أعد تشغيل الـ Terminal بعد تثبيت Node.js
  • تأكد من تثبيت Node.js بشكل صحيح
  • أعد تشغيل الكمبيوتر

مشكلة: خطأ في التثبيت

لو ظهر خطأ أثناء تثبيت الحزم:

Terminal bash 
# حذف مجلد node_modules وإعادة التثبيت
rm -rf node_modules
npm install

مشكلة: المنفذ مشغول

لو المنفذ 3000 مشغول، ممكن نغيره:

Terminal bash 
# تشغيل على منفذ آخر
npm run dev -- -p 3001
