إعداد بيئة التطوير

🎯 أهداف الدرس

أهلاً بيك في الدرس التاني! في الدرس ده هنعمل حاجة مهمة جداً وهي إعداد بيئة التطوير على جهازك. يعني إيه بيئة تطوير؟ ببساطة هي الأدوات اللي محتاجها عشان تقدر تكتب كود وتشغله وتشوف النتيجة.

خلّيني أقولك حاجة مهمة: إعداد بيئة التطوير صح من الأول بيوفر عليك وقت كبير بعدين. كتير من الطلاب بيقضوا ساعات في مشاكل سهلة كانت هتتلافى لو كان الإعداد صح.

  • تثبيت Node.js على جهازك
  • تثبيت Visual Studio Code (محرر الكود)
  • إنشاء مشروع Next.js جديد
  • فهم هيكل المشروع الأساسي
  • تشغيل خادم التطوير والتأكد من عمل المشروع
  • تثبيت المكتبات الأساسية اللي هنستخدمها
💡 ليه الدرس ده مهم؟
بدون بيئة تطوير صحيحة، مش هتقدر تشتغل على أي مشروع. تخيل إنك عايز تطبخ من غير مطبخ أو أدوات - إزاي هتقدر؟ نفس الشيء بالظبط! الدرس ده هيحطلك الأساس عشان أي درس جاي يبقى سهل عليك.

📋 المتطلبات المسبقة

قبل ما نبدأ، محتاجين نتأكد من توفر الأدوات التالية على جهازك. لو مش لاقي أي أداة من دول، متقلقش - هنقولك إزاي تنزلها وثبّتها خطوة بخطوة.

الأداة الحد الأدنى التثبيت ليه محتاجها؟
Node.js 18.0 أو أعلى nodejs.org بيشغّل JavaScript على جهازك
npm 9.0 أو أعلى يتضمن مع Node.js بيثبت الحزم والمكتبات
VS Code أي إصدار حديث code.visualstudio.com محرر الكود الأساسي
Terminal أي terminal bash أو PowerShell لتشغيل الأوامر
Git أي إصدار git-scm.com لتتبع التغييرات (اختياري)
متصفح حديث Chrome أو Firefox مثبت مسبقاً لـ testing و debugging
⚠️ مهم جداً
لو عندك Windows قديم (أقدم من Windows 10)، ممكن بعض الأدوات ما تشتغلش كويس. كمان تأكد إنك عندك مساحة كافية على القرص الصلب (على الأقل 2GB فاضية).

📥 الخطوة 1: تثبيت Node.js

Node.js هو بيئة تشغيل JavaScript على السيرفر. يعني بيخلي JavaScript تشتغل برة المتصفح - على جهازك مباشرة. ده مهم جداً عشان Next.js (اللي هنستخدمه) بيعتمد عليه في التشغيل.

خلّيني أقولك حاجة بسيطة: JavaScript في الأصل كانت بتشتغل بس في المتصفح. Node.js جه وقال "ليه ما نشغّلهاش برة المتصفح كمان؟" وده خلا البرمجة بكاملها تتم على جهازك.

1.1 حمل Node.js

  1. ادخل على الموقع الرسمي: https://nodejs.org
  2. هتلاقي اختيارين: LTS و Current
  3. حمّل الإصدار LTS (Long Term Support) - ده الإصدار المستقر والمناسب للمشاريع الإنتاجية
  4. شغّل ملف التثبيت واتبع الخطوات (Next → Next → Install)
  5. استنى لين يخلّص التثبيت (ممكن ياخد 2-5 دقائق حسب سرعة جهازك)
💡 نصيحة
اختار إصدار LTS لأن ده الإصدار المستقر والمناسب للمشاريع الإنتاجية. الإصدار المحدث (Current) قد يكون فيه بعض الأخطاء لأنهم بيجربوا عليه مميزات جديدة. لحد ما تبقى متقدم، LTS هو الأفضل ليك.

1.2 التحقق من التثبيت

بعد التثبيت، لازم نتأكد إن كل حاجة اتثبّت صح. افتح Terminal (في Windows اضغط Windows + R واكتب cmd) واكتب الأوامر التالية:

Terminal bash 
# التحقق من إصدار Node.js
node --version

# التحقق من إصدار npm
npm --version

لو ظهر أرقام الإصدارات (مثلاً v20.11.0 و 10.2.4)، يبقى التثبيت ناجح. لو ظهرت رسالة خطأ زي "node is not recognized"، يبقى لسه محتاج تعيد تشغيل الـ Terminal أو الكمبيوتر.

💡 فرق بين npm و npx
npm (Node Package Manager) هو أداة لتثبيت الحزم. npx هو أداة لتشغيل الحزم بدون ما تثبّتها. مثلاً: npm install بيثبّت الحزمة، و npx create-next-app بيشغّل الحزمة مباشرة. في أكتر من 90% من الحالات، هتستخدم npx.

1.3 أخطاء شائعة في تثبيت Node.js

⚠️ الأخطاء الأكثر شيوعاً
  • رسالة "node is not recognized": يعني إن Node.js مش في الـ PATH. الحل: أعد تشغيل الـ Terminal أو أعد تشغيل الكمبيوتر
  • خطأ في الاتصال: تأكد إنك متصلاً بالإنترنت
  • صلاحيات مرفوضة: في Mac/Linux، استخدم sudo قبل الأمر
  • إصدار قديم: لو ظهر إصدار أقدم من 18، حمّل الإصدار الأحدث من الموقع

📝 الخطوة 1.5: تثبيت VS Code

Visual Studio Code (أو VS Code) هو محرر الكود الأشهر في العالم. بيقدم مميزات كتير بتسهّل عليك الكتابة والتعديل. مجاناً ومن سهل التثبيت.

1.5.1 حمل VS Code

  1. ادخل على: https://code.visualstudio.com
  2. اضغط "Download for Windows"
  3. شغّل الملف اللي اتنزّل
  4. اتبع خطوات التثبيت (استخدم الإعدادات الافتراضية)

1.5.2 إضافات مهمة لـ VS Code

بعد التثبيت، محتاجين نثبّت بعض الإضافات (Extensions) اللي هتسهّل عليك الشغل:

  • ES7+ React/Redux/React-Native snippets - بيوفر عليك كتابة أكواد React
  • Prettier - Code formatter - بينظّم الكود تلقائياً
  • ESLint - بيكتشف الأخطاء في الكود
  • Tailwind CSS IntelliSense - بيساعدك في كتابة أكواد Tailwind
  • Auto Rename Tag - بيعيد تسمية الـ HTML tags تلقائياً
  • Path Intellisense - بيكمّل مسارات الملفات
💡 إزاي تثبّت الإضافات
افتح VS Code، اضغط Ctrl+Shift+X عشان تفتح صفحة الإضافات، واكتب اسم الإضافة في خانة البحث واضغط Install على كل إضافة. الموضوع بسيط ومش هيياخد منك أكتر من 5 دقائق.

1.5.3 إعدادات مهمة في VS Code

في بعض الإعدادات اللي ممكن تساعدك في الشغل:

  • Auto Save: افتح الإعدادات (Ctrl+,) واكتب "auto save" وفعّلها عشان الملفات تنحفظ تلقائياً
  • Font Size: غيّر حجم الخط لـ 16 عشان يبقى مريح للعين
  • Format On Save: بينظّم الكود تلقائياً كل ما تحفظ الملف

📦 الخطوة 2: إنشاء مشروع Next.js

Next.js هو إطار عمل (Framework) مبني على React. بيقدم مميزات جاهزة كتير زي App Router و Server Components والـ Static Generation والـ Server Side Rendering. ببساطة، بيوفر عليك شغل كتير وبيخلي المشروع أسرع وأحسن.

في الخطوات الجاية هنستخدم الأمر npx create-next-app@latest عشان نإنشاء مشروع جديد. الأمر ده بيثبّت كل اللي محتاجه تلقائياً.

2.1 اختيار مكان المشروع

قبل ما تكتب الأمر، فكّر فين عايز تحط المشروع. مثلاً:

  • Desktop: cd Desktop
  • مجلد مخصص: cd "D:\My Projects"
  • جذر القرص: cd C:\
⚠️ اسم المجلد
متستخدمش مسافات أو أحرف خاصة في اسم المشروع. استخدم شرطة سفلية (_) أو شرطة (-) بدل المسافة. مثلاً: my-market ومش my market.

2.2 إنشاء المشروع

افتح Terminal واكتب الأمر التالي:

Terminal bash 
npx create-next-app@latest market
⚠️ تنبيه
تأكد إنك متصلاً بالإنترنت أثناء تشغيل الأمر. كما قد يستغرق الأمر بضع دقائق للتحميل. لو الاتصال ضعيف، ممكن ياخد أطول شوية - متقلقش واستنى.

2.3 اختيارات التثبيت

هتظهر لك مجموعة من الأسئلة. اختار الإعدادات التالية بالظبط:

Terminal bash 
Would you like to use TypeScript? → No
Would you like to use ESLint? → Yes
Would you like to use Tailwind CSS? → Yes
Would you like to use src directory? → No
Would you like to use App Router? → Yes
Would you like to use Turbopack for next dev? → Yes
Would you like to customize the import alias? → No

2.4 شرح كل سؤال بالتفصيل

خلّيني أشرحلك كل سؤال وليه اختارنا الإجابة دي:

السؤال الاختيار السبب بالتفصيل
TypeScript No TypeScript ممتاز بس هو لغة منفصلة عن JavaScript. هنشتغل على JavaScript عشان نبدأ بأسهل ونركز على الأساسيات
ESLint Yes أداة لاكتشاف الأخطاء مبكراً. بتحذرلك لو في مشاكل في الكود قبل ما تشغّله
Tailwind CSS Yes هنشتغل عليه في الدرس الجاي عشان التصميم. من الأفضل نثبّته دلوقتي
src directory No هيكل بسيط بدون مجلد src. في المشاريع الكبيرة المجلد ده بيكون مفيد، بس للمبتدئين الأفضل يبقى كل حاجة في المجلد الرئيسي
App Router Yes أحدث نظام توجيه في Next.js. الأفضل تتعلم عليه من الأول لأن ده المستقبل
Turbopack Yes أسرع بكثير من Webpack القديم. بيخلي تشغيل خادم التطوير أسرع 10 مرات

📁 الخطوة 3: فهم هيكل المشروع

بعد إنشاء المشروع، هتلاقي المجلدات والملفات التالية. فهم الهيكل ده مهم جداً عشان تعرف تتنقل في المشروع وتعمل فيه تعديلات.

📁 market/
├── 📁 app/ # صفحات المشروع (App Router)
│ ├── 📄 layout.js # التخطيط الرئيسي
│ ├── 📄 page.js # الصفحة الرئيسية
│ ├── 📄 page.module.css # أنماط الصفحة الرئيسية
│ └── 📄 globals.css # الأنماط العامة
├── 📁 public/ # الملفات العامة (صور، أيقونات)
├── 📄 next.config.mjs # إعدادات Next.js
├── 📄 package.json # حزم المشروع
├── 📄 postcss.config.mjs # إعدادات PostCSS
├── 📄 tailwind.config.js # إعدادات Tailwind CSS
└── 📄 .gitignore # ملفات تجاهل Git

3.1 شرح المجلدات والملفات الرئيسية

📁 مجلد app/

ده المجلد الأساسي اللي فيه كل صفحات المشروع. في Next.js مع App Router، كل ملف في مجلد app يمثل صفحة أو مكون. خلّيك دايماً ترجع للمجلد ده لما تدوّر على أي صفحة.

المجلد ده هو قلب المشروع. كل ما عايز تضيف صفحة جديدة، بتعمل مجلد جوا app وبتحط ملف page.js جواه. سهل بسيط!

app/layout.js javascript 
import "./globals.css";

export const metadata = {
  title: "Market - متجر إلكتروني",
  description: "متجر إلكتروني كامل",
};

export default function RootLayout({ children }) {
  return (
    <html lang="ar">
      <body>
        {children}
      </body>
    </html>
  );
}

الملف ده (layout.js) بيمثل التخطيط الرئيسي لكل صفحات المشروع. كل صفحة هتتحط جوا الـ children. الـ metadata دي بتحدد عنوان الصفحة والوصف اللي بيظهر في نتائج البحث.

app/page.js javascript 
export default function Home() {
  return (
    <main>
      <h1>مرحباً بكم في متجرنا</h1>
      <p>ده أول صفحة في المشروع</p>
    </main>
  );
}

الملف ده (page.js) بيمثل الصفحة الرئيسية للمتجر. لما تزور http://localhost:3000، هتلاقي المحتوى اللي جوا الـ return.

📁 مجلد public/

المجلد ده للملفات العامة زي الصور والأيقونات والـ favicons. أي ملف تحطه هنا هيتعرض مباشرة في المتصفح بدون معالجة. مثلاً لو حطيت صورة اسمها logo.png هنا، تقدر توصلها بالرابط /logo.png.

📄 ملف package.json

ده ملف بيحتوي على معلومات المشروع والـ dependencies (المكتبات المستخدمة). لما تثبّت حزمة جديدة، بتتضاف هنا تلقائياً.

package.json json 
{
  "name": "market",
  "version": "0.1.0",
  "private": true,
  "scripts": {
    "dev": "next dev --turbopack",
    "build": "next build",
    "start": "next start",
    "lint": "next lint"
  },
  "dependencies": {
    "next": "15.1.0",
    "react": "^19.0.0",
    "react-dom": "^19.0.0"
  },
  "devDependencies": {
    "@tailwindcss/postcss": "^4.0.0",
    "tailwindcss": "^4.0.0"
  }
}

📄 ملف globals.css

ده ملف الأنماط العامة للمشروع. فيه الأنماط اللي بتتطبق على كل الصفحات. في Next.js مع Tailwind CSS، الملف ده بيحتوي على أمر @import عشان يحمّل Tailwind.

3.2 شرح أوامر package.json

في الجدول جوا package.json، فيه أوامر مهمة ممكن تشغّلها:

  • npm run dev - بيشغّل خادم التطوير عشان تقدر تشوف المشروع في المتصفح
  • npm run build - بيبني المشروع للإنتاج (لما تبقى عايز تنشره)
  • npm run start - بيشغّل المشروع المبني
  • npm run lint - بيفحص الكود ويقولك لو في أخطاء

🚀 الخطوة 4: تشغيل خادم التطوير

الآن خلّينا نشغّل المشروع ونشوف النتيجة. الخطوة دي مهمة جداً عشان تتأكد إن كل حاجة اشتغلت صح.

4.1 الدخول لمجلد المشروع

Terminal bash 
cd market

الأمر ده بيقودك لمجلد المشروع. لازم تكون في المجلد الصح قبل ما تشغّل أي أمر تاني.

4.2 تشغيل خادم التطوير

Terminal bash 
npm run dev

بعد ما تضغط Enter، هتبدأ Next.js تحمّل المشروع. ممكن ياخد ثواني قليلة. لما تشوف الرسالة اللي جاية، يبقى خلاص!

✅ نجاح
لو شفت الرسالة "Ready in 1.2s" أو شيء مشابه، يبقى المشروع اتشغّل بنجاح. افتح المتصفح على http://localhost:3000

4.3 اختبار الموقع

افتح المتصفح واذهب إلى:

المتصفح text 
http://localhost:3000

هتلاقي الصفحة الرئيسية لمشروع Next.js مع شعار Next.js وكلمة "Get started by editing app/page.js". ده معناه إن المشروع اشتغل بنجاح!

💡 تجربة Hot Reload
جرّب تغيير النص في ملف app/page.js وشوف التغيير بيظهر مباشرة في المتصفح بدون ما تعمل إعادة تحميل. ده اسمه Hot Reload وبيسهّل عملية التطوير جداً. لما بتحفظ الملف، الصفحة بتتحدث تلقائياً!

4.4 إيقاف خادم التطوير

لما تبقى عايز تقفل خادم التطوير، اضغط Ctrl+C في الـ Terminal. ده هيوقف الخادم بأمان. متضغطش زر X عشان ممكن يسيب عملية شغالة في الخلفية.

🔧 الخطوة 5: تثبيت المكتبات الإضافية

محتاجين نثبّت بعض المكتبات اللي هنستخدمها في المشروع ده. كل مكتبة ليها وظيفة معينة وبنحتاجها عشان نبني المشروع الكامل.

Terminal bash 
# مكتبة الحركات والتأثيرات
npm install framer-motion

# مكتبة أيقونات
npm install lucide-react

# Clerk للمصادقة
npm install @clerk/nextjs

# Supabase لقاعدة البيانات
npm install @supabase/supabase-js
npm install @supabase/ssr

5.1 شرح كل مكتبة

المكتبة الوظيفة ليه محتاجينها؟
framer-motion حركات وتأثيرات لإضافة حركات سلسة للعناصر زي الظهور والاختفاء
lucide-react أيقونات مكتبة أيقونات جميلة وخفيفة للتصميم
@clerk/nextjs المصادقة نظام تسجيل الدخول والخروج للمستخدمين
@supabase/supabase-js قاعدة البيانات الاتصال بقاعدة البيانات Supabase
@supabase/ssr دعم Server Side للاتصال بـ Supabase من السيرفر

📝 ملخص أوامر npm المهمة

الأمر الوظيفة متى تستخدمه؟
npm run dev تشغيل خادم التطوير لما تبقى عايز تشتغل على المشروع
npm run build بناء المشروع للإنتاج لما تبقى عايز تنشر المشروع
npm run start تشغيل المشروع المبني بعد البناء مباشرة
npm run lint فحص الأخطاء لما تتأكد إن الكود نظيف
npm install [package] تثبيت حزمة جديدة لما تحتاج مكتبة جديدة
npm uninstall [package] حذف حزمة لما معدتش تحتاج الحزمة
npm update تحديث الحزم لما تحب تحدّث الحزم لإصدارات أحدث

🔍 استكشاف الأخطاء

في بعض المشاكل اللي ممكن تقابلها. خلّيني أقولك الحلول:

مشكلة: الأمر node أو npm لا يعمل

لو لم يتعرف على الأمر node أو npm:

  • أعد تشغيل الـ Terminal بعد تثبيت Node.js
  • تأكد من تثبيت Node.js بشكل صحيح
  • أعد تشغيل الكمبيوتر
  • لو لسه مش شغال، جرّب تثبّته تاني

مشكلة: خطأ في تثبيت الحزم

لو ظهر خطأ أثناء تثبيت الحزم:

Terminal bash 
# حذف مجلد node_modules وإعادة التثبيت
rm -rf node_modules
npm install

مشكلة: المنفذ مشغول

لو المنفذ 3000 مشغول (مثلاً في مشروع تاني شغال)، ممكن نغيره:

Terminal bash 
# تشغيل على منفذ آخر
npm run dev -- -p 3001

مشكلة: الصفحة بيضاء

لو الصفحة اتفتحت بس مفيش محتوى:

  • تأكد إنك في المجلد الصح (cd market)
  • تأكد إن خادم التطوير شغال
  • افتح Console في المتصفح (F12) وشوف لو في أخطاء
  • أعد تشغيل خادم التطوير
💡 نصيحة للبحث عن الأخطاء
اضغط F12 في المتصفح عشان تفتح Developer Tools، وروح على تبويب Console. أي خطأ في الكود هيبان هنا. كمان روح على تبويب Network وشوف لو في طلبات فاشلة (حمراء).
