نظام Admin وحماية المسارات

⏱️ 20 دقيقة 📖 عملي الوحدة 2: قاعدة البيانات والمصادقة

🎯 أهداف الدرس

أهلاً بيك في الدرس السابع والأخير في الوحدة التانية! في الدرس ده هنتعلم إزاي نحمي صفحات Admin ونتحكم في مين يقدر يدخل على صفحات معينة. ده مهم جداً لأمان أي مشروع إلكتروني.

  • إعداد Middleware لحماية المسارات
  • حماية صفحات Admin
  • التحقق من صلاحيات المستخدم
  • التعامل مع المسارات العامة والمحمية
  • إعادة التوجيه (Redirect) بشكل صحيح
💡 ليه الحماية مهمة؟
بدون حماية صحيحة، أي حد يقدر يفتح صفحات Admin ويعدّل في البيانات أو يمسحها. ده ممكن يسبب مشاكل كبيرة. الحماية بتطمن إن بس الأشخاص المصرح لهم يقدروا يوصلوا للصفحات الحساسة.

🛡️什么是 Middleware؟

Middleware في Next.js هو كود بيشتغل قبل ما الطلب يوصل للصفحة. يعني لو حد زار /admin، الـ Middleware بيشتغل الأول وبيقرر يسيبه يدخل ولا يعيد توجيهه.

تخيّل إن الـ Middleware زي الحارس على باب المبنى. قبل ما حد يدخل، الحارس بيسأله: "مين أنت؟ عندك تصريح؟" لو الإجابة صح، بيسيبه يدخل. لو لأ، بيرجّعه.

استخدامات Middleware:

  • حماية المسارات المحمية (زي صفحات Admin)
  • إعادة التوجيه بناءً على الحالة (لو مش مسجل دخول)
  • إضافة headers إضافية للطلب
  • التحقق من الصلاحيات (Admin ولا مستخدم عادي)
  • إعادة كتابة الروابط (Rewriting)
  • فحص اللغة أو المنطقة الجغرافية

🔧 الخطوة 1: إعداد Middleware

أنشئ ملف middleware.js في جذر المشروع (مش جوا مجلد app):

📁 market/
├── 📄 middleware.js # Middleware الرئيسي
├── 📁 app/
├── 📁 components/
└── 📁 lib/
middleware.js javascript 
import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server';

// تعريف المسارات العامة (ممكن أي حد يوصلها)
const publicPaths = createRouteMatcher([
  '/',
  '/products(.*)',
  '/about',
  '/contact',
  '/sign-in(.*)',
  '/sign-up(.*)',
  '/api(.*)',
]);

// تعريف مسارات Admin
const adminPaths = createRouteMatcher([
  '/admin(.*)',
  '/admin/dashboard(.*)',
  '/admin/products(.*)',
  '/admin/orders(.*)',
]);

export default clerkMiddleware(async (auth, req) => {
  // إذا كان المسار عام، اسمح بالوصول
  if (publicPaths(req)) {
    return;
  }
  
  // إذا كان المسار Admin، تحقق من الصلاحيات
  if (adminPaths(req)) {
    const { userId } = await auth();
    
    // إذا لم يكن المستخدم مسجلاً، أعد التوجيه لصفحة تسجيل الدخول
    if (!userId) {
      return Response.redirect(new URL('/sign-in', req.url));
    }
    
    // تحقق من أن المستخدم Admin
    // يمكنك تخزين معلومات المستخدم في database أو JWT
    // هنا مثال بسيط للتحقق من البريد الإلكتروني
    const user = await auth.user?.();
    const isAdmin = user?.emailAddresses?.[0]?.emailAddress === 'admin@example.com';
    
    if (!isAdmin) {
      // إعادة التوجيه للصفحة الرئيسية إذا لم يكن Admin
      return Response.redirect(new URL('/', req.url));
    }
  }
  
  // لبقية المسارات، تحقق من المصادقة
  const { userId } = await auth();
  
  if (!userId) {
    // إعادة التوجيه لصفحة تسجيل الدخول
    return Response.redirect(new URL('/sign-in', req.url));
  }
});

export const config = {
  matcher: [
    // تخطي ملفات النثاث والصور
    '/((?!_next|[^?]*\\.(?:html?|css|js(?!on)|jpe?g|webp|png|gif|svg|ttf|woff2?|ico|csv|docx?|xlsx?|zip|webmanifest)).*)',
    '/(api|trpc)(.*)',
  ],
};
💡 شرح الكود
  • createRouteMatcher - دالة بتساعدك تحدد المسارات بشكل سهل
  • clerkMiddleware - Middleware خاص بـ Clerk للمصادقة
  • auth() - بيرجع معلومات المستخدم الحالي
  • config.matcher - بيحدد على أي طلبات يشتغل الـ Middleware (بيتخطى ملفات الصور والـ CSS)

📐 الخطوة 3: Layout خاص بـ Admin

app/(admin)/layout.js javascript 
"use client";

import { useAuth, useUser } from '@clerk/nextjs';
import { useRouter } from 'next/navigation';
import { useEffect } from 'react';
import Link from 'next/link';

export default function AdminLayout({ children }) {
  const { userId } = useAuth();
  const { user } = useUser();
  const router = useRouter();
  
  // التحقق من أن المستخدم Admin
  useEffect(() => {
    if (userId) {
      // هنا ممكن تتحقق من Database إذا كان المستخدم Admin
      // للتبسيط، هنستخدم البريد الإلكتروني
      const isAdmin = user?.emailAddresses?.[0]?.emailAddress === 'admin@example.com';
      
      if (!isAdmin) {
        router.push('/');
      }
    }
  }, [userId, user, router]);
  
  // إذا لم يكن المستخدم مسجلاً
  if (!userId) {
    return (
      <div className="flex items-center justify-center min-h-screen">
        <p>جاري التحقق من الصلاحيات...</p>
      </div>
    );
  }
  
  return (
    <div className="flex min-h-screen">
      {/* الشريط الجانبي */}
      <aside className="w-64 bg-gray-900 text-white p-4">
        <div className="mb-8">
          <h2 className="text-xl font-bold">لوحة التحكم</h2>
          <p className="text-sm text-gray-400">مرحباً {user?.firstName}</p>
        </div>
        
        <nav className="space-y-2">
          <Link 
            href="/admin"
            className="block p-2 rounded hover:bg-gray-800 transition-colors"
          >
            Dashboard
          </Link>
          <Link 
            href="/admin/products"
            className="block p-2 rounded hover:bg-gray-800 transition-colors"
          >
            المنتجات
          </Link>
          <Link 
            href="/admin/orders"
            className="block p-2 rounded hover:bg-gray-800 transition-colors"
          >
            الطلبات
          </Link>
        </nav>
      </aside>
      
      {/* المحتوى الرئيسي */}
      <main className="flex-1 p-8">
        {children}
      </main>
    </div>
  );
}

الـ Layout ده بيعمل حاجتين مهمتين: أول حاجة بيتحقق إن المستخدم Admin، وثاني حاجة بيعرض شريط جانبي فيه روابط لوحة التحكم.

🔐 الخطوة 5: حماية API Routes

ممكن نحمي API Routes باستخدام Middleware أو Clerk مباشرة. ده مهم عشان حتى لو حد عرف الـ URL بتاع الـ API، ما يقدرش يستخدمه من غير تصريح:

app/api/admin/products/route.js javascript 
import { auth } from '@clerk/nextjs/server';
import { NextResponse } from 'next/server';
import { supabase } from '@/lib/supabase';

// GET: جلب كل المنتجات (Admin فقط)
export async function GET() {
  const { userId } = await auth();
  
  if (!userId) {
    return NextResponse.json(
      { error: 'غير مصرح' },
      { status: 401 }
    );
  }
  
  // هنا ممكن تتحقق من أن المستخدم Admin
  // ...
  
  const { data, error } = await supabase
    .from('products')
    .select('*')
    .order('created_at', { ascending: false });
  
  if (error) {
    return NextResponse.json(
      { error: error.message },
      { status: 500 }
    );
  }
  
  return NextResponse.json(data);
}

// POST: إضافة منتج جديد (Admin فقط)
export async function POST(request) {
  const { userId } = await auth();
  
  if (!userId) {
    return NextResponse.json(
      { error: 'غير مصرح' },
      { status: 401 }
    );
  }
  
  const body = await request.json();
  
  const { data, error } = await supabase
    .from('products')
    .insert([body])
    .select();
  
  if (error) {
    return NextResponse.json(
      { error: error.message },
      { status: 500 }
    );
  }
  
  return NextResponse.json(data[0], { status: 201 });
}

🔄 الخطوة 6: إعادة التوجيه الذكية

ممكن نحسن تجربة المستخدم بإعادة توجيه ذكية. مثلاً لو المستخدم كان عايز يدخل صفحة معينة ولّحه لتسجيل الدخول، بعد التسجيل يرجع تاني للصفحة الأصلية:

middleware.js - إعادة توجيه محسنة javascript 
import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server';

const publicPaths = createRouteMatcher([
  '/',
  '/products(.*)',
  '/about',
  '/contact',
  '/sign-in(.*)',
  '/sign-up(.*)',
]);

const adminPaths = createRouteMatcher([
  '/admin(.*)',
]);

export default clerkMiddleware(async (auth, req) => {
  const { userId } = await auth();
  
  // إذا كان المسار عام، اسمح بالوصول
  if (publicPaths(req)) {
    // إذا كان المستخدم مسجلاً وحاول الدخول لصفحة تسجيل الدخول
    if (userId && (req.nextUrl.pathname === '/sign-in' || req.nextUrl.pathname === '/sign-up')) {
      return Response.redirect(new URL('/', req.url));
    }
    return;
  }
  
  // إذا كان المسار Admin
  if (adminPaths(req)) {
    if (!userId) {
      // حفظ المسار الأصلي للعودة إليه بعد تسجيل الدخول
      const signInUrl = new URL('/sign-in', req.url);
      signInUrl.searchParams.set('redirect_url', req.url);
      return Response.redirect(signInUrl);
    }
    
    // تحقق من الصلاحيات هنا
    // ...
  }
  
  // لبقية المسارات
  if (!userId) {
    return Response.redirect(new URL('/sign-in', req.url));
  }
});

📋 ملخص أنواع الحماية

الطريقة الاستخدام مثال
Middleware حماية على مستوى السيرفر حماية كل المسارات بناءً على الشرط
SignedIn / SignedOut إظهار/إخفاء المحتوى إظهار زر تسجيل الدخول للضيوف
useAuth() التحقق في Client Components إظهار معلومات المستخدم
auth() التحقق في Server Components جلب بيانات المستخدم من قاعدة البيانات
API Route Protection حماية Endpoints التحقق قبل تنفيذ عملية

🔍 استكشاف الأخطاء

مشكلة: الـ Middleware مش شغال

لو الحماية مش بتشتغل:

  • تأكد إن ملف middleware.js في جذر المشروع (مش جوا مجلد app)
  • تأكد إنك ما حطيتش middleware.js في مجلد خطأ
  • أعد تشغيل خادم التطوير بعد إنشاء الملف

مشكلة: إعادة توجيه لا نهائية

لو الصفحة بتعمل reload باستمرار:

  • تأكد إن صفحات Auth (/sign-in, /sign-up) موجودة في publicPaths
  • تأكد إن في حلقة لا نهائية في شروط الإعادة

مشكلة: المستخدم العادي بيدخل صفحات Admin

  • تأكد إن الشرط بيتحقق من الصلاحيات صح
  • جرّب تطبع userId في Console وتشوف قيمته
  • تأكد إن التحقق من Admin بيتم صح
