Next.js هو القلب النابض للمشروع. لو فهمت إزاي App Router بيشتغل، هتقدر تبني أي صفحة بأي تصميم. الدرس ده هيديك الأساس القوي اللي محتاجه.

📂 إيه هو App Router؟

App Router هو نظام التوجيه الجديد في Next.js 13+ اللي بيعتمد على مجلد app. بدل ما نستخدم مجلد pages القديم، بنستخدم app اللي بيقدم مميزات أقوى وأحدث.

يعني إيه نظام توجيه؟ يعني إزاي المشروع بيعرف إنك عايز تفتح صفحة معينة لما تكتب رابط معين في المتصفح. مثلاً لما تكتب /products، Next.js بيدور على ملف page.js جوا مجلد products عشان يعرضلك الصفحة.

المميزات الرئيسية لـ App Router:

  • Layouts - تخطيطات مشتركة بين الصفحات (زي الـ Header والـ Footer)
  • Server Components - مكونات بتشتغل على السيرفر (أسرع وأأمن)
  • Streaming - تحميل تدريجي للمحتوى (بيحسّن السرعة)
  • Async Components - مكونات asynchronous (بتقدر تجلب بيانات مباشرة)
  • Error Handling - معالجة أخطاء متقدمة
  • Loading States - حالات تحميل تلقائية

ليه App Router أحسن من Pages Router؟

الميزة App Router Pages Router (القديم)
Layouts مدعوم أصلاً محتاج أكواد إضافية
Server Components الافتراضي مش مدعوم
التحميل تدريجي (Streaming) دفعة واحدة
Error Handling متقدم مع error.js محدود

📄 إنشاء صفحة جديدة

في App Router، كل ملف page.js في مجلد يمثل صفحة. يعني لو عايز تضيف صفحة جديدة، بتعمل مجلد وبتحط جواه ملف page.js. سهل بسيط!

مثال: إنشاء صفحة "المنتجات"

📁 app/
├── 📁 products/ # مجلد المنتجات
│ └── 📄 page.js # صفحة المنتجات
├── 📄 layout.js
└── 📄 page.js

الخطوات:

  1. أنشئ مجلد جديد اسمه products داخل مجلد app
  2. أنشئ ملف page.js جوا مجلد products
  3. اكتب الكود التالي في الملف
app/products/page.js javascript 
export default function ProductsPage() {
  return (
    <div>
      <h1>المنتجات</h1>
      <p>هنا هنعرض كل المنتجات المتاحة</p>
    </div>
  );
}

الآن ممكن نزور الصفحة على: http://localhost:3000/products

💡 قاعدة مهمة
في App Router، لازم اسم المجلد يبقى نفس الـ URL اللي عايزه. يعني لو عايز صفحة /about، لازم تعمل مجلد اسمه about وبتحط جواه page.js. لو المجلد مختلف، الرابط مش هيشتغل!

📐 المخططات (Layouts)

الـ Layout بيمثل التخطيط المشترك بين صفحات كتير. مثلاً، الـ Header والـ Footer بيكونوا في الـ Layout عشان يظهروا في كل الصفحات بدون ما نكتبهم كل مرة.

تخيّل إنك كل ما تفتح صفحة جديدة في أي موقع، لازم تكتب الـ Header والـ Footer من الأول. ده هيكون متعب جداً! الـ Layout بحل المشكلة دي.

app/layout.js javascript 
import "./globals.css";

export const metadata = {
  title: "Market - متجر إلكتروني",
  description: "متجر إلكتروني كامل",
};

export default function RootLayout({ children }) {
  return (
    <html lang="ar">
      <body>
        <header>
          <nav>
            <a href="/">الرئيسية</a>
            <a href="/products">المنتجات</a>
            <a href="/about">من نحن</a>
          </nav>
        </header>
        
        <main>{children}</main>
        
        <footer>
          <p>جميع الحقوق محفوظة</p>
        </footer>
      </body>
    </html>
  );
}

لاحظ إن {children} هنا بيمثل أي صفحة فتحتها. يعني لما تفتح /products، محتوى صفحة المنتجات بيتحط مكان الـ children في الـ Layout.

💡 نصيحة
الـ Layout بيتكرر تلقائياً لكل الصفحات اللي تحتيه. لو عندك Layout خاص بمجلد معين، حط ملف layout.js في المجلد ده. الـ Layout الأعلى هيحتوي على الـ Layout الأدنى.

Layout مخصص لمجلد معين

ممكن تعمل Layout خاص بمجلد معين. مثلاً لو عندك قسم Admin كل صفحاته ليها تصميم مختلف:

📁 app/
├── 📁 admin/
│ ├── 📄 layout.js # Layout خاص بـ Admin
│ ├── 📄 page.js
│ └── 📁 products/
│ └── 📄 page.js
├── 📄 layout.js # Layout رئيسي
└── 📄 page.js

🔗 المسارات الديناميكية (Dynamic Routes)

المسارات الديناميكية بتسمحلك تقبل параметرات في الـ URL. في Next.js بنستخدم الأقواس المربعة [param]. يعني لو عايز صفحة تفاصيل لكل منتج، بدل ما تعمل صفحة لكل منتج، بتعمل صفحة واحدة بتاخد الرقم كبرامتر.

مثال: صفحة تفاصيل المنتج

📁 app/
├── 📁 products/
│ ├── 📁 [id]/ # مسار ديناميكي
│ │ └── 📄 page.js # صفحة تفاصيل المنتج
│ └── 📄 page.js # صفحة كل المنتجات
├── 📄 layout.js
└── 📄 page.js

يعني لما تزور /products/123، الـ id هتبقى 123. ولما تزور /products/456، الـ id هتبقى 456. نفس الصفحة بس البيانات بتتغير!

app/products/[id]/page.js javascript 
export default function ProductDetails({ params }) {
  const { id } = params;
  
  return (
    <div>
      <h1>تفاصيل المنتج رقم {id}</h1>
      <p>هنا هنعرض تفاصيل المنتج {id}</p>
    </div>
  );
}

الوصول للبرامترز في Next.js 15 و 16

في Next.js 15 و 16، الـ params بقت Promise. يعني لازم نستخدم await عشان نحصل عليها. ده تغيير مهم!

app/products/[id]/page.js javascript 
export default async function ProductDetails({ params }) {
  const { id } = await params;
  
  return (
    <div>
      <h1>تفاصيل المنتج رقم {id}</h1>
    </div>
  );
}
⚠️ تنبيه مهم
في Next.js 15+، لازم نستخدم await مع الـ params عشان بقت Promise. لو مش عملت كده، هتلاقي أخطاء غريبة. كمان لازم الدالة تبقى async.

🖥️ Client vs Server Components

في Next.js، فيه نوعين من المكونات. فهم الفرق بينهم مهم جداً عشان تعرف تستخدم كل نوع في المكان الصح.

Server Components Client Components
بتشتغل على السيرفر بتشتغل على المتصفح (Client)
مش بتحتاج "use client" محتاجة "use client" في الأول
أسرع في التحميل ممكن تفاعل مع المستخدم
ممكن تستخدم async/await ممكن تستخدم useState, useEffect
الـ Default في Next.js لازم نحددّها صراحة
ممكن تجلب بيانات من قاعدة البيانات مباشرة ممكن تستخدم Browser APIs

مثال: Server Component

app/products/page.js javascript 
// Server Component - الافتراضي
// ممكن نجلب بيانات من قاعدة البيانات مباشرة

async function getProducts() {
  const res = await fetch('https://api.example.com/products');
  return res.json();
}

export default async function ProductsPage() {
  const products = await getProducts();
  
  return (
    <div>
      <h1>المنتجات</h1>
      <ul>
        {products.map(product => (
          <li key={product.id}>{product.name}</li>
        ))}
      </ul>
    </div>
  );
}

مثال: Client Component

app/components/Counter.js javascript 
"use client"; // لازم نكتبها في الأول

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>العدد: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        زيادة
      </button>
      <button onClick={() => setCount(count - 1)}>
        نقصان
      </button>
    </div>
  );
}
💡 متى نستخدم Client Component؟
استخدم Client Component لو محتاج:
  • useState أو useReducer (حالة البيانات)
  • useEffect (التأثيرات الجانبية)
  • أحداث المستخدم (onClick, onChange, onSubmit)
  • Browser APIs (localStorage, window, navigator)
  • Hook مخصص بيعتمد على أي حاجة من دول
⚠️ خطأ شائع
متكتبش "use client" في كل المكونات! Server Components هي الافتراضي وأسرع. اكتبها بس لما تكون فعلاً محتاجها. كتير من المبتدئين بيحطوها في كل مكان وده بيبطئ المشروع.

📁 هيكل المجلدات في المشروع

في المشروع ده، هنستخدم Route Groups عشان ننظم الصفحات. الـ Route Groups بتسمحلك تجمّع صفحات معينة من غير ما تأثر على الـ URL.

📁 app/
├── 📁 (shop)/ # صفحات المتجر
│ ├── 📁 products/
│ │ ├── 📁 [id]/
│ │ │ └── 📄 page.js
│ │ └── 📄 page.js
│ ├── 📄 layout.js
│ └── 📄 page.js
├── 📁 (admin)/ # صفحات Admin
│ ├── 📁 dashboard/
│ │ └── 📄 page.js
│ ├── 📄 layout.js
│ └── 📄 page.js
├── 📁 (auth)/ # صفحات تسجيل الدخول
│ ├── 📄 layout.js
│ └── 📄 page.js
├── 📄 layout.js # Layout رئيسي
└── 📄 page.js # الصفحة الرئيسية
💡 Route Groups
الأقواس المدورة (shop) بتوصل إن المجلد ده Route Group ومش بتتأثر على الـ URL. يعني (shop)/products هتكون نفس /products. الميزة إنك تقدر تعمل Layout خاص لكل مجموعة صفحات.

🔍 إنشاء مسار ديناميكي مع query parameters

app/products/[id]/page.js javascript 
// استخدام searchParams للحصول على query parameters
export default async function ProductDetails({ params, searchParams }) {
  const { id } = await params;
  const { color, size } = await searchParams;
  
  return (
    <div>
      <h1>المنتج {id}</h1>
      <p>اللون: {color || 'غير محدد'}</p>
      <p>المقاس: {size || 'غير محدد'}</p>
    </div>
  );
}

يعني لما تزور /products/123?color=red&size=large، هتلاقي اللون أحمر والمقاس large. الـ searchParams دي بتساعدك تفلتر أو تغيّر المحتوى بناءً على الرابط.

⚡ مثال عملي: صفحة \"من نحن\"

خلّيني نعمل تمرين بسيط. عايزين نعمل صفحة "من نحن" للمتجر.

  1. أنشئ مجلد about داخل app
  2. أنشئ ملف page.js جوا المجلد
  3. اكتب الكود التالي
app/about/page.js javascript 
export default function AboutPage() {
  return (
    <div className="max-w-4xl mx-auto p-8">
      <h1 className="text-4xl font-bold mb-6">من نحن</h1>
      <p className="text-lg text-gray-600 mb-4">
        نحن متجر إلكتروني متخصص في بيع أفضل المنتجات بأفضل الأسعار.
      </p>
      <p className="text-lg text-gray-600">
        فريقنا يعمل باستمرار لتقديم أفضل تجربة تسوق لعملائنا.
      </p>
    </div>
  );
}

الآن افتح المتصفح على http://localhost:3000/about وهتلاقي الصفحة شغالة!
