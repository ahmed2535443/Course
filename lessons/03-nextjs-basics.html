📂什么是 App Router؟

App Router هو نظام التوجيه الجديد في Next.js 13+ اللي بيعتمد على مجلد app. بدل ما نستخدم مجلد pages القديم، بنستخدم app اللي بيقدم مميزات أقوى.

المميزات الرئيسية لـ App Router:

  • Layouts - تخطيطات مشتركة بين الصفحات
  • Server Components - مكونات بتشتغل على السيرفر
  • Streaming - تحميل تدريجي للمحتوى
  • Async Components - مكونات asynchronous
  • Error Handling - معالجة أخطاء متقدمة

📄 إنشاء صفحة جديدة

في App Router، كل ملف page.js في مجلد يمثل صفحة.

مثال: إنشاء صفحة "المنتجات"

📁 app/
├── 📁 products/ # مجلد المنتجات
│ └── 📄 page.js # صفحة المنتجات
├── 📄 layout.js
└── 📄 page.js
app/products/page.js javascript 
export default function ProductsPage() {
  return (
    <div>
      <h1>المنتجات</h1>
      <p>هنا هنعرض كل المنتجات المتاحة</p>
    </div>
  );
}

الآن ممكن نزور الصفحة على: http://localhost:3000/products

📐 المخططات (Layouts)

الـ Layout بيمثل التخطيط المشترك بين صفحات كتير. مثلاً، الـ Header والـ Footer بيكونوا في الـ Layout عشان يظهروا في كل الصفحات.

app/layout.js javascript 
import "./globals.css";

export const metadata = {
  title: "Market - متجر إلكتروني",
  description: "متجر إلكتروني كامل",
};

export default function RootLayout({ children }) {
  return (
    <html lang="ar">
      <body>
        <header>
          <nav>
            <a href="/">الرئيسية</a>
            <a href="/products">المنتجات</a>
            <a href="/about">من نحن</a>
          </nav>
        </header>
        
        <main>{children}</main>
        
        <footer>
          <p>جميع الحقوق محفوظة</p>
        </footer>
      </body>
    </html>
  );
}
💡 نصيحة
الـ Layout بيتكرر تلقائياً لكل الصفحات اللي تحتيه. لو عندك Layout خاص بمجلد معين، حط ملف layout.js في المجلد ده.

🔗 المسارات الديناميكية (Dynamic Routes)

المسارات الديناميكية بتسمحلك تقبل параметriers في الـ URL. في Next.js بنستخدم الأقواس المربعة [param].

مثال: صفحة تفاصيل المنتج

📁 app/
├── 📁 products/
│ ├── 📁 [id]/ # مسار ديناميكي
│ │ └── 📄 page.js # صفحة تفاصيل المنتج
│ └── 📄 page.js # صفحة كل المنتجات
├── 📄 layout.js
└── 📄 page.js
app/products/[id]/page.js javascript 
export default function ProductDetails({ params }) {
  const { id } = params;
  
  return (
    <div>
      <h1>تفاصيل المنتج رقم {id}</h1>
      <p>هنا هنعرض تفاصيل المنتج {id}</p>
    </div>
  );
}

الوصول للبرامترز

في Next.js 15 و 16، الـ params بقت Promise:

app/products/[id]/page.js javascript 
export default async function ProductDetails({ params }) {
  const { id } = await params;
  
  return (
    <div>
      <h1>تفاصيل المنتج رقم {id}</h1>
    </div>
  );
}
⚠️ تنبيه
في Next.js 15+، لازم نستخدم await مع الـ params عشان بقت Promise. لو مش عملت كده، هتلاقي أخطاء.

🖥️ Client vs Server Components

في Next.js، فيه نوعين من المكونات:

Server Components Client Components
بتشتغل على السيرفر بتشتغل على المتصفح (Client)
مش بتحتاج "use client" محتاجة "use client" في الأول
أسرع في التحميل ممكن تفاعل مع المستخدم
ممكن تستخدم async/await ممكن تستخدم useState, useEffect
الـ Default في Next.js لازم نحددّها صراحة

مثال: Server Component

app/products/page.js javascript 
// Server Component - الافتراضي
// ممكن نجلب بيانات من قاعدة البيانات مباشرة

async function getProducts() {
  const res = await fetch('https://api.example.com/products');
  return res.json();
}

export default async function ProductsPage() {
  const products = await getProducts();
  
  return (
    <div>
      <h1>المنتجات</h1>
      <ul>
        {products.map(product => (
          <li key={product.id}>{product.name}</li>
        ))}
      </ul>
    </div>
  );
}

مثال: Client Component

app/components/Counter.js javascript 
"use client"; // لازم نكتبها في الأول

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>العدد: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        زيادة
      </button>
      <button onClick={() => setCount(count - 1)}>
        نقصان
      </button>
    </div>
  );
}
💡 متى نستخدم Client Component؟
استخدم Client Component لو محتاج:
  • useState أو useReducer
  • useEffect
  • أحداث المستخدم (onClick, onChange)
  • Browser APIs (localStorage, window)

📁 هيكل المجلدات في المشروع

في المشروع ده، هنستخدم Route Groups عشان ننظم الصفحات:

📁 app/
├── 📁 (shop)/ # صفحات المتجر
│ ├── 📁 products/
│ │ ├── 📁 [id]/
│ │ │ └── 📄 page.js
│ │ └── 📄 page.js
│ ├── 📄 layout.js
│ └── 📄 page.js
├── 📁 (admin)/ # صفحات Admin
│ ├── 📁 dashboard/
│ │ └── 📄 page.js
│ ├── 📄 layout.js
│ └── 📄 page.js
├── 📁 (auth)/ # صفحات تسجيل الدخول
│ ├── 📄 layout.js
│ └── 📄 page.js
├── 📄 layout.js # Layout رئيسي
└── 📄 page.js # الصفحة الرئيسية
💡 Route Groups
الأقواس المدورة (shop) بتوصل إن المجلد ده Route Group ومش بتتأثر على الـ URL. يعني (shop)/products هتكون نفس /products.

🔍 إنشاء مسار ديناميكي مع query

app/products/[id]/page.js javascript 
// استخدام searchParams للحصول على query parameters
export default async function ProductDetails({ params, searchParams }) {
  const { id } = await params;
  const { color, size } = await searchParams;
  
  return (
    <div>
      <h1>المنتج {id}</h1>
      <p>اللون: {color || 'غير محدد'}</p>
      <p>المقاس: {size || 'غير محدد'}</p>
    </div>
  );
}
