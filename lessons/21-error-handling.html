💡 مقدمة عن معالجة الأخطاء

أهلاً بيكم في الدرس الحادي والعشرين يا صديقي! الدرس ده من أهم الدروس في الكورس كله. ليه؟ لأن الأخطاء بتحصل دايماً. مش سؤال "لو حصل خطأ"، السؤال هو "امتى هتحصل الأخطاء".

تخيل معايا الموقف ده: عميل فتح صفحة منتج والصفحة اتعملها crash. أو العميل ضغط على زرار "شراء" والسيرفر رجع خطأ 500. أو العميل كتب بريد إلكتروني غلط والنماذج بعتتله رسالة خطأ مش مفهومة. كل المواقف دي لو مش معالجها كويس، هتخسر عملاء.

💡 ليه معالجة الأخطاء مهمة جداً؟
  • بتحافظ على سمعة موقعك وويبعث الاحترافي
  • بتمنع خسارة العملاء بسبب مشاكل تقنية
  • بتساعدك تكتشف المشاكل قبل ما العملاء يشوفوها
  • بتدي المستخدم ثقة في موقعك
  • بتقلل ضغط فريق الدعم الفني

في الدرس ده هنتعلم ازاي نتعامل مع كل أنواع الأخطاء: أخطاء 404 (صفحة غير موجودة)، أخطاء 500 (خطأ في السيرفر)، أخطاء الشبكة، أخطاء التحقق من البيانات، وأكتر من كده. وهنتعلم ازاي نعرض رسائل خطأ واضحة ومضحكة للمستخدم بدل رسائل تقنية مش مفهومة.

📁 هيكل ملفات معالجة الأخطاء

📁 app/
├── 📄 not-found.js # صفحة 404
├── 📄 error.js # صفحة الخطأ العامة
├── 📁 (shop)/
│ └── 📄 error.js # خطأ صفحات المتجر
📁 components/ui/
├── 📄 Toast.jsx # نظام الإشعارات
├── 📄 ErrorBoundary.jsx # حافة الخطأ
└── 📄 ErrorMessage.jsx # رسالة الخطأ
📁 hooks/
└── 📄 useErrorHandler.js # Hook لمعالجة الأخطاء

شوف يا صديقي، الهيكل ده بيبينلك كل الملفات اللي محتاجها لمعالجة الأخطاء. كل ملف ليه دور مختلف:

  • not-found.js: الصفحة اللي بتظهر لما المستخدم يفتح رابط غلط (404).
  • error.js: الصفحة اللي بتظهر لما يحصل خطأ غير متوقع (500).
  • Toast.jsx: نظام الإشعارات الصغيرة اللي بتظهر في الزاوية.
  • ErrorBoundary.jsx: مكون بيحمي أجزاء معينة من الصفحة من الأخطاء.
  • ErrorMessage.jsx: مكون موحد لعرض رسائل الخطأ.
  • useErrorHandler.js: Hook بيجمع كل منطق معالجة الأخطاء في مكان واحد.

شوف يا صديقي، الصفحة دي من أحلى صفحات الخطأ اللي ممكن تعملها. بدل ما تظهر صفحة بيضاء أو رسالة تقنية مش مفهومة، عملنا صفحة جميلة وممتعة:

  • شبح (Ghost): أيقونة شبح بتتحرك من Framer Motion. بتبدأ دوران 180 درجة وبعدين بتثبت.
  • علامة استفهام (?): بتتحرك لفوق وللتحت (y: [0, -10, 0]). وده بيخليها تحسك إنها حية.
  • رقم 404 كبير: حجم 8xl. ده أول حاجة المستخدم بيشوفها.
  • رسالة واضحة: "الصفحة غير موجودة". مش رسالة تقنية مش مفهومة.
  • زرين: "العودة للرئيسية" و "تصفح المنتجات". المستخدم يعرف يروح فين.
  • رسالة مضحكة: "يبدو أن الصفحة ذهبت في إجازة". ده بيخلي الخطأ يكون أقل إزعاجاً.
💡 ليه بنتكلم عن 404؟
الخطأ 404 بيحصل لما المستخدم يفتح رابط غلط أو يضغط على رابط قديم مش موجود. في المتاجر الإلكترونية، ده ممكن يحصل لو العميل عنده bookmark لمنتج اتشال أو صفحة اتغيّر رابطها. الصفحة الجميلة دي بتخلي المستخدم يفضل في الموقع بدل ما يمشي.

        <motion.div
          initial={{ opacity: 0, y: 20 }}

            <RefreshCw className="w-5 h-5 ml-2" />

دي صفحة الخطأ العامة. شوف ازاي عملناها ذكية:

  • error و reset: المكون بياخد props من Next.js. error فيه تفاصيل الخطأ، وreset دالة بتعمّل reset للصفحة.
  • useEffect: بيستخدم لتسجيل الخطأ. في بيئة التطوير (development)، الخطأ بيظهر للمطور. في الإنتاج (production)، الخطأ بيتبعت لنظام مراقبة زي Sentry.
  • تفاصيل الخطأ للمطورين فقط: لاحظ الشرط process.env.NODE_ENV === 'development'. ده بيخلي تفاصيل الخطأ تظهر بس للمطورين، مش للمستخدمين العاديين. لأن المستخدم العادي مش محتاج يعرف إن الخطأ "Cannot read property 'name' of undefined".
  • زرار إعادة المحاولة: المستخدم ممكن يضغط على الزرار ده عشان يجرب تاني. وده أهم ميزة في الصفحة دي.
⚠️ غلطة شائعة
بعض المطورين بيحطوا تفاصيل الخطأ الكاملة في صفحة الخطأ. ده غلط! المستخدم العادي مش محتاج يعرف إن الخطأ "TypeError: Cannot read property 'map' of undefined". كفاية إنك تقوله "حدث خطأ غير متوقع" وتعطيه زرار "إعادة المحاولة".

🔔 الخطوة 3: نظام Toast Messages

components/ui/Toast.jsx jsx 
"use client";

import { createContext, useContext, useState, useCallback } from 'react';
import { motion, AnimatePresence } from 'framer-motion';
import { X, CheckCircle, AlertCircle, Info, AlertTriangle } from 'lucide-react';

const ToastContext = createContext(null);

const toastTypes = {
  success: {
    icon: CheckCircle,
    bgColor: 'bg-green-500/20',
    borderColor: 'border-green-500/30',
    textColor: 'text-green-400',
  },
  error: {
    icon: AlertCircle,
    bgColor: 'bg-red-500/20',
    borderColor: 'border-red-500/30',
    textColor: 'text-red-400',
  },
  warning: {
    icon: AlertTriangle,
    bgColor: 'bg-yellow-500/20',
    borderColor: 'border-yellow-500/30',
    textColor: 'text-yellow-400',
  },
  info: {
    icon: Info,
    bgColor: 'bg-blue-500/20',
    borderColor: 'border-blue-500/30',
    textColor: 'text-blue-400',
  },
};

export function ToastProvider({ children }) {
  const [toasts, setToasts] = useState([]);

  const addToast = useCallback((message, type = 'info', duration = 5000) => {
    const id = Date.now() + Math.random();
    setToasts((prev) => [...prev, { id, message, type }]);

    if (duration > 0) {
      setTimeout(() => {
        removeToast(id);
      }, duration);
    }

    return id;
  }, []);

  const removeToast = useCallback((id) => {
    setToasts((prev) => prev.filter((toast) => toast.id !== id));
  }, []);

  const success = useCallback((message, duration) => {
    return addToast(message, 'success', duration);
  }, [addToast]);

  const error = useCallback((message, duration) => {
    return addToast(message, 'error', duration);
  }, [addToast]);

  const warning = useCallback((message, duration) => {
    return addToast(message, 'warning', duration);
  }, [addToast]);

  const info = useCallback((message, duration) => {
    return addToast(message, 'info', duration);
  }, [addToast]);

  return (
    <ToastContext.Provider value={{ addToast, removeToast, success, error, warning, info }}>
      {children}
      
      {/* Toast Container */}
      <div className="fixed bottom-4 right-4 z-50 flex flex-col gap-2 max-w-sm">
        <AnimatePresence>
          {toasts.map((toast) => {
            const config = toastTypes[toast.type] || toastTypes.info;
            const Icon = config.icon;

            return (
              <motion.div
                key={toast.id}
                initial={{ opacity: 0, x: 100, scale: 0.8 }}
                animate={{ opacity: 1, x: 0, scale: 1 }}
                exit={{ opacity: 0, x: 100, scale: 0.8 }}
                className={`flex items-center gap-3 p-4 rounded-xl border backdrop-blur-lg ${config.bgColor} ${config.borderColor}`}
              >
                <Icon className={`w-5 h-5 flex-shrink-0 ${config.textColor}`} />
                <p className="text-white text-sm flex-1">{toast.message}</p>
                <button
                  onClick={() => removeToast(toast.id)}
                  className="p-1 text-gray-400 hover:text-white transition-colors"
                >
                  <X className="w-4 h-4" />
                </button>
              </motion.div>
            );
          })}
        </AnimatePresence>
      </div>
    </ToastContext.Provider>
  );
}

export function useToast() {
  const context = useContext(ToastContext);
  if (!context) {
    throw new Error('useToast must be used within a ToastProvider');
  }
  return context;
}

ده نظام Toast Messages أو الإشعارات الصغيرة. شوف ازاي عملناه:

  • ToastContext: ده Context من React. بيسمح لأي مكون في التطبيق يستخدم الـ toast من غير ما يمرره كـ prop.
  • toastTypes: عندك 4 أنواع: success (أخضر)، error (أحمر)، warning (أصفر)، info (أزرق). كل نوع ليه لون مختلف ونوع أيقونة.
  • addToast: الدالة دي بتضيف toast جديد. بتاخد الرسالة، النوع، والمدة. المدة الافتراضية 5 ثواني.
  • removeToast: الدالة دي بتحذف toast معين. بتستخدم لما المستخدم يضغط على زرار X.
  • useCallback: بيستخدم لتحسين الأداء. بيمنع إعادة إنشاء الدوال في كل render.

الـ toast بيظهر في الزاوية السفلية اليسرى (bottom-4 right-4). بيبدأ بـ opacity 0 و x: 100 (بره الشاشة من اليمين) وبعدين بيظهر بسلاسة. ولما بيختفي، بيختفي بنفس الطريقة.

💡 امتى تستخدم Toast؟
  • نجاح: "تمت إضافة المنتج للسلة بنجاح!"
  • خطأ: "حدث خطأ أثناء إضافة المنتج. يرجى المحاولة مرة أخرى."
  • تحذير: "الكمية المطلوبة غير متوفرة. الكمية المتاحة: 3."
  • معلومات: "تم نسخ رابط المنتج."

🛡️ الخطوة 4: Error Boundary

components/ui/ErrorBoundary.jsx jsx 
"use client";

import { Component } from 'react';
import { AlertTriangle, RefreshCw } from 'lucide-react';
import Button from './Button';

class ErrorBoundary extends Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  componentDidCatch(error, errorInfo) {
    console.error('ErrorBoundary caught:', error, errorInfo);
    
    // إرسال الخطأ إلى نظام المراقبة
    // Sentry.captureException(error, { extra: errorInfo });
  }

  handleRetry = () => {
    this.setState({ hasError: false, error: null });
  };

  render() {
    if (this.state.hasError) {
      if (this.props.fallback) {
        return this.props.fallback;
      }

      return (
        <div className="min-h-[200px] flex items-center justify-center p-8">
          <div className="text-center">
            <div className="w-16 h-16 mx-auto rounded-full bg-red-500/20 flex items-center justify-center mb-4">
              <AlertTriangle className="w-8 h-8 text-red-400" />
            </div>
            <h3 className="text-lg font-bold text-white mb-2">
              حدث خطأ
            </h3>
            <p className="text-gray-400 text-sm mb-4">
              {this.props.fallbackMessage || 'حدث خطأ غير متوقع في هذا المكون'}
            </p>
            <Button size="sm" onClick={this.handleRetry}>
              <RefreshCw className="w-4 h-4 ml-2" />
              إعادة المحاولة
            </Button>
          </div>
        </div>
      );
    }

    return this.props.children;
  }
}

export default ErrorBoundary;

ده Error Boundary. شوف ازاي بيشتغل:

  • Class Component: لازم يكون Class Component مش Functional Component. ده لأن React بيقدم getDerivedStateFromError و componentDidCatch في Class Components بس.
  • getDerivedStateFromError: دي static method بتتحدد لما يحصل خطأ في أي child component. بتغير state لـ hasError: true.
  • componentDidCatch: دي بتشتغل بعد الخطأ وبتسجل التفاصيل. مفيدة لتسجيل الأخطاء في نظام مراقبة.
  • fallback: لو انت عايز تتحكم في شكل الخطأ، بتبعت fallback component كـ prop.
  • handleRetry: لما المستخدم يضغط "إعادة المحاولة"، الـ state بيرجع لـ hasError: false والـ children بتظهر تاني.

الـ Error Boundary مهم جداً لأنه بيحمي أجزاء معينة من الصفحة من الأخطاء. يعني لو مكون الصور اتعمله crash، الصفحة كلها مش هتتأثر. الـ Error Boundary هيظهر رسالة خطأ في مكان الصور بس.

⚠️ ملاحظة مهمة
Error Boundary مش بيمسك أخطاء في الأحداث (Events) أو asynchronous code (async/await). بيمسك بس أخطاء في أثناء عرض المكونات (render). يعني لو حصل خطأ في useEffect، الـ Error Boundary مش هيمسكه.

📝 الخطوة 5: معالجة أخطاء النماذج

components/ui/FormValidation.jsx jsx 
"use client";

import { useState } from 'react';
import { AlertCircle, CheckCircle } from 'lucide-react';
import { cn } from '@/lib/cn';

export function FormField({
  label,
  name,
  type = 'text',
  value,
  onChange,
  error,
  success,
  placeholder,
  disabled,
  required,
  className,
}) {
  return (
    <div className={cn("space-y-2", className)}>
      {label && (
        <label htmlFor={name} className="block text-sm font-medium text-gray-200">
          {label}
          {required && <span className="text-red-400 mr-1">*</span>}
        </label>
      )}
      <div className="relative">
        <input
          id={name}
          name={name}
          type={type}
          value={value}
          onChange={onChange}
          placeholder={placeholder}
          disabled={disabled}
          className={cn(
            "w-full px-4 py-3 rounded-xl bg-white/5 border text-white placeholder-gray-500",
            "focus:outline-none focus:ring-2 transition-all duration-200",
            error
              ? "border-red-500 focus:ring-red-500"
              : success
              ? "border-green-500 focus:ring-green-500"
              : "border-white/10 focus:ring-indigo-500",
            disabled && "opacity-50 cursor-not-allowed"
          )}
        />
        {(error || success) && (
          <div className="absolute left-3 top-1/2 -translate-y-1/2">
            {error ? (
              <AlertCircle className="w-5 h-5 text-red-400" />
            ) : (
              <CheckCircle className="w-5 h-5 text-green-400" />
            )}
          </div>
        )}
      </div>
      {error && (
        <p className="text-red-400 text-sm flex items-center gap-1">
          <AlertCircle className="w-4 h-4" />
          {error}
        </p>
      )}
      {success && (
        <p className="text-green-400 text-sm flex items-center gap-1">
          <CheckCircle className="w-4 h-4" />
          {success}
        </p>
      )}
    </div>
  );
}

export function FormError({ message, className }) {
  if (!message) return null;

  return (
    <div className={cn(
      "flex items-center gap-2 p-3 bg-red-500/10 border border-red-500/20 rounded-xl",
      className
    )}>
      <AlertCircle className="w-5 h-5 text-red-400 flex-shrink-0" />
      <p className="text-red-400 text-sm">{message}</p>
    </div>
  );
}

export function FormSuccess({ message, className }) {
  if (!message) return null;

  return (
    <div className={cn(
      "flex items-center gap-2 p-3 bg-green-500/10 border border-green-500/20 rounded-xl",
      className
    )}>

ده نظام التحقق من صحة النماذج. شوف ازاي عملنا 3 مكونات:

  • FormField: ده مكون حقل إدخال ذكي. بيغيّر لونه حسب الحالة: أحمر لو فيه خطأ، أخضر لو البيانات صحيحة، وعادي لو مفيش حاجة. كمان بيعرض أيقونة بجانب الحقل.
  • FormError: ده رسالة خطأ عامة للنموذج. بيظهر في أعلى النموذج لما يكون فيه أخطاء.

الـ FormField مهم جداً لأنه بيوفر تجربة مستخدم ممتازة. المستخدم بيشوف الخطأ فوراً في الحقل نفسه، مش في مكان تاني. وده بيقلل الارتباك.

🌐 الخطوة 6: معالجة أخطاء API

utils/api.js javascript 
/**
 * دالة موحدة لإجراء استدعاءات API مع معالجة الأخطاء
 */
export async function apiCall(url, options = {}) {
  const {
    method = 'GET',
    headers = {},
    body,
    timeout = 10000,
  } = options;

  // إعداد Headers الافتراضية
  const defaultHeaders = {
    'Content-Type': 'application/json',
  };

  // دمج Headers
  const mergedHeaders = {
    ...defaultHeaders,
    ...headers,
  };

  // إعداد الـ AbortController للـ timeout
  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), timeout);

  try {
    const fetchOptions = {
      method,
      headers: mergedHeaders,
      signal: controller.signal,
    };

    if (body && method !== 'GET') {
      fetchOptions.body = JSON.stringify(body);
    }

    const response = await fetch(url, fetchOptions);
    clearTimeout(timeoutId);

    // التحقق من نوع المحتوى
    const contentType = response.headers.get('content-type');
    let data;

    if (contentType?.includes('application/json')) {
      data = await response.json();
    } else {
      data = await response.text();
    }

    // التحقق من نجاح الطلب
    if (!response.ok) {
      const error = new Error(data.message || data.error || 'حدث خطأ في الخادم');
      error.status = response.status;
      error.data = data;
      throw error;
    }

    return { success: true, data };
  } catch (error) {
    clearTimeout(timeoutId);

    // معالجة أنواع الأخطاء المختلفة
    if (error.name === 'AbortError') {
      return {
        success: false,
        error: {
          type: 'timeout',
          message: 'انتهت مهلة الاتصال بالخادم',
        },
      };
    }

    if (!navigator.onLine) {
      return {
        success: false,
        error: {
          type: 'network',
          message: 'لا يوجد اتصال بالإنترنت',
        },
      };
    }

    return {
      success: false,
      error: {
        type: error.status ? 'api' : 'unknown',
        status: error.status,
        message: error.message || 'حدث خطأ غير متوقع',
        data: error.data,
      },
    };
  }
}

/**
 * Hook لاستخدام API مع حالة التحميل
 */
export function useApi() {
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const execute = async (url, options) => {
    setLoading(true);
    setError(null);

    const result = await apiCall(url, options);

    if (!result.success) {
      setError(result.error);
    }

    setLoading(false);
    return result;
  };

  const clearError = () => setError(null);

  return { loading, error, execute, clearError };
}

ده المكون الأهم في معالجة أخطاء API. شوف ازاي عملنا دالة موحدة بتعالج كل أنواع الأخطاء:

  • Timeout: لو السيرفر مردش خلال 10 ثواني، الـ request بتتلغى وبنرجع خطأ timeout.
  • Network Error: لو المستخدم مش متصال بالإنترنت، بنرجع خطأ network.
  • API Error: لو السيرفر رجع status code غلط (زي 404 أو 500)، بنرجع الخطأ مع الرسالة.
  • Unknown Error: لو حصل أي حاجة مش معروفة، بنرجع خطأ عام.

الـ AbortController مهم جداً. بيعمل timeout للطلب. لو السيرفر مردش خلال 10 ثواني، الطلب بتلغى تلقائياً. وده بيمنع المستخدم من الانتظار للأبد.

💡 ليه بنعمل دالة موحدة؟
بدل ما نكتب كود معالجة أخطاء في كل مكان، بنكتبها مرة واحدة في apiCall وبنستخدمها في كل مكان. وده بيقلل تكرار الكود وبيخلي الصيانة أسهل.

🔄 الخطوة 7: Hook لمعالجة الأخطاء

hooks/useErrorHandler.js javascript 
"use client";

import { useCallback } from 'react';
import { useToast } from '@/components/ui/Toast';

/**
 * Hook موحد لمعالجة الأخطاء
 */
export function useErrorHandler() {
  const { error: showError, warning, info } = useToast();

  const handleError = useCallback((error, context = {}) => {
    console.error('Error:', error, context);

    // تحديد نوع الخطأ وعرض رسالة مناسبة
    if (error?.status === 401) {
      showError('يرجى تسجيل الدخول أولاً', 5000);
      // يمكنك إعادة التوجيه لصفحة تسجيل الدخول
      // router.push('/sign-in');
    } else if (error?.status === 403) {
      showError('ليس لديك صلاحية للقيام بهذه العملية', 5000);
    } else if (error?.status === 404) {
      warning('العنصر المطلوب غير موجود', 5000);
    } else if (error?.status === 422) {
      // أخطاء التحقق من صحة البيانات
      const validationErrors = error.data?.errors;
      if (validationErrors) {
        Object.values(validationErrors).flat().forEach((msg) => {
          showError(msg, 5000);
        });
      } else {
        showError(error.message || 'بيانات غير صالحة', 5000);
      }
    } else if (error?.status === 429) {
      warning('طلبات كثيرة جداً، يرجى الانتظار قليلاً', 5000);
    } else if (error?.status >= 500) {
      showError('خطأ في الخادم، يرجى المحاولة لاحقاً', 5000);
    } else if (error?.type === 'network') {
      showError('لا يوجد اتصال بالإنترنت', 5000);
    } else if (error?.type === 'timeout') {
      warning('انتهت مهلة الاتصال، يرجى المحاولة مجدداً', 5000);
    } else {
      showError(error?.message || 'حدث خطأ غير متوقع', 5000);
    }

    // يمكنك إرسال الخطأ إلى نظام مراقبة
    // Sentry.captureException(error, { extra: context });
  }, [showError, warning]);

  const handleAsyncError = useCallback(async (asyncFn, context = {}) => {
    try {
      return await asyncFn();
    } catch (error) {
      handleError(error, context);
      throw error;
    }
  }, [handleError]);

  return {
    handleError,
    handleAsyncError,
  };
}

/**
 * Hook للتعامل مع أخطاء النماذج
 */
export function useFormErrors() {
  const { handleError } = useErrorHandler();

  const handleFormError = useCallback((error, setErrors) => {
    if (error?.status === 422 && error.data?.errors) {
      // تحويل أخطاء الـ Validation إلى شكل يفهمه النموذج
      const formErrors = {};
      Object.entries(error.data.errors).forEach(([field, messages]) => {
        formErrors[field] = Array.isArray(messages) ? messages[0] : messages;
      });
      setErrors(formErrors);
    } else {
      handleError(error);
    }
  }, [handleError]);

  return { handleFormError };
}

ده الـ Hook اللي بيجمع كل منطق معالجة الأخطاء في مكان واحد. شوف ازاي بيعالج كل نوع خطأ برسالة مناسبة:

  • 401 (غير مصرح): "يرجى تسجيل الدخول أولاً". ممكن كمان نتوجيه المستخدم لصفحة تسجيل الدخول.
  • 403 (ممنوع): "ليس لديك صلاحية للقيام بهذه العملية". يعني المستخدم مش admin مثلاً.
  • 404 (غير موجود): "العنصر المطلوب غير موجود". رسالة واضحة ومضحكة.
  • 422 (بيانات غير صالحة): بيعرض كل الأخطاء في الحقول. يعني لو البريد الإلكتروني غلط واسم المستخدم غلط، بيتعرض الخطئين.
  • 429 (طلبات كثيرة): "طلبات كثيرة جداً، يرجى الانتظار قليلاً". ده لما المستخدم يضغط كتير بسرعة.
  • 500+ (خطأ في السيرفر): "خطأ في الخادم، يرجى المحاولة لاحقاً". رسالة عامة ومفهومة.
  • Network Error: "لا يوجد اتصال بالإنترنت". مفيد لل usoires اللي بيستخدموا الموبايل.
  • Timeout: "انتهت مهلة الاتصال، يرجى المحاولة مجدداً".
💡 نصيحة مهمة
لاحظ إن كل رسالة خطأ بتختفي بعد 5 ثواني (duration = 5000). ده عشان الرسالة متأخرش كتير وتزحم الشاشة. لكن لو الرسالة مهمة زي "يرجى تسجيل الدخول"، ممكن نسيبها تظهر لحد ما المستخدم يتفاعل.

ده مكون موحد لعرض رسائل الخطأ. شوف ازاي بينوع حسب نوع الخطأ:

  • error: أحمر مع أيقونة AlertCircle. للخطأ العام.
  • network: أصفر مع أيقونة WifiOff. لمشاكل الشبكة.
  • notFound: أزرق مع أيقونة AlertCircle. لما العنصر مش موجود.

💡 امتى تستخدم ErrorMessage؟
استخدمه في الأماكن اللي ممكن يحصل فيها خطأ ومش عايز تكمل الصفحة. مثلاً: لو قسم المنتجات مش عارف يتحمل، حط ErrorMessage في مكان القسم ده. وده أحسن ما تسيب الصفحة كلها تتعمل crash.

🎯 الأخطاء الشائعة في معالجة الأخطاء

⚠️ غلطات لازم تتجنبها
  • إظهار أخطاء تقنية للمستخدم: لا تعرض رسائل زي "TypeError: Cannot read property 'map' of undefined". المستخدم مش فاهم حاجة من دول.
  • مفيش retry mechanism: لو التحميل فشل ومفيش زرار "إعادة المحاولة"، المستخدم هيسيب الصفحة.
  • كل الأخطاء بلون أحمر: استخدم ألوان مختلفة حسب نوع الخطأ. أحمر للخطأ الخطير، أصفر للتحذير، أزرق للمعلومات.
  • رسائل خطأ طويلة: رسائل الخطأ لازم تكون قصيرة ومباشرة. "حدث خطأ" أحسن من "حدث خطأ غير متوقع أثناء محاولة جلب البيانات من الخادم".
  • مفيش logging: لازم تسجل الأخطاء في console أو في نظام مراقبة عشان تعرف بيحصل ايه.

📝 تمارين عملية

🎯 تمرين 1: أنشئ صفحة 404 مخصصة
جرّب تعدل على صفحة 404 الموجودة. غيّر الرسالة والشكل. مثلاً: استبدل الشبح بشاحنة بتوصيل مكسورة، أو استبدل الرسالة برسالة مضحكة مختلفة.
🎯 تمرين 2: أضف Error Boundary لقسم معين

🎯 تمرين 3: جرّب Toast Messages
استخدم useToast في أي صفحة. اطبع success و error و warning و info. شوف ازاي الإشعارات بتظهر وتختفي.
🎯 تمرين 4: أنشئ صفحة خطأ مخصصة لقسم الدفع
أنشئ ملف app/(shop)/checkout/error.js بتصميم مختلف عن صفحة الخطأ العامة. مثلاً: استخدم أيقونة بطاقة ائتمان مكسورة بدل AlertTriangle.
