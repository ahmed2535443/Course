💡 مقدمة عن حالات التحميل

يعيش يا صديقي! انت في الدرس العشرين من الكورس، وده درس مهم جداً. خليني أقولك حاجة مهمة: لما مستخدم يفتح صفحة في موقعك والبيانات بتتحمل، لو مفيش أي مؤشر على إن في حاجة بيحصل، هو هيحس إن الموقع واقف أو مش شغال. وده أكبر عدو لأي متجر إلكتروني!

حالات التحميل (Loading States) هي בעצם الوضع اللي بيبين للمستخدم إن في عملية تحميل بتحصل. لما بتحط loading state كويس، المستخدم بيفضل ي等待 потому что بيشوف إن الموقع شغال ومحتاج وقت بسيط. لكن لو مفيش loading state، المستخدم ممكن يفكر إن الموقع عطلان ويرجع لورا.

💡 ليه حالات التحميل مهمة؟
  • بتقلل معدل الارتداد (Bounce Rate) لأن المستخدم بيفضل في الصفحة
  • بتحسن تجربة المستخدم بشكل كبير
  • بتدي انطباع احترافي للموقع
  • بتمنع المستخدم من الضغط على أزرار متكررة أثناء التحميل
  • بتقلل القلق والتوتر عند المستخدم

تخيل معايا الموقف ده: انت فتحت موقع عشان تشتري موبايل جديد. دوست على "إضافة للسلة" ومفيش أي رد. هتضغط تاني؟ هتضغط تالت؟ وبعدين هتلاقي عندك 4 موبايلات في السلة بدل واحد! الوضع ده بيحصل لما مفيش loading state كويس.

📁 هيكل ملفات حالات التحميل

📁 app/
├── 📄 loading.js # تحميل عام
├── 📁 (shop)/
│ ├── 📄 loading.js # تحميل صفحات المتجر
│ ├── 📁 products/
│ │ └── 📄 loading.js # تحميل صفحة المنتجات
│ └── 📁 products/[id]/
│ └── 📄 loading.js # تحميل تفاصيل المنتج
📁 components/ui/
├── 📄 Skeleton.jsx # مكون Skeleton
├── 📄 Spinner.jsx # مكون Spinner
├── 📄 LoadingCard.jsx # بطاقة تحميل
└── 📄 PageLoader.jsx # حامل صفحة كامل

شوف يا صديقي، الهيكل ده بيبينلك ازاي تنظم ملفات التحميل. كل مجلد ممكن يكون عنده ملف loading.js خاص بيه. يعني مثلاً لو انت فتحت صفحة المنتجات، هيظهر تحميل خاص بالمنتجات. ولو فتحت تفاصيل منتج معين، هيظهر تحميل مختلف. الهدف إن كل صفحة يكون عندها تحميل يناسب محتواها.

⏳ الخطوة 1: ملفات loading.js في Next.js

في Next.js App Router، يمكنك إنشاء ملف loading.js في أي مجلد لعرض حالة التحميل أثناء جلب البيانات. هذا الملف يظهر تلقائياً أثناء تحميل الصفحة. يعني انت بس بتنشئ الملف وبتحط فيه التصميم اللي عايزه، وNext.js هتتولى باقي المهام.

⚠️ ملاحظة مهمة
ملف loading.js لازم يكون في نفس المجلد مع page.js. يعني لو عندك مجلد app/products، لازم يكون في loading.js جوا مجلد products. لو حطيته في مجلد تاني، مش هيشتغل.
app/loading.js jsx 
export default function Loading() {
  return (
    <div className="min-h-screen flex items-center justify-center bg-gray-950">
      <div className="text-center">
        {/* Spinner */}
        <div className="relative w-16 h-16 mx-auto mb-6">
          <div className="absolute inset-0 border-4 border-indigo-500/20 rounded-full" />
          <div className="absolute inset-0 border-4 border-transparent border-t-indigo-500 rounded-full animate-spin" />
        </div>
        
        {/* Text */}
        <p className="text-gray-400 text-lg">جاري التحميل...</p>
      </div>
    </div>
  );
}

شوف الكود ده يا صديقي. ايه اللي بيحصل هنا؟ احنا بنعمل دالة اسمها Loading وبترجع JSX فيه Spinner (الدوامة اللي بتدور) ونص بيقول "جاري التحميل". لما المستخدم يفتح أي صفحة، النسخة دي بتظهر لحد ما البيانات تتحمل.

الـ Spinner ده عبارة عن دائرة فاضية ودوامة جواها بتلف. الدائرة الفاضية بلون رمادي شفاف (border-indigo-500/20) والدوامة بلون نيلي (border-t-indigo-500). الحركة بتتعمل بـ animate-spin بتاع Tailwind CSS.

💡 نصيحة ذهبية
حاول دايماً إن الـ skeleton يكون نفس حجم وشكل المحتوى الحقيقي. لو المنتج النهائي فيه صورة مربعة ونص وسعر، الـ skeleton لازم يكون فيه مربع رمادي وخط رمادي وخط رمادي أقصر. المستخدم لازم يحس إن الصفحة بتترسم وبتترسم.

💀 الخطوة 2: مكون Skeleton

components/ui/Skeleton.jsx jsx 
"use client";

import { cn } from '@/lib/cn';

export function Skeleton({ className, ...props }) {
  return (
    <div
      className={cn(
        "animate-pulse bg-gray-800 rounded",
        className
      )}
      {...props}
    />
  );
}

export function SkeletonText({ lines = 3, className }) {
  return (
    <div className={cn("space-y-2", className)}>
      {[...Array(lines)].map((_, i) => (
        <Skeleton
          key={i}
          className={cn(
            "h-4",
            i === lines - 1 ? "w-3/4" : "w-full"
          )}
        />
      ))}
    </div>
  );
}

export function SkeletonCircle({ size = 40, className }) {
  return (
    <Skeleton
      className={cn("rounded-full", className)}
      style={{ width: size, height: size }}
    />
  );
}

export function SkeletonRectangle({ width, height, className }) {
  return (
    <Skeleton
      className={cn("rounded-lg", className)}
      style={{ width, height }}
    />
  );
}

ده المكون الأساسي للـ Skeleton. شوف ازاي عملنا 4 مكونات مختلفة:

  • Skeleton: المكون الأساسي. ده مربع رمادي متحرك. بتاع animate-pulse بيخلي المربع يظهر ويختفي ببطء (opacity goes up and down). وده بيخلي المربع يحسسك إنه حي ومش ثابت.
  • SkeletonText: ده بيعرض أسطر نص رمادية. كل سطر فيه ارتفاع ثابت (h-4) والسطر الأخير بيكون أقصر (w-3/4) عشان يحاكي الشكل الطبيعي للنصوص.
  • SkeletonCircle: ده دائرة رمادية. مفيد لصور البروفايل والأيقونات.
  • SkeletonRectangle: ده مستطيل بالحجم اللي انت تختاره. مفيد للصور والبطاقات.

الدالة cn اللي بنستخدمها دي مهمة جداً. هي بتمزج الأكلاسات بشكل ذكي. يعني لو انت عملت custom className وداخله animate-pulse كمان، الدالة هتمزجهم من غير ما يتكرروا.

⭕ الخطوة 4: مكون Spinner

components/ui/Spinner.jsx jsx 
"use client";

import { cn } from '@/lib/cn';

const sizes = {
  sm: 'w-4 h-4 border-2',
  md: 'w-8 h-8 border-3',
  lg: 'w-12 h-12 border-4',
  xl: 'w-16 h-16 border-4',
};

export default function Spinner({
  size = 'md',
  color = 'indigo',
  className,
  fullScreen = false,
  text,
}) {
  const colorClasses = {
    indigo: 'border-indigo-500 border-t-transparent',
    white: 'border-white border-t-transparent',
    gray: 'border-gray-500 border-t-transparent',
  };

  const spinner = (
    <div className={cn("flex flex-col items-center justify-center gap-4", className)}>
      <div
        className={cn(
          "rounded-full animate-spin",
          sizes[size],
          colorClasses[color]
        )}
      />
      {text && (
        <p className="text-gray-400 text-sm">{text}</p>
      )}
    </div>
  );

  if (fullScreen) {
    return (
      <div className="fixed inset-0 bg-gray-950/80 backdrop-blur-sm z-50 flex items-center justify-center">
        {spinner}
      </div>
    );
  }

  return spinner;
}

// Inline Spinner for buttons
export function ButtonSpinner({ className }) {
  return (
    <div
      className={cn(
        "w-5 h-5 border-2 border-white/30 border-t-white rounded-full animate-spin",
        className
      )}
    />
  );
}

ده المكون اللي بيعمل الدوامة (Spinner). شوف ازاي عملناه مرن جداً:

  • الحجم (size): عندك 4 أحجام مختلفة. sm للأزرار الصغيرة، md للعناصر المتوسطة، lg للعناصر الكبيرة، xl للشاشات الكاملة.
  • اللون (color): عندك 3 ألوان. indigo للون الأساسي، white للخلفيات الداكنة، gray للخلفيات الفاتحة.
  • fullScreen: لو ضغطت true، هيبان في نص الشاشة مع خلفية شفافة وضبابية. مفيد للتحميل الكامل.
  • text: لو حطيت نص، هيظهر تحت الـ Spinner. مفيد عشان توضح للمستخدم بيحصل ايه.

الـ ButtonSpinner ده مكون صغير مخصص للأزرار. لما المستخدم يضغط على زرار "إضافة للسلة"، الـ Spinner ده بيظهر جوه الزرار.

⚠️ غلطة شائعة
بعض المطورين بيستخدموا Spinner في كل مكان. الـ Spinner مفيد للعمليات القصيرة (أقل من 2 ثانية). لو العملية بتاخد أكتر من كده، استخدم Skeleton عشان المستخدم يفهم المحتوى اللي هيشوفه.

📄 الخطوة 5: حامل صفحة كامل

components/ui/PageLoader.jsx jsx 
"use client";

import { motion } from 'framer-motion';
import { Loader2 } from 'lucide-react';

export default function PageLoader({ text = 'جاري التحميل...' }) {
  return (
    <div className="min-h-[400px] flex items-center justify-center">
      <motion.div
        initial={{ opacity: 0, scale: 0.8 }}
        animate={{ opacity: 1, scale: 1 }}
        className="text-center"
      >
        {/* Animated Spinner */}
        <div className="relative w-20 h-20 mx-auto mb-6">
          {/* Outer Ring */}
          <motion.div
            animate={{ rotate: 360 }}
            transition={{ duration: 2, repeat: Infinity, ease: "linear" }}
            className="absolute inset-0 border-4 border-indigo-500/20 rounded-full"
          />
          
          {/* Inner Ring */}
          <motion.div
            animate={{ rotate: -360 }}
            transition={{ duration: 1.5, repeat: Infinity, ease: "linear" }}
            className="absolute inset-2 border-4 border-transparent border-t-purple-500 rounded-full"
          />
          
          {/* Center Icon */}
          <div className="absolute inset-0 flex items-center justify-center">
            <Loader2 className="w-6 h-6 text-indigo-400 animate-spin" />
          </div>
        </div>

        {/* Text */}
        <motion.p
          initial={{ opacity: 0 }}
          animate={{ opacity: 1 }}
          transition={{ delay: 0.3 }}
          className="text-gray-400"
        >
          {text}
        </motion.p>
      </motion.div>
    </div>
  );
}

ده مكون أنيق جداً للتحميل الكامل للصفحة. شوف ازاي عملنا حركات مع Framer Motion:

  • الحلقة الخارجية: بلف 360 درجة في 2 ثانية وبتكرر للأبد. السرعة ثابتة (ease: "linear").
  • الحلقة الداخلية: بلف 360 درجة في الاتجاه المعاكس (-360) وفي 1.5 ثانية. ده بيخلق حركة مثيره للانتباه.
  • الأيقونة الوسط: أيقونة Loader2 من مكتبة lucide-react وبتدور بدوران عادي (animate-spin).
  • النص: بيظهر بعد 0.3 ثانية (delay: 0.3) عشان يظهر بعد الحركات.

الـ motion.div الأولاني عنده initial: opacity 0 و scale 0.8. يعني بيبدأ مش ظاهر وصغير. وبعدين بيظهر (opacity: 1) وبيكبر (scale: 1) بحركة ناعمة. وده بيخلي التحميل يبدأ بتأثير جميل.

📊 الخطوة 6: هيكل تحميل للجدول

components/ui/TableSkeleton.jsx jsx 
import { Skeleton } from './Skeleton';

export default function TableSkeleton({ rows = 5, columns = 5 }) {
  return (
    <div className="bg-white/5 backdrop-blur-lg rounded-2xl border border-white/10 overflow-hidden">
      {/* Header */}
      <div className="flex items-center gap-4 p-4 border-b border-white/10">
        {[...Array(columns)].map((_, i) => (
          <Skeleton key={i} className="flex-1 h-4" />
        ))}
      </div>

      {/* Rows */}
      <div className="divide-y divide-white/5">
        {[...Array(rows)].map((_, rowIndex) => (
          <div key={rowIndex} className="flex items-center gap-4 p-4 animate-pulse">
            {[...Array(columns)].map((_, colIndex) => (
              <div key={colIndex} className="flex-1">
                {colIndex === 0 ? (
                  <div className="flex items-center gap-3">
                    <Skeleton className="w-10 h-10 rounded-lg flex-shrink-0" />
                    <Skeleton className="h-4 flex-1" />
                  </div>
                ) : colIndex === columns - 1 ? (
                  <div className="flex gap-2">
                    <Skeleton className="w-8 h-8 rounded-lg" />
                    <Skeleton className="w-8 h-8 rounded-lg" />
                  </div>
                ) : (
                  <Skeleton className="h-4 w-3/4" />
                )}
              </div>
            ))}
          </div>
        ))}
      </div>
    </div>
  );
}

ده مكون لتحميل الجداول. شوف ازاي بنميّز بين الأعمدة المختلفة:

  • العمود الأول (colIndex === 0): فيه صورة صغيرة (مربع 10x10) ونص. ده بيحاكي صورة المنتج واسمه.
  • العمود الأخير (colIndex === columns - 1): فيه زرين صغيرين. ده بيحاكي أزرار التعديل والحذف.
  • باقي الأعمدة: مجرد سطر رمادي (Skeleton بحجم h-4 w-3/4).

الـ props اللي في المكون (rows و columns) بتسمحلك تتحكم في عدد الأعمدة والصفوف. يعني لو عندك جدول فيه 10 صفوف و 3 أعمدة، بتعمل كده:

مثال على الاستخدام jsx 
<TableSkeleton rows={10} columns={3} />

✨ الخطوة 8: استخدام حالات التحميل

components/products/ProductGrid.jsx jsx 
"use client";

import { useState, useEffect } from 'react';
import ProductCard from './ProductCard';
import ProductCardSkeleton from '@/components/ui/ProductCardSkeleton';
import { motion, AnimatePresence } from 'framer-motion';

export default function ProductGrid({ initialProducts = [], filters }) {
  const [products, setProducts] = useState(initialProducts);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetchProducts();
  }, [filters]);

  const fetchProducts = async () => {
    setLoading(true);
    setError(null);

    try {
      const params = new URLSearchParams();
      if (filters?.category) params.set('category', filters.category);
      if (filters?.minPrice) params.set('minPrice', filters.minPrice);
      if (filters?.maxPrice) params.set('maxPrice', filters.maxPrice);
      if (filters?.sort) params.set('sort', filters.sort);

      const response = await fetch(`/api/products?${params.toString()}`);
      
      if (!response.ok) {
        throw new Error('فشل في جلب المنتجات');
      }

      const data = await response.json();
      setProducts(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  if (error) {
    return (
      <div className="text-center py-12">
        <p className="text-red-400 mb-4">{error}</p>
        <button
          onClick={fetchProducts}
          className="px-4 py-2 bg-indigo-500 text-white rounded-xl hover:bg-indigo-600 transition-colors"
        >
          إعادة المحاولة
        </button>
      </div>
    );
  }

  return (
    <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-6">
      <AnimatePresence mode="wait">
        {loading ? (
          // Show skeletons while loading
          [...Array(8)].map((_, i) => (
            <motion.div
              key={`skeleton-${i}`}
              initial={{ opacity: 0 }}
              animate={{ opacity: 1 }}
              exit={{ opacity: 0 }}
              transition={{ delay: i * 0.05 }}
            >
              <ProductCardSkeleton />
            </motion.div>
          ))
        ) : (
          // Show actual products
          products.map((product, index) => (
            <motion.div
              key={product.id}
              initial={{ opacity: 0, y: 20 }}
              animate={{ opacity: 1, y: 0 }}
              transition={{ delay: index * 0.05 }}
            >
              <ProductCard product={product} />
            </motion.div>
          ))
        )}
      </AnimatePresence>
    </div>
  );
}

ده المكون الأخير اللي بيجمع كل حاجة مع بعض. شوف ازاي بيستخدم الـ state للتحكم في التحميل:

  • loading: boolean. لو true، بنعرض الـ skeletons. لو false، بنعرض المنتجات الحقيقية.
  • error: لو حصل خطأ، بنعرض رسالة خطأ مع زرار "إعادة المحاولة".
  • AnimatePresence: ده من Framer Motion. بيخلي الـ skeletons تظهر وتختفي بسلاسة.
  • delay: i * 0.05: ده بيخلي كل skeleton يظهر بعد اللي قبله بـ 0.05 ثانية. وده بيخليهم يظهروا بالتسلسل (stagger effect).

الـ finally block مهم جداً. حتى لو حصل خطأ، الـ loading هيبقى false. يعني الـ skeletons اختفت والمستخدم شاف رسالة الخطأ.

💡 نصائح لتحسين تجربة التحميل
  • استخدم Suspense مع loading.js لتحميل الأجزاء البطيئة
  • أضف تأخير صغير (300ms) قبل إظهار الـ Skeleton لتجنب الوميض
  • استخدم staggerChildren في Framer Motion لتحريك العناصر تدريجياً
  • حافظ على حجم الـ Skeleton نفس حجم المحتوى الفعلي
  • لا تستخدم Spinner للعمليات طويلة - استخدم Skeleton
  • أضف نص توضيحي مع الـ Spinner عشان المستخدم يعرف بيحصل ايه
  • اختبر على اتصال بطيء (3G) عشان تشوف التحميل من نظرة المستخدم

🎨 الأخطاء الشائعة في حالات التحميل

⚠️ غلطات لازم تتجنبها
  • الkeleton بحجم مختلف عن المحتوى الحقيقي: لو الـ skeleton أكبر أو أصغر من المحتوى النهائي، المستخدم هيشوف "قفزة" لما المحتوى يظهر. وده بيكون مش لطيف.
  • استخدام Spinner في كل مكان: الـ Spinner مفيد للعمليات القصيرة. لو العملية بتاخد وقت، الـ Skeleton أحسن.
  • مفيش تحميل على الأزرار: لو المستخدم ضغط على زرار ومفيش تغيير، هيضغط كتير. لازم الزرار يبين إنه شغال.
  • الـ Loading state بيختفي فجأة: استخدم Framer Motion عشان الـ skeletons تختفي بسلاسة مش فجأة.
  • مفيش retry mechanism: لو التحميل فشل، لازم يكون فيه زرار "إعادة المحاولة".

📝 تمارين عملية

عشان تثبّت اللي اتعلمته، جرّب تعمل التمارين دي:

🎯 تمرين 1: أنشئ loading.js لصفحة السلة
أنشئ ملف app/(shop)/cart/loading.js يستخدم CartSkeleton. جرب تغير عدد العناصر وتغير الألوان.
🎯 تمرين 2: أنشئ Spinner مخصص
أنشئ نسخة جديدة من الـ Spinner بتصميم مختلف. مثلاً: دائرة بدل مربع، أو 3 نقاط بدل دوامة. جرب تغير الألوان والأحجام.
🎯 تمرين 3: أضف loading state للزرار
خد أي زرار في المشروع وأضفله loading state. لما المستخدم يضغط عليه، الزرار يتحول لـ Spinner لمدة ثانية وبعدين يرجع عادي.
🎯 تمرين 4: أنشئ Skeleton لصفحة الدفع
أنشئ مكون PaymentPageSkeleton يحاكي صفحة الدفع مع الحقول والمعلومات. جرب تستخدم الـ Skeleton components المختلفة.
