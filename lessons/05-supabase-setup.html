📦什么是 Supabase؟

Supabase هو بديل مفتوح المصدر لـ Firebase. بيقدم قاعدة بيانات PostgreSQL وتخزين سحابي ومصادقة و Realtime subscriptions.

مميزات Supabase:

  • PostgreSQL - قاعدة بيانات قوية وموثوقة
  • Realtime - تحديثات مباشرة
  • Storage - تخزين الصور والملفات
  • Auth - نظام مصادقة جاهز
  • Dashboard - لوحة تحكم سهلة
  • Free Tier - خطة مجانية سخية

🚀 الخطوة 1: إنشاء مشروع Supabase

1.1 التسجيل في Supabase

  1. ادخل على https://supabase.com
  2. اضغط "Start your project"
  3. سجّل الدخول باستخدام GitHub أو Email

1.2 إنشاء مشروع جديد

  1. من Dashboard، اضغط "New Project"
  2. اختار Organization (أو أنشأ واحد جديد)
  3. أدخل تفاصيل المشروع:
    • Name: market
    • Database Password: كلمة مرور قوية
    • Region: اختار أقرب منطقة (مثلاً EU West)
  4. اضغط "Create new project"
⚠️ تنبيه مهم
احتفظ بكلمة المرور في مكان آمن! مش هتقدر تتغير بعدين. كمان، استخدم كلمة مرور قوية تحتوي على أرقام وحروف وأحرف خاصة.

🗄️ الخطوة 2: إنشاء جدول المنتجات (Products)

بعد إنشاء المشروع، هنحتاج نإنشاء جداول قاعدة البيانات.

2.1 فتح SQL Editor

  1. من Dashboard، اختار "SQL Editor" من القائمة الجانبية
  2. اضغط "New query"

2.2 إنشاء جدول المنتجات

انسخ والصق الكود التالي في SQL Editor واضغط "Run":

SQL - إنشاء جدول المنتجات sql 
-- إنشاء جدول المنتجات
CREATE TABLE products (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT,
  price DECIMAL(10, 2) NOT NULL,
  image_url TEXT,
  category TEXT,
  stock INTEGER DEFAULT 0,
  featured BOOLEAN DEFAULT false,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- إنشاء فهرس على الفئة
CREATE INDEX idx_products_category ON products(category);

-- إنشاء فهرس على المنتجات المميزة
CREATE INDEX idx_products_featured ON products(featured);

2.3 شرح الحقول

الحقل النوع الوصف
id UUID المعرّف الفريد للمنتج (يُنشأ تلقائياً)
name TEXT اسم المنتج (إلزامي)
description TEXT وصف المنتج
price DECIMAL سعر المنتج (إلزامي)
image_url TEXT رابط صورة المنتج
category TEXT فئة المنتج
stock INTEGER المخزون المتوفر
featured BOOLEAN هل المنتج مميز؟
created_at TIMESTAMP تاريخ الإنشاء
updated_at TIMESTAMP تاريخ آخر تحديث

2.4 إنشاء جدول الطلبات (Orders)

SQL - إنشاء جدول الطلبات sql 
-- إنشاء جدول الطلبات
CREATE TABLE orders (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id TEXT NOT NULL,
  total DECIMAL(10, 2) NOT NULL,
  status TEXT DEFAULT 'pending',
  shipping_address JSONB,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- إنشاء جدول عناصر الطلب
CREATE TABLE order_items (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  order_id UUID REFERENCES orders(id) ON DELETE CASCADE,
  product_id UUID REFERENCES products(id),
  quantity INTEGER NOT NULL,
  price DECIMAL(10, 2) NOT NULL
);

-- إنشاء فهرس
CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_order_items_order_id ON order_items(order_id);

📁 الخطوة 3: إعداد Storage لرفع الصور

Supabase بيوفر Storage سحابي لرفع وتخزين الصور.

3.1 إنشاء Bucket جديد

  1. من Dashboard، اختار "Storage" من القائمة الجانبية
  2. اضغط "New bucket"
  3. أدخل التفاصيل:
    • Name: products
    • Public:فعّل (عشان الصور تظهر للكل)
  4. اض唾 "Create bucket"
💡 ملاحظة
الـ Public bucket يعني أي حد يقدر يوصل للصور بالـ URL مباشرة. ده مناسب لصور المنتجات اللي محتاجينها تكون متاحة للجميع.

🔐 الخطوة 4: RLS Policies

Row Level Security (RLS) هي ميزة في PostgreSQL بتسمحلك تتحكم في مين يقدر يقرأ أو يكتب البيانات.

4.1 تفعيل RLS على جدول المنتجات

SQL - تفعيل RLS sql 
-- تفعيل Row Level Security
ALTER TABLE products ENABLE ROW LEVEL SECURITY;

-- سياسة للقراءة: أي حد يقدر يقرأ المنتجات
CREATE POLICY "Products are viewable by everyone" 
ON products FOR SELECT 
USING (true);

-- سياسة للإدراج: فقط المستخدمين المسجلين
CREATE POLICY "Users can insert products" 
ON products FOR INSERT 
WITH CHECK (auth.role() = 'authenticated');

-- سياسة للتحديث: فقط المستخدمين المسجلين
CREATE POLICY "Users can update products" 
ON products FOR UPDATE 
USING (auth.role() = 'authenticated');

-- سياسة للحذف: فقط المستخدمين المسجلين
CREATE POLICY "Users can delete products" 
ON products FOR DELETE 
USING (auth.role() = 'authenticated');

4.2 شرح السياسات

السياسة العملية الشرط
Products are viewable by everyone SELECT true (الكل يقدر يقرأ)
Users can insert products INSERT auth.role() = 'authenticated'
Users can update products UPDATE auth.role() = 'authenticated'
Users can delete products DELETE auth.role() = 'authenticated'

4.3 RLS للطلبات

SQL - RLS للطلبات sql 
-- تفعيل RLS على جدول الطلبات
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;

-- المستخدم يقدر يشوف طلباته فقط
CREATE POLICY "Users can view their own orders" 
ON orders FOR SELECT 
USING (auth.uid()::text = user_id);

-- المستخدم يقدر ينشئ طلبات
CREATE POLICY "Users can create orders" 
ON orders FOR INSERT 
WITH CHECK (auth.uid()::text = user_id);

-- تفعيل RLS على جدول عناصر الطلب
ALTER TABLE order_items ENABLE ROW LEVEL SECURITY;

-- المستخدم يقدر يشوف عناصر طلباته
CREATE POLICY "Users can view their order items" 
ON order_items FOR SELECT 
USING (
  order_id IN (
    SELECT id FROM orders 
    WHERE user_id = auth.uid()::text
  )
);

🔗 الخطوة 5: الاتصال بـ Next.js

الآن محتاجين نوصل مشروع Next.js بـ Supabase.

5.1 تثبيت Supabase Client

Terminal bash 
npm install @supabase/supabase-js @supabase/ssr

5.2 إنشاء ملف البيئة

أنشئ ملف .env.local في جذر المشروع:

.env.local bash 
NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
💡 أين أجد هذه المفاتيح؟
من Supabase Dashboard → Settings → API. هتلاقي:
  • Project URL: رابط المشروع
  • anon public key: المفتاح العام

5.3 إنشاء Supabase Client

📁 lib/
└── 📄 supabase.js # Supabase Client
lib/supabase.js javascript 
import { createClient } from '@supabase/supabase-js';

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL;
const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY;

export const supabase = createClient(supabaseUrl, supabaseAnonKey);

5.4 اختبار الاتصال

أنشئ صفحة اختبار للتأكد من الاتصال:

app/test-supabase/page.js javascript 
import { supabase } from '@/lib/supabase';

export default async function TestSupabase() {
  // جلب المنتجات من قاعدة البيانات
  const { data: products, error } = await supabase
    .from('products')
    .select('*');

  if (error) {
    return <div>خطأ: {error.message}</div>;
  }

  return (
    <div>
      <h1>اختبار Supabase</h1>
      <p>عدد المنتجات: {products?.length || 0}</p>
      <pre>{JSON.stringify(products, null, 2)}</pre>
    </div>
  );
}
