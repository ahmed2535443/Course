🔍 لماذا SEO مهم للمتجر الإلكتروني؟

محركات البحث هي المصدر الرئيسي للزيارات للمتاجر الإلكترونية. بدون تحسين محركات البحث، ستفقد了很多 من الزوار المحتملين. في هذا الدرس، هنتعلم كيف نحسن موقعنا ليظهر في أول نتائج البحث.

💡 لماذا Next.js ممتاز لـ SEO؟
Next.js يوفر Server Side Rendering (SSR) و Static Site Generation (SSG) اللي بتساعد محركات البحث على فهم محتوى موقعك بشكل أفضل. كمان بيوفر prerendering للصفحات مما يحسن سرعة التحميل.

تخصيص Meta tags لكل صفحة

يمكنك تخصيص Meta tags لكل صفحة على حدة باستخدام generateMetadata:

app/products/[id]/page.js javascript 
export async function generateMetadata({ params }) {
  const product = await getProduct(params.id);
  
  return {
    title: product.name,
    description: product.description,
    openGraph: {
      title: product.name,
      description: product.description,
      images: [product.image],
      type: 'product',
      price: {
        amount: product.price,
        currency: 'SAR'
      }
    },
    twitter: {
      card: 'summary_large_image',
      title: product.name,
      description: product.description,
      images: [product.image]
    }
  };
}

📱 Open Graph tags

Open Graph tags بتستخدم لتحسين مظهر الروابط لما يتم مشاركتها على وسائل التواصل الاجتماعي مثل Facebook و LinkedIn.

⚠️ ملاحظة مهمة
تأكد من أن صور Open Graph بحجم 1200x630 بكسل للحصول على أفضل عرض. استخدم صيغة JPG أو PNG بحجم أقل من 1MB.
next.config.js javascript 
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: '**.supabase.co'
      },
      {
        protocol: 'https',
        hostname: '**.supabase.in'
      }
    ],
    formats: ['image/avif', 'image/webp'],
    minimumCacheTTL: 60 * 60 * 24 * 30
  },
  experimental: {
    optimizePackageImports: ['lucide-react', '@radix-ui/react-icons']
  }
};

module.exports = nextConfig;

🖼️ تحسين الصور مع Next.js Image

مكون Image في Next.js بيوفر تحسينات تلقائية للصور مثل:

  • التحميل الكسول (Lazy Loading)
  • التحجيم التلقائي حسب حجم الشاشة
  • تحويل الصور لصيغة WebP أو AVIF
  • منع Cumulative Layout Shift (CLS)
components/products/ProductCard.jsx jsx 
import Image from 'next/image';

export default function ProductCard({ product }) {
  return (
    <div className="product-card">
      <div className="relative aspect-square">
        <Image
          src={product.image}
          alt={product.name}
          fill
          sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
          className="object-cover rounded-lg"
          priority={false}
          loading="lazy"
          placeholder="blur"
          blurDataURL="data:image/jpeg;base64,/9j/4AAQSkZJRg..."
        />
      </div>
      <div className="p-4">
        <h3 className="font-semibold text-lg mb-2">{product.name}</h3>
        <p className="text-primary font-bold">{product.price} ر.س</p>
      </div>
    </div>
  );
}

✂️ Code Splitting التلقائي

Next.js يقوم بـ Code Splitting تلقائي لكل صفحة. هذا يعني أن المستخدم يحمّل فقط الكود المطلوب للصفحة الحالية.

✅ ميزة Code Splitting
مع Next.js، لا حاجة لإعداد Code Splitting يدوياً. كل صفحة في مجلد app تتحول إلى chunk منفصل يُحمّل عند الحاجة.
Dynamic Import jsx 
// استيراد ديناميكي لمكون ثقيل
import dynamic from 'next/dynamic';

const HeavyChart = dynamic(() => import('@/components/HeavyChart'), {
  loading: () => <p>جاري التحميل...</p>,
  ssr: false
});

// استيراد ديناميكي بمكونات مشروطة
const AdminDashboard = dynamic(
  () => import('@/components/AdminDashboard'),
  { ssr: false }
);

// استخدام loading state
const ProductModal = dynamic(
  () => import('@/components/ProductModal'),
  {
    loading: () => <div className="modal-loading"><Spinner /></div>,
    ssr: false
  }
);

⚡ أداء Server Components

Server Components في Next.js بتشتغل على السيرفر فقط، مما يحسن الأداء بشكل كبير:

1. تقليل حجم JavaScript المرسل للعميل

Server Component (الافتراضي) jsx 
// هذا Server Component - لا يُرسل JavaScript للعميل
import { getProducts } from '@/services/products';

export default async function ProductList() {
  // يتم تنفيذ هذا الكود على السيرفر فقط
  const products = await getProducts();
  
  return (
    <div className="grid grid-cols-1 md:grid-cols-3 gap-6">
      {products.map(product => (
        <ProductCard key={product.id} product={product} />
      ))}
    </div>
  );
}

2. Client Component عند الحاجة فقط

Client Component jsx 
"use client";

import { useState } from 'react';

export default function AddToCartButton({ productId }) {
  const [loading, setLoading] = useState(false);
  
  const handleAddToCart = async () => {
    setLoading(true);
    // إضافة للسلة
    await addToCart(productId);
    setLoading(false);
  };
  
  return (
    <button
      onClick={handleAddToCart}
      disabled={loading}
      className="btn-primary"
    >
      {loading ? 'جاري الإضافة...' : 'أضف للسلة'}
    </button>
  );
}

📊 قياس الأداء

استخدم أدوات مثل Lighthouse و Web Vitals لقياس أداء موقعك:

المقياس الهدف الشرح
LCP < 2.5 ثانية Largest Contentful Paint - وقت تحميل أكبر عنصر
FID < 100 مللي ثانية First Input Delay - وقت استجابة أول تفاعل
CLS < 0.1 Cumulative Layout Shift - تغيرات التخطيط
TTFB < 800 مللي ثانية Time to First Byte - وقت استجابة السيرفر

🛠️ أفضل الممارسات

⚠️ تجنب هذه الأخطاء
  • لا تستخدم "use client" إلا عند الضرورة (عند الحاجة للـ hooks)
  • لا تُصدّر بيانات حساسة من Server Components
  • لا تستخدم useEffect لجلب البيانات في Server Components
  • تأكد من أن الصور بحجم مناسب قبل الرفع

1. تحسين قاعدة البيانات

services/products.js javascript 
import { cache } from 'react';

// استخدام cache لتجنب الطلبات المتكررة
export const getProducts = cache(async (filters = {}) => {
  const { data, error } = await supabase
    .from('products')
    .select('*')
    .order('created_at', { ascending: false });
  
  if (error) throw error;
  return data;
});

// استخدام Revalidation
export const revalidate = 3600; // تحديث كل ساعة

2. تحسين التخزين المؤقت

app/api/products/route.js javascript 
import { NextResponse } from 'next/server';

export async function GET(request) {
  const products = await getProducts();
  
  return NextResponse.json(products, {
    headers: {
      'Cache-Control': 'public, s-maxage=3600, stale-while-revalidate=86400'
    }
  });
}
