💡 ليه Tailwind؟
Tailwind CSS هي أداة بتغير طريقة التصميم. بدل ما تكتب CSS كتير في ملفات منفصلة، بتكتب الـ classes مباشرة في الـ HTML. ده بيوفر عليك وقت كبير وبيخلي التصميم أسرع وأسهل.

🎨 إيه هو Tailwind CSS؟

Tailwind CSS هو إ framework CSS بيشتغل على مبدأ الـ Utility-first. يعني بدل ما تكتب CSS كتير، بتستخدم class names جاهزة في الـ HTML. كل class بتمثل property معينة في الـ CSS.

مثال مقارن:

CSS التقليدي Tailwind CSS
.button { padding: 10px 20px; background: blue; color: white; } class="px-5 py-2.5 bg-blue-500 text-white"

لاحظ الفرق! في CSS التقليدي، كنت بتكتب ملف CSS منفصل وبتربطه بالـ HTML. في Tailwind، بتكتب الـ classes مباشرة في الـ HTML. ده بيوفر عليك فتح ملفات كتير وتعديل أكتر.

مميزات Tailwind CSS:

  • سريع في التطوير - محتاج أقل كتابة CSS
  • Consistent Design - تصميم متناسق لأنك بتستخدم قيم ثابتة
  • Responsive سهل - مع prefixes زي md:, lg:
  • Mobile First - تصميم يبدأ من الموبايل
  • Dark Mode - وضع الليلي بسهولة
  • Hot Reload - التغييرات بتظهر مباشرة
  • Community - مجتمع كبير و Documentation ممتازة

⚙️ التكوين مع Next.js

Next.js 14+ بيدعم Tailwind CSS 4 بشكل أصلي. لو عملت create-next-app مع Tailwind، فهو متكون تلقائياً. يعني مش محتاج تعمل أي حاجة إضافية!

الملفات المهمة:

📁 market/
├── 📄 postcss.config.mjs # إعدادات PostCSS
├── 📄 tailwind.config.js # إعدادات Tailwind
└── 📁 app/
└── 📄 globals.css # الأنماط العامة مع Tailwind
postcss.config.mjs javascript 
/** @type {import('postcss-load-config').Config} */
const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};

export default config;

الملف ده بيوصل بين PostCSS و Tailwind CSS. PostCSS هو أداة بتحول CSS عادي لـ CSS مع Tailwind. متقلقش من التفاصيل التقنية دي - المهم إنك تعرف إن الملف ده لازم يكون موجود.

app/globals.css css 
@import "tailwindcss";

:root {
  --background: #ffffff;
  --foreground: #171717;
}

@media (prefers-color-scheme: dark) {
  :root {
    --background: #0a0a0a;
    --foreground: #ededed;
  }
}

body {
  color: var(--foreground);
  background: var(--background);
  font-family: Arial, Helvetica, sans-serif;
}

الملف ده فيه أمر @import "tailwindcss" اللي بيحمّل كل classes بتاعت Tailwind. كمان فيه بعض الأنماط الأساسية للمشروع.

🎯 استخدام Utility Classes

خلّينا نشوف أمثلة عملية على الـ classes اللي ممكن نستخدمها. Tailwind فيها مئات الـ classes، بس هنبدأ بالأساسيات.

1. التخطيط (Layout)

مثال على التخطيط html 
<!-- Flexbox -->
<div class="flex items-center justify-between">
  <span>العنصر 1</span>
  <span>العنصر 2</span>
</div>

<!-- Grid -->
<div class="grid grid-cols-3 gap-4">
  <div>عمود 1</div>
  <div>عمود 2</div>
  <div>عمود 3</div>
</div>

<!-- Padding و Margin -->
<div class="p-4 m-2">محتوى</div>
<div class="px-8 py-4">محتوى</div>

شرح كل class:

  • flex - بيعمل العنصر flexible container
  • items-center - بيحط العناصر في النص عمودياً
  • justify-between - بيبعد العناصر عن بعض
  • grid - بيعمل Grid layout
  • grid-cols-3 - 3 أعمدة
  • gap-4 - مسافة بين العناصر
  • p-4 - padding من كل الجهات (1rem)
  • px-8 - padding أفقي (2rem)
  • py-4 - padding عمودي (1rem)

2. الألوان والخلفيات

الألوان html 
<!-- ألوان النص -->
<p class="text-gray-900">نص أسود</p>
<p class="text-blue-600">نص أزرق</p>
<p class="text-white">نص أبيض</p>

<!-- ألوان الخلفية -->
<div class="bg-white">خلفية بيضاء</div>
<div class="bg-blue-500">خلفية زرقاء</div>
<div class="bg-gradient-to-r from-blue-500 to-purple-500">
  خلفية متدرجة
</div>

في Tailwind، الألوان لها مستويات من 50 (فاتح) لـ 900 (غامق). مثلاً blue-500 هو اللون الأزرق الأساسي، و blue-100 أفتح منه، و blue-900 أغمق منه.

3. الخطوط والنصوص

النصوص html 
<h1 class="text-4xl font-bold">عنوان رئيسي</h1>
<p class="text-lg text-gray-600">نص عادي</p>
<p class="text-sm font-medium">نص صغير</p>

<!-- محاذاة النص -->
<p class="text-center">نص في النص</p>
<p class="text-right">نص لليمين</p>
  • text-4xl - حجم الخط كبير جداً (2.25rem)
  • font-bold - خط سميك
  • text-lg - حجم خط كبير (1.125rem)
  • text-gray-600 - لون الرمادي المتوسط
  • text-center - النص في النص
  • text-right - النص لليمين (مهم للمواقع العربية)

4. الحدود والظلال

الحدود والظلال html 
<!-- حدود -->
<div class="border border-gray-200 rounded-lg">
  محتوى بحد
</div>

<!-- ظلال -->
<div class="shadow-lg">
  محتوى بظل
</div>

<!-- تأثيرات Hover -->
<button class="bg-blue-500 hover:bg-blue-700 transition-colors">
  زر مع تأثير
</button>
  • border - بيحط حد حول العنصر
  • rounded-lg - بيعمل حروف دائرية كبيرة
  • shadow-lg - ظل كبير
  • hover:bg-blue-700 - لون الخلفية يتغير لما تمرر الماوس
  • transition-colors - تحول سلس بين الألوان

🎨 تخصيص الثيم (Custom Theme)

ممكن نخصّص ألوان المشروع و Sizes في ملف tailwind.config.js. ده مهم عشان تحافظ على تناسق التصميم.

tailwind.config.js javascript 
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./app/**/*.{js,ts,jsx,tsx,mdx}",
    "./components/**/*.{js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {
      colors: {
        primary: {
          50: '#eef2ff',
          100: '#e0e7ff',
          200: '#c7d2fe',
          300: '#a5b4fc',
          400: '#818cf8',
          500: '#6366f1',
          600: '#4f46e5',
          700: '#4338ca',
          800: '#3730a3',
          900: '#312e81',
        },
        secondary: {
          50: '#faf5ff',
          100: '#f3e8ff',
          200: '#e9d5ff',
          300: '#d8b4fe',
          400: '#c084fc',
          500: '#a855f7',
          600: '#9333ea',
          700: '#7e22ce',
          800: '#6b21a8',
          900: '#581c87',
        },
      },
      fontFamily: {
        sans: ['Inter', 'Cairo', 'sans-serif'],
      },
    },
  },
  plugins: [],
};

استخدام الألوان المخصصة:

استخدام الألوان المخصصة html 
<!-- بعد التخصيص -->
<div class="bg-primary-500 text-white">
  خلفية باللون الأساسي
</div>

<div class="bg-secondary-100 border border-secondary-300">
  خلفية بلون ثانوي فاتح
</div>

لما تخصّص الألوان، بتقدر تستخدمها في كل مكان في المشروع. مثلاً لو لون المتجر الأساسي بنفسجي، بتخصصه مرة واحدة وبتستخدمه في كل مكان.

📱 التصميم المتجاوب (Responsive Design)

Tailwind CSS بيستخدم Mobile First approach. يعني بنبدأ بالتصميم للموبايل ونضيف الـ breakpoints للشاشات الأكبر. ده أفضل طريقة للتصميم لأن أكتر الناس بتفتح المواقع من الموبايل.

الـ Breakpoints:

Prefix الحجم الجهاز
sm: 640px موبايل كبير
md: 768px تابلت
lg: 1024px لابتوب
xl: 1280px شاشة كبيرة
2xl: 1536px شاشة ضخمة

مثال: تصميم متجاوب

تصميم متجاوب html 
<!-- شبكة متجاوبة -->
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <div class="bg-white p-4 rounded-lg shadow">
    منتج 1
  </div>
  <div class="bg-white p-4 rounded-lg shadow">
    منتج 2
  </div>
  <div class="bg-white p-4 rounded-lg shadow">
    منتج 3
  </div>
</div>

<!-- تخطيط متجاوب -->
<div class="flex flex-col md:flex-row items-center justify-between">
  <h1 class="text-2xl md:text-4xl font-bold mb-4 md:mb-0">
    عنوان رئيسي
  </h1>
  <button class="w-full md:w-auto bg-blue-500 text-white px-6 py-3 rounded">
    زر
  </button>
</div>

شرح:

  • grid-cols-1 - عمود واحد في الموبايل
  • md:grid-cols-2 - عمودين في التابلت
  • lg:grid-cols-3 - 3 أعمدة في الابتوب
  • flex-col - ترتيب عمودي في الموبايل
  • md:flex-row - ترتيب أفقي في التابلت
  • w-full - عرض 100% في الموبايل
  • md:w-auto - عرض تلقائي في الشاشات الكبيرة
💡 نصيحة
ابدأ دائماً بالتصميم للموبايل (بدون prefix) ثم أضف الـ prefixes للشاشات الأكبر. ده اسمه Mobile First approach. في الأغلب، المستخدمين هيفتحوا الموقع من الموبايل أولاً.

🌙 Dark Mode

Tailwind CSS بيدعم Dark Mode بسهولة. ممكن نستخدم الـ dark: prefix عشان نحدد الأنماط اللي تتطبق في الوضع الليلي.

Dark Mode html 
<!-- تفعيل Dark Mode في globals.css -->
<html class="dark">

<!-- استخدام dark: prefix -->
<div class="bg-white dark:bg-gray-900 text-black dark:text-white">
  محتوى يدعم الوضع الليلي
</div>

<!-- في JavaScript -->
<div className={
  p-4 
}>
  محتوى ديناميكي
</div>

يعني لما المستخدم يفعّل Dark Mode في المتصفح، العناصر اللي فيها dark: هتتغير تلقائياً.
