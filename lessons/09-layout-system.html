1️⃣ مقدمة عن Layout System

💡 إيه هو التخطيط (Layout)؟

مرحب بيك في الدرس التاسع! النهارده هنتكلم عن موضوع أساسي جداً في أي موقع ويب: التخطيط (Layout). فكر كده كده: لو هتبني بيت، محتاج هيكلاً أساسياً يحدد هيكل الغرف والطوابق. التخطيط في الويب زيه بالظبط — بيحدد هيكل الصفحة كلها.

في أي متجر إلكتروني، كل الصفحات عندها عناصر مشتركة: الشريط العلوي (Header) فيه الشعار والتنقل وسلة المشتريات، والشريط السفلي (Footer) فيه الروابط ومعلومات التواصل. بدل ما نكرر الكود ده في كل صفحة، بنعمل Layout واحد نستخدمه في كل الصفحات.

في Next.js App Router، الموضوع سهل جداً. بنعمل ملف layout في كل مجلد، والـ Next.js بيطبقه تلقائياً على كل الصفحات جوا المجلد ده. حلو أوي!

ليه بنستخدم Layout في Next.js؟

قبل ما نبدأ، خلينا نفهم ليه الـ Layout مهم أوي:

  • تقليل تكرار الكود: بدل ما نكتب نفس الـ Header والـ Footer في كل صفحة، بنكتبهم مرة واحدة في الـ Layout
  • استمرارية التجربة: المستخدم بيفضل يشوف نفس الـ Header والـ Footer وهو بينقل بين الصفحات
  • أداء أفضل: لما المستخدم بينقل بين الصفحات، الـ Next.js بيحافظ على الـ Layout وبيحفظ المحتوى اللي مش محتاج يتغير
  • تنظيم أفضل: الكود بيكون منظم ومرتب وسهل الصيانة
  • سهولة التعديل: لو عايز تغير الـ Header، بتغيره مرة واحدة وبيتأثر على كل الصفحات

في المتجر بتاعنا، بنحتاج تخطيطين مختلفين:

  • ShopLayout: تخطيط للمتجر العام — فيه Header وFooter عادي
  • AdminLayout: تخطيط للوحة التحكم — فيه شريط جانبي وشريط علوي مختلف

2️⃣ شرح layout.js الأساسي

💡 إيه هو Root Layout؟

الـ Root Layout ده أهم ملف layout في مشروعك. هو الملف اللي في مجلد app الرئيسي، وبي contient كل الصفحات في مشروعك. بدون الـ Root Layout، مشروعك مش هيشتغل خالص!

الـ Root Layout بيعمل حاجات مهمة جداً:

  • بي contient الـ HTML structure الأساسية (html, head, body)
  • بي contient الـ Providers (زي Auth, Cart, Theme)
  • بي contient الـ Global CSS
  • بي contient الـ Metadata الأساسية للموقع
app/layout.js — الـ Root Layout jsx 
import './globals.css';

export const metadata = {
  title: 'Market - متجرك الإلكتروني',
  description: 'تسوّق أفضل المنتجات بأسعار منافسة',
};

export default function RootLayout({ children }) {
  return (
    <html lang="ar" dir="rtl">
      <body>
        {children}
      </body>
    </html>
  );
}

خلينا نشرح الكود خطوة بخطوة:

السطر الأول: import './globals.css';

بنستورد الـ CSS الرئيسي للموقع. الـ globals.css ده فيه الأنماط العامة اللي بتطبق على كل الصفحات. زيه زي ما تلبس هدومك الأساسية قبل ما تلبس الهدوم الخارجية.

السطر التاني: export const metadata

ده объект فيه معلومات عن الموقع بتاعك. الـ metadata دي مهمة جداً لمحركات البحث (SEO). لما حد يبحث عن موقعك في Google، المعلومات دي هي اللي بتظهر في نتايج البحث.

الدالة RootLayout

دي الدالة الرئيسية اللي بتعمل الـ Layout. بتأخذ children كمعامل — وده المحتوى اللي جوا الـ Layout ده. يعني لو عندك صفحة الرئيسية، الـ children هتكون محتوى الصفحة الرئيسية.

الـ HTML Structure

بنحدد lang="ar" عشان نقول للمتصفح إن اللغة عربية، وdir="rtl" عشان نحدد إن الاتجاه من اليمين لليسار.

⚠️ ملاحظات مهمة للـ Root Layout
  • الـ Root Layout لازم يكون في app/layout.js — مش في مجلد تاني
  • لازم يكون فيه <html> و <body> tags
  • الـ children لازم يكون جوا الـ <body>
  • مش هتقدر تستخدم "use client" في الـ Root Layout لأن الـ metadata مش بتشتغل معاه

3️⃣ شرح children prop

💡 إيه هو الـ children prop؟

الـ children prop دي واحدة من أهم المفاهيم في React. فكر فيها كده: لو عندك صندوق (الـ Layout)، والـ children هو المحتوى اللي جوا الصندوق ده.

يعني لما بتكتب <Layout><Page /></Layout>، الـ <Page /> ده هو الـ children بتاع الـ Layout.

خلينا نشوف مثال بسيط:

مثال بسيط على children jsx 
// ده الـ Layout
function Layout({ children }) {
  return (
    <div>
      <header>الشريط العلوي</header>
      <main>{children}</main>
      <footer>الشريط السفلي</footer>
    </div>
  );
}

// ده المستخدم للـ Layout
function HomePage() {
  return (
    <Layout>
      <h1>مرحباً بالعالم</h1>
      <p>ده محتوى الصفحة الرئيسية</p>
    </Layout>
  );
}

// النتيجة النهائية هتكون:
// <div>
//   <header>الشريط العلوي</header>
//   <main>
//     <h1>مرحباً بالعالم</h1>
//     <p>ده محتوى الصفحة الرئيسية</p>
//   </main>
//   <footer>الشريط السفلي</footer>
// </div>

في Next.js App Router، الـ children بتتحول تلقائياً. يعني لما بتكتب ملف app/page.js (صفحة الرئيسية)، المحتوى بتاعها بيتحول لـ children في الـ app/layout.js.

مثال عملي في مشروعنا

app/(shop)/layout.js jsx 
import ShopLayout from '@/components/layout/ShopLayout';

export default function Layout({ children }) {
  // الـ children هنا هو محتوى أي صفحة جوا مجلد (shop)
  // يعني لو عندك app/(shop)/products/page.js
  // الـ children هتكون محتوى صفحة المنتجات
  
  return (
    <ShopLayout>
      {children}
    </ShopLayout>
  );
}
💡 نصيحة مهمة

فكّر في الـ children كـ "ثغرة" في الـ Layout. الـ Layout بيحةط الهيكل العام (Header, Footer)، والـ children بتملا المحتوى الخاص بكل صفحة. الطريقة دي بتخلي الـ Layout مرن وقابل لإعادة الاستخدام.

4️⃣ إنشاء ShopLayout

الآن هنبني التخطيط الرئيسي للمتجر. الـ ShopLayout ده هيكون في كل صفحات المتجر — الصفحة الرئيسية، صفحة المنتجات، صفحة التفاصيل، السلة، وغيرها.

هيكل ملفات التخطيط

📁 components/layout/
├── 📄 Header.jsx # الشريط العلوي
├── 📄 Footer.jsx # الشريط السفلي
├── 📄 ShopLayout.jsx # تخطيط المتجر
├── 📄 AdminLayout.jsx # تخطيط Admin
└── 📄 MobileMenu.jsx # قائمة الموبايل
💡 ليه المكونات في مجلد layout منفصل؟

في تطوير الواجهات، في فرق بين مكونات الـ UI الأساسية (زراير، بطاقات) ومكونات التخطيط (Header, Footer). مكونات التخطيط بتتحكم الصفحة كلها، فمحتاجة مجلد منفصل عشان نحافظ على التنظيم. كمان، في أوقات كتير بتفرق تخطيط المتجر عن تخطيط لوحة التحكم.

components/layout/ShopLayout.jsx jsx 
import Header from './Header';
import Footer from './Footer';

export default function ShopLayout({ children }) {
  return (
    <div className="min-h-screen flex flex-col bg-gray-950">
      {/* Header */}
      <Header />

      {/* Main Content */}
      <main className="flex-1 pt-16">
        {children}
      </main>

      {/* Footer */}
      <Footer />
    </div>
  );
}

شرح الكود بالتفصيل

خلينا نشرح كل جزء:

min-h-screen flex flex-col

دي classes مهمة أوي من Tailwind CSS:

  • min-h-screen: بيخلي الـ div على الأقل بارتفاع الشاشة كلها. ده مهم عشان الـ Footer يفضل في الآخر حتى لو الصفحة قصيرة
  • flex: بيعمل الـ div container مرن (flexbox). ده بيساعدنا نرتب العناصر أفقياً أو عمودياً
  • flex-col: بيرتب العناصر عمودياً (من فوق لتحت). يعني الـ Header فوق، والـ main في النص، والـ Footer تحت

flex-1 على الـ main

ده معناه إن الـ main ياخد كل المساحة المتاحة. لو الصفحة طويلة، الـ Footer هينزل لتحت. لو الصفحة قصيرة، الـ Footer هيفضل في الآخر. الطريقة دي بتخلي الـ Footer دايماً في المكان الصح.

pt-16 على الـ main

ده من أهم الأسطر! الـ Header بتاعنا ثابت (fixed) في أعلى الشاشة، فمحتاجين مسافة فوق عشان المحتوى ما يختفيش تحته. بدون الـ padding ده، أول محتوى في الصفحة هيبان تحت الـ Header. الـ 16 هنا معناها 4rem (64px) — نفس ارتفاع الـ Header.

⚠️ خطأ شائع

كتير الناس بتنسى تضيف pt-16 وبتتطلع ليه أول محتوى مختفي! لو الـ Header بتاعك fixed أو sticky، لازم تضيف padding عشان المحتوى ما يختفيش تحته.

5️⃣ إنشاء AdminLayout

الـ AdminLayout مختلف تماماً عن ShopLayout. فيه شريط جانبي (Sidebar) فيه الروابط الإدارية، وشريط علوي (TopBar) فيه معلومات المستخدم. الـ Layout ده بيستخدم في كل صفحات لوحة التحكم.

ليه AdminLayout مختلف؟

  • الـ ShopLayout فيه Header وFooter عادي — ده للمستخدم العادي
  • الـ AdminLayout فيه Sidebar وTopBar — ده للمدير
  • الـ AdminLayout محتاج مصادقة (Auth) عشان نتأكد إن المستخدم مدير
  • الـ AdminLayout فيه روابط خاصة بالإدارة زي إدارة المنتجات والطلبات والمستخدمين
components/layout/AdminLayout.jsx jsx 
"use client";

import { useState } from 'react';
import Link from 'next/link';
import { usePathname } from 'next/navigation';
import { useAuth, useUser, SignOutButton } from '@clerk/nextjs';
import {
  LayoutDashboard,
  Package,
  ShoppingCart,
  Users,
  Settings,
  LogOut,
  ChevronLeft,
  Menu,
  Store,
} from 'lucide-react';
import { cn } from '@/lib/cn';
import AdminSidebar from './AdminSidebar';

export default function AdminLayout({ children }) {
  const [isSidebarOpen, setIsSidebarOpen] = useState(true);
  const pathname = usePathname();
  const { userId } = useAuth();
  const { user } = useUser();

  return (
    <div className="min-h-screen bg-gray-950 flex">
      {/* Sidebar */}
      <AdminSidebar
        isOpen={isSidebarOpen}
        onToggle={() => setIsSidebarOpen(!isSidebarOpen)}
        pathname={pathname}
        user={user}
      />

      {/* Main Content */}
      <div className={cn(
        "flex-1 transition-all duration-300",
        isSidebarOpen ? "mr-64" : "mr-20"
      )}>
        {/* Top Bar */}
        <header className="h-16 bg-gray-900/50 backdrop-blur-xl border-b border-white/10 flex items-center justify-between px-6">
          <button
            onClick={() => setIsSidebarOpen(!isSidebarOpen)}
            className="md:hidden p-2 rounded-lg text-gray-400 hover:text-white hover:bg-white/10 transition-colors"
          >
            <Menu className="w-5 h-5" />
          </button>
          
          <div className="flex items-center gap-4">
            <span className="text-gray-400 text-sm">
              مرحباً، {user?.firstName}
            </span>
          </div>
        </header>

        {/* Page Content */}
        <main className="p-6">
          {children}
        </main>
      </div>
    </div>
  );
}

شرح مكونات AdminLayout

  • isSidebarOpen: State بيحكم هل الشريط الجانبي مفتوح ولا مطوي. لما يكون مفتوح، العرض 64 (w-64). لما يكون مطوي، العرض 20 (w-20)
  • mr-64 / mr-20: المحتوى الرئيسي بيتحرك يمين حسب حالة الشريط الجانبي. لأننا شغالين RTL، بنستخدم margin-right بدل margin-left
  • "use client": محتاجينه عشان بنستخدم useState و React hooks
  • useAuth() و useUser(): من Clerk للتحقق من المستخدم

6️⃣ إنشاء AdminSidebar

الـ AdminSidebar ده الشريط الجانبي اللي فيه الروابط الإدارية. منفصل عن الـ AdminLayout عشان يبقى سهل التعديل وإعادة الاستخدام.

components/layout/AdminSidebar.jsx jsx 
"use client";

import Link from 'next/link';
import { cn } from '@/lib/cn';
import {
  LayoutDashboard,
  Package,
  ShoppingCart,
  Users,
  Settings,
  LogOut,
  ChevronLeft,
  Store,
} from 'lucide-react';
import { SignOutButton } from '@clerk/nextjs';

const sidebarLinks = [
  { href: '/admin', label: 'Dashboard', icon: LayoutDashboard },
  { href: '/admin/products', label: 'المنتجات', icon: Package },
  { href: '/admin/orders', label: 'الطلبات', icon: ShoppingCart },
  { href: '/admin/users', label: 'المستخدمين', icon: Users },
  { href: '/admin/settings', label: 'الإعدادات', icon: Settings },
];

export default function AdminSidebar({ isOpen, onToggle, pathname, user }) {
  return (
    <aside
      className={cn(
        "fixed right-0 top-0 h-full bg-gray-900 border-l border-white/10 transition-all duration-300 z-40",
        isOpen ? "w-64" : "w-20"
      )}
    >
      {/* Logo */}
      <div className="h-16 flex items-center justify-between px-4 border-b border-white/10">
        <Link href="/admin" className="flex items-center gap-2">
          <div className="w-10 h-10 rounded-xl bg-gradient-to-br from-indigo-500 to-purple-500 flex items-center justify-center">
            <Store className="w-5 h-5 text-white" />
          </div>
          {isOpen && (
            <span className="text-white font-bold">Admin</span>
          )}
        </Link>
        <button
          onClick={onToggle}
          className="p-2 rounded-lg text-gray-400 hover:text-white hover:bg-white/10 transition-colors"
        >
          <ChevronLeft className={cn(
            "w-5 h-5 transition-transform",
            !isOpen && "rotate-180"
          )} />
        </button>
      </div>

      {/* Navigation */}
      <nav className="p-4 space-y-2">
        {sidebarLinks.map((link) => {
          const isActive = pathname === link.href;
          return (
            <Link
              key={link.href}
              href={link.href}
              className={cn(
                "flex items-center gap-3 px-4 py-3 rounded-xl transition-all duration-200",
                isActive
                  ? "bg-indigo-500/20 text-indigo-300"
                  : "text-gray-400 hover:text-white hover:bg-white/5"
              )}
            >
              <link.icon className="w-5 h-5 flex-shrink-0" />
              {isOpen && (
                <span className="font-medium">{link.label}</span>
              )}
            </Link>
          );
        })}
      </nav>

      {/* User Section */}
      <div className="absolute bottom-0 left-0 right-0 p-4 border-t border-white/10">
        <div className={cn(
          "flex items-center gap-3",
          !isOpen && "justify-center"
        )}>
          <div className="w-10 h-10 rounded-full bg-gradient-to-br from-indigo-500 to-purple-500 flex items-center justify-center flex-shrink-0">
            <span className="text-white text-sm font-medium">
              {user?.firstName?.[0] || "A"}
            </span>
          </div>
          {isOpen && (
            <div className="flex-1 min-w-0">
              <p className="text-white text-sm font-medium truncate">
                {user?.firstName} {user?.lastName}
              </p>
              <p className="text-gray-400 text-xs truncate">
                {user?.emailAddresses?.[0]?.emailAddress}
              </p>
            </div>
          )}
        </div>
        
        {isOpen && (
          <div className="mt-4 flex gap-2">
            <Link
              href="/"
              className="flex-1 flex items-center justify-center gap-2 px-4 py-2 rounded-lg bg-white/5 text-gray-400 hover:text-white hover:bg-white/10 transition-colors text-sm"
            >
              <Store className="w-4 h-4" />
              <span>المتجر</span>
            </Link>
            <SignOutButton>
              <button className="flex items-center justify-center gap-2 px-4 py-2 rounded-lg bg-red-500/10 text-red-400 hover:bg-red-500/20 transition-colors text-sm">
                <LogOut className="w-4 h-4" />
              </button>
            </SignOutButton>
          </div>
        )}
      </div>
    </aside>
  );
}

شرح AdminSidebar بالتفصيل

  • fixed right-0: الشريط الجانبي ثابت في مكانه، مش بيتغير مع التمرير. لأننا RTL، بنحطه على اليمين
  • z-40: بيخلي الشريط يظهر فوق المحتوى التاني
  • rotate-180: الأيقونة بتدور 180 درجة لما الشريط يتقلص. ده بيدي إحساس بالتفاعلية
  • flex-shrink-0: الأيقونات ما بتتقلصش لو الشريط اتضغط
  • truncate: لو النص طويل، بيقطعه وبيحط "..." في الآخر
💡 ليه فصلنا الـ Sidebar في ملف منفصل؟

فصلنا الـ AdminSidebar في ملف منفصل عشان:

  • يسهل التعديل عليه من غير ما نلمس الـ AdminLayout
  • ممكن نعيد استخدامه في مكان تاني
  • الكود يكون أنظف وأسهل في القراءة
  • لو عايز نغير الروابط، بنغيرها في مكان واحد بس

7️⃣ شرح Route Groups

💡 إيه هي الـ Route Groups؟

الـ Route Groups دي ميزة في Next.js App Router بتخليك تgroup الصفحات في مجلدات من غير ما تغير المسار الفعلي. يعني لما تكتب (shop) في اسم المجلد، الأقواس معناها إن المجلد ده مش جزء من المسار.

مثال عملي

📁 app/
├── 📁 (shop)/
│ ├── 📄 layout.js # ShopLayout
│ ├── 📄 page.js # الصفحة الرئيسية
│ ├── 📁 products/
│ │ ├── 📄 page.js # /products
│ │ └── 📁 [id]/
│ │ └── 📄 page.js # /products/123
├── 📁 (admin)/
│ ├── 📄 layout.js # AdminLayout
│ ├── 📄 page.js # /admin
│ └── 📁 products/
│ └── 📄 page.js # /admin/products
└── 📄 layout.js # Root Layout

لاحظ الأرقام دي:

  • صفحة المنتجات: /products — مش /shop/products
  • صفحة تفاصيل المنتج: /products/123 — مش /shop/products/123
  • لوحة التحكم: /admin — مش /admin/admin

يعني الأقواس ( ) بتقول للـ Next.js: "المجلد ده بس للتنظيم الداخلي، مش في المسار الفعلي".

ليه بنستخدم Route Groups؟

  • تنظيم أفضل: ممكن تgroup الصفحات حسب الوظيفة
  • تخطيطات مختلفة: كل مجموعة ممكن يكون عندها layout مختلف
  • metadata مختلفة: كل مجموعة ممكن يكون عندها metadata مختلفة
app/(shop)/layout.js jsx 
import ShopLayout from '@/components/layout/ShopLayout';

export const metadata = {

8️⃣ شرح Dynamic Routes

💡 إيه هي الـ Dynamic Routes؟

الـ Dynamic Routes دي بتخليك تعمل صفحات بتتغير حسب الرابط. يعني مثلاً صفحة تفاصيل المنتج — كل منتج عنده ID مختلف، فمحتاجين صفحة واحدة تشتغل مع كل الـ IDs.

أنواع Dynamic Routes

1. [id] — مسار بمتغير واحد

بيستخدم لما يكون عندك متغير واحد في الرابط:

app/products/[id]/page.js jsx 
export default function ProductPage({ params }) {
  // params.id هتكون القيمة من الرابط
  // يعني لو الرابط /products/123، params.id هتكون "123"
  
  return (
    <div>
      <h1>تفاصيل المنتج رقم {params.id}</h1>
    </div>
  );
}

2. [...slug] — مسار بعدد متغير من الأجزاء

بيستخدم لما يكون عندك عدد متغير من أجزاء الرابط:

app/[...slug]/page.js jsx 
export default function Page({ params }) {
  // params.slug هتكون array فيها كل أجزاء الرابط
  // يعني لو الرابط /categories/electronics/phones
  // params.slug هتكون ["categories", "electronics", "phones"]
  
  return (
    <div>
      <h1>المسار: {params.slug.join(' / ')}</h1>
    </div>
  );
}

3. [[...slug]] — مسار اختياري

زي الـ [...slug] بس ممكن يكون فاضي:

app/[[...slug]]/page.js jsx 
export default function Page({ params }) {
  // params.slug ممكن تكون undefined لو الرابط فاضي
  // يعني الرابط / كمان بيشتغل
  
  return (
    <div>
      <h1>المسار: {params.slug?.join(' / ') || 'الرئيسية'}</h1>
    </div>
  );
}

مثال عملي في مشروعنا

📁 app/(shop)/products/
├── 📄 page.js # /products
└── 📁 [id]/
└── 📄 page.js # /products/123
💡 نصيحة

لما تستخدم Dynamic Routes، تأكد إنك بتعمل validation للـ params. يعني تتأكد إن الـ ID فعلاً رقم مش نص عشوائي.

9️⃣ شرح Metadata و SEO

💡 إيه هو الـ Metadata؟

الـ Metadata دي معلومات عن الصفحة بتاعتك بتتغير في الـ HTML، مش بتظهر للمستخدم مباشرة، لكن محركات البحث (زي Google) والسوشيال ميديا بتستخدمها.

الـ Metadata بتتحكم في حاجات زي:

  • عنوان الصفحة اللي بيظهر في تبويب المتصفح
  • الوصف اللي بيظهر في نتايج البحث
  • الصورة اللي بتظهر لما حد يشاركن الرابط
  • كلمات المفتاحية اللي بتساعد الموقع يظهر في البحث

⚠️ ملاحظات مهمة على Metadata

1️⃣1️⃣ إنشاء Header Component

الHeader هو أهم عنصر في أي موقع ويب. هو أول حاجة المستخدم بيشوفها لما يفتح الصفحة. لازم يكون واضح وجميل وسهل الاستخدام.

أدوات الـ Header

  • الشعار (Logo): رابط للصفحة الرئيسية
  • التنقل (Navigation): روابط للصفحات المختلفة
  • البحث (Search): زرار للبحث عن المنتجات
  • قائمة الأمنيات (Wishlist): المنتجات اللي المستخدم حبها
  • سلة المشتريات (Cart): المنتجات اللي المستخدم عايش يشتريها
  • حساب المستخدم (User Menu): تسجيل الدخول أو الحساب
  • قائمة الموبايل (Mobile Menu): قائمة للشاشات الصغيرة
components/layout/Header.jsx jsx 
"use client";

import { useState, useEffect } from 'react';
import Link from 'next/link';
import { usePathname } from 'next/navigation';
import { useAuth, useUser } from '@clerk/nextjs';
import { ShoppingCart, User, Menu, X, Search, Heart } from 'lucide-react';
import { cn } from '@/lib/cn';
import { useCart } from '@/context/CartContext';
import MobileMenu from './MobileMenu';

const navLinks = [
  { href: '/', label: 'الرئيسية' },
  { href: '/products', label: 'المنتجات' },
  { href: '/categories', label: 'التصنيفات' },
  { href: '/offers', label: 'العروض' },
  { href: '/about', label: 'من نحن' },
];

export default function Header() {
  const [isScrolled, setIsScrolled] = useState(false);
  const [isMobileMenuOpen, setIsMobileMenuOpen] = useState(false);
  const pathname = usePathname();
  const { isSignedIn } = useAuth();
  const { user } = useUser();
  const { cart } = useCart();

  // تأثير التمرير
  useEffect(() => {
    const handleScroll = () => {
      setIsScrolled(window.scrollY > 20);
    };
    window.addEventListener('scroll', handleScroll);
    return () => window.removeEventListener('scroll', handleScroll);
  }, []);

  const cartItemsCount = cart?.items?.reduce((sum, item) => sum + item.quantity, 0) || 0;

  return (
    <header
      className={cn(
        "fixed top-0 left-0 right-0 z-50 transition-all duration-300",
        isScrolled
          ? "bg-gray-900/80 backdrop-blur-xl border-b border-white/10 shadow-xl"
          : "bg-transparent"
      )}
    >
      <div className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8">
        <div className="flex items-center justify-between h-16">
          
          {/* Logo */}
          <Link href="/" className="flex items-center gap-2">
            <div className="w-10 h-10 rounded-xl bg-gradient-to-br from-indigo-500 to-purple-500 flex items-center justify-center">
              <span className="text-white font-bold text-xl">M</span>
            </div>
            <span className="text-white font-bold text-xl hidden sm:block">Market</span>
          </Link>

          {/* Desktop Navigation */}
          <nav className="hidden md:flex items-center gap-1">
            {navLinks.map((link) => (
              <Link
                key={link.href}
                href={link.href}
                className={cn(
                  "px-4 py-2 rounded-lg text-sm font-medium transition-all duration-200",
                  pathname === link.href
                    ? "bg-white/10 text-white"
                    : "text-white/70 hover:text-white hover:bg-white/5"
                )}
              >
                {link.label}
              </Link>
            ))}
          </nav>

          {/* Actions */}
          <div className="flex items-center gap-2">
            {/* Search */}
            <button className="p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors">
              <Search className="w-5 h-5" />
            </button>

            {/* Wishlist */}
            <Link
              href="/wishlist"
              className="p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors"
            >
              <Heart className="w-5 h-5" />
            </Link>

            {/* Cart */}
            <Link
              href="/cart"
              className="relative p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors"
            >
              <ShoppingCart className="w-5 h-5" />
              {cartItemsCount > 0 && (
                <span className="absolute -top-1 -right-1 w-5 h-5 bg-indigo-500 rounded-full text-white text-xs flex items-center justify-center font-bold">
                  {cartItemsCount}
                </span>
              )}
            </Link>

            {/* User Menu */}
            {isSignedIn ? (
              <Link
                href="/account"
                className="flex items-center gap-2 p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors"
              >
                <div className="w-8 h-8 rounded-full bg-gradient-to-br from-indigo-500 to-purple-500 flex items-center justify-center">
                  <span className="text-white text-sm font-medium">
                    {user?.firstName?.[0] || "U"}
                  </span>
                </div>
              </Link>
            ) : (
              <Link
                href="/sign-in"
                className="px-4 py-2 rounded-lg bg-indigo-500 text-white text-sm font-medium hover:bg-indigo-600 transition-colors"
              >
                تسجيل الدخول
              </Link>
            )}

            {/* Mobile Menu Button */}
            <button
              onClick={() => setIsMobileMenuOpen(!isMobileMenuOpen)}
              className="md:hidden p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors"
            >
              {isMobileMenuOpen ? (
                <X className="w-5 h-5" />
              ) : (
                <Menu className="w-5 h-5" />
              )}
            </button>
          </div>
        </div>
      </div>

      {/* Mobile Menu */}
      <MobileMenu
        isOpen={isMobileMenuOpen}
        onClose={() => setIsMobileMenuOpen(false)}
        navLinks={navLinks}
        pathname={pathname}
      />
    </header>
  );
}

شرح المكونات المستخدمة

💡 شرح الـ Hooks المستخدمة
  • usePathname(): لمعرفة المسار الحالي وتحديد الرابط النشط
  • useAuth() و useUser(): للتحقق من حالة تسجيل الدخول
  • useCart(): لجلب عدد عناصر السلة
  • useEffect: للكشف عن التمرير وتغيير شكل Header

تأثير التمرير (Scroll Effect)

في السطور اللي بتستخدم useEffect، بنسمع لحدث التمرير (scroll event). لما المستخدم ينزل أكتر من 20 بكسل، بنغير قيمة isScrolled لـ true. وبما إن الـ className بتاع الـ Header بيعتمد على القيمة دي، بتتغير الشكل تلقائياً.

الروابط النشطة (Active Links)

بنستخدم usePathname() عشان نعرف الصفحة الحالية. وبعدين بنقارنها بكل رابط. لو متطابقة، بنضيف فئة "bg-white/10" عشان نبين إن الرابط ده هو النشط.

عداد السلة (Cart Badge)

بنحسب عدد العناصر في السلة باستخدام reduce. لو العدد أكبر من صفر، بنظهر Badge صغير فوق أيقونة السلة فيه العدد. لو صفر، مش بنظهر حاجة.

⚠️ أخطاء شائعة مع Header
  • الخطأ: استخدام position: fixed بدون padding-top للمحتوى — النتيجة: أول محتوى بيختفي تحت الـ Header
  • الخطأ: نسيان z-50 — النتيجة: عناصر تانية بتظهر فوق الـ Header
  • الخطأ: عدم التعامل مع window في Server Component — النتيجة: خطأ في Server Side Rendering
  • الحل: دايماً استخدم "use client" في المكونات اللي بتتفاعل مع المستخدم

النمط 3: Multi-level Layout

ده نمط متقدم فيه تخطيطات متعددة المستويات:

التخطيط متعدد المستويات jsx 
// Root Layout
function RootLayout({ children }) {
  return (
    <html>
      <body>
        <Providers>
          {children}
        </Providers>
      </body>
    </html>
  );
}

// Shop Layout
function ShopLayout({ children }) {
  return (
    <div>
      <Header />
      <main>{children}</main>
      <Footer />
    </div>
  );
}

// Product Layout
function ProductLayout({ children }) {
  return (
    <div>
      <ProductBreadcrumb />
      {children}
    </div>
  );
}

أفضل الممارسات

  • فصل المكونات: كل Layout في ملف منفصل
  • إعادة الاستخدام: استخدم نفس الـ Layout في أماكن كتير
  • المرونة: خلي الـ Layout يقبل children عشان يكون مرن
  • الأداء: استخدم "use client" بس لما تكون محتاج hooks
  • الوصولية: استخدم semantic HTML و aria-labels
  • المتجاوبة: خلي التخطيط يشتغل على كل الشاشات
⚠️ أخطاء يجب تجنبها
  • الخطأ 1: استخدام position: fixed للـ Header بدون padding للمحتوى — النتيجة: أول محتوى بيختفي
  • الخطأ 2: نسيان "use client" في المكونات اللي فيها useState — النتيجة: خطأ في الـ Server
  • الخطأ 3: عدم استخدام usePathname() لتحديد الرابط النشط — النتيجة: المستخدم مش ه يعرف هو فين
  • الخطأ 4: عدم التعامل مع الشاشات الصغيرة — النتيجة: الموقع مش هيشتغل على الموبايل
  • الخطأ 5: نسيان z-index — النتيجة: عناصر بتظهر فوق بعضها بالغلط

1️⃣4️⃣ تمرين عملي

📝 جرّب اللي اتعلمتة!

الآن بعد ما فهمت الـ Layout System، جرّب التمرين ده عشان تثبّت معلوماتك:

التمرين 1: بحث في Header

أضف شريط بحث في الـ Header يظهر لما المستخدم يضغط على أيقونة البحث. الشريط يظهر بتأثير slide down.

  • اعمل state جديد اسمه isSearchOpen
  • لما المستخدم يضغط على أيقونة البحث، اقلب القيمة
  • اعرض input field بتأثير slide down
  • خلي الـ input يكون فيه placeholder "ابحث عن منتجات..."

التمرين 2: بيانات ديناميكية في Footer

عدّل الـ Footer يعرض شعار وسائل التواصل بناءً على بيانات من API.

  • اعمل async function تجيب بيانات التواصل من API
  • استخدم الـ data في عرض الأيقونات
  • لو الـ API فشل، اعرض أيقونات افتراضية

التمرين 3: زرار رجوع للأعلى

أضف زرار "رجوع للأعلى" يظهر لما المستخدم ينزل الصفحة أكتر من 500px.

  • استخدم useEffect عشان تسمع لحدث التمرير
  • اعمل state اسمه showScrollTop
  • اعرض الزرار بتأثير fade in/out
  • لما تضغط عليه، رجّع المستخدم لأعلى الصفحة بـ smooth scroll

التمرين 4: Dark Mode في Admin

عدّل AdminLayout يضيف Dark Mode Toggle.

  • اعمل ThemeContext يتحكم في الوضع
  • أضف زرار في TopBar يقلب بين Light و Dark
  • احفظ الاختيار في localStorage
  • اعرض التغيير فوراً من غير ما المستخدم يعيد تحميل الصفحة

التمرين 5: Breadcrumb

أضف Breadcrumb في كل صفحة يبين المسار الحالي.

  • اعمل component اسمه Breadcrumb
  • استخدم usePathname() عشان تعرف المسار
  • اعرض الروابط بشكل صحيح (من اليمين لليسار)
  • خلي الصفحة الحالية مش رابط

التمرين 6: DashboardLayout جديد

أنشئ DashboardLayout جديد للصفحات الإدارية يختلف عن AdminLayout.

  • اعمل تخطيط جديد فيه TopBar بس (من غير Sidebar)
  • استخدمه في صفحات معينة زي الملف الشخصي
  • اربطه بـ Route Group جديد

1️⃣5️⃣ ملخص الدرس

النقاط الرئيسية

  • ✅ شرح Metadata وSEO
  • ✅ فهم إزاي تتحكم في العنوان والـ Head
  • ✅ إنشاء Header Component احترافي
  • ✅ إنشاء Footer Component كامل

  • Route Groups: تنظيم الصفحات من غير تغيير المسار
  • Dynamic Routes: صفحات بتتغير حسب الرابط
  • Metadata: معلومات عن الصفحة لمحركات البحث
  • "use client": مكون بيتفاعل مع المستخدم
