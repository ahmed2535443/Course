💡 إيه هو نظام التخطيط (Layout System)؟

مرحب بيك في الدرس التاسع! النهارده هنتكلم عن موضوع أساسي جداً في أي موقع ويب: التخطيط (Layout). فكر كده كده: لو هتبني بيت، محتاج هيكلاً أساسياً يحدد هيكل الغرف والطوابق. التخطيط في الويب زيه بالظبط — بيحدد هيكل الصفحة كلها.

في أي متجر إلكتروني، كل الصفحات عندها عناصر مشتركة: الشريط العلوي (Header) فيه الشعار والتنقل وسلة المشتريات، والشريط السفلي (Footer) فيه الروابط ومعلومات التواصل. بدل ما نكرر الكود ده في كل صفحة، بنعمل Layout واحد نستخدمه في كل الصفحات.

في Next.js App Router، الموضوع سهل جداً. بنعمل ملف layout في كل مجلد، والـ Next.js بيطبقه تلقائياً على كل الصفحات جوا المجلد ده. حلو أوي!

📁 هيكل ملفات التخطيط

خلينا نشوف هيكل الملفات اللي هنبناه النهارده:

📁 components/layout/
├── 📄 Header.jsx # الشريط العلوي
├── 📄 Footer.jsx # الشريط السفلي
├── 📄 ShopLayout.jsx # تخطيط المتجر
├── 📄 AdminLayout.jsx # تخطيط Admin
└── 📄 MobileMenu.jsx # قائمة الموبايل
💡 ليه المكونات في مجلد layout منفصل؟

في تطوير الواجهات، في فرق بين مكونات الـ UI الأساسية (زراير، بطاقات) ومكونات التخطيط (Header, Footer). مكونات التخطيط بتتحكم الصفحة كلها، فمحتاجة مجلد منفصل عشان نحافظ على التنظيم. كمان، في أوقات كتير بتفرق تخطيط المتجر عن تخطيط لوحة التحكم.

🏗️ الخطوة 1: مكون Header

الHeader هو أهم عنصر في أي موقع ويب. هو أول حاجة المستخدم بيشوفها لما يفتح الصفحة. لازم يكون واضح وجميل وسهل الاستخدام.

في المتجر بتاعنا، الHeader هيعمل الأدوار دي:

  • الشعار (Logo): رابط للصفحة الرئيسية
  • التنقل (Navigation): روابط للصفحات المختلفة
  • البحث (Search): زرار للبحث عن المنتجات
  • قائمة الأمنيات (Wishlist): المنتجات اللي المستخدم حبها
  • سلة المشتريات (Cart): المنتجات اللي المستخدم عايش يشتريها
  • حساب المستخدم (User Menu): تسجيل الدخول أو الحساب
  • قائمة الموبايل (Mobile Menu): قائمة للشاشات الصغيرة

الHeader بتاعنا فيه تأثير مهم اسمه Glass Effect. ده تأثير بيخلي الـ Header شفاف شوية لما المستخدم يكون في أعلى الصفحة، وبيتمسخ لما ينزل. التأثير ده بيعطي إحساس بالعمق والحداثة.

components/layout/Header.jsx jsx 
"use client";

import { useState, useEffect } from 'react';
import Link from 'next/link';
import { usePathname } from 'next/navigation';
import { useAuth, useUser } from '@clerk/nextjs';
import { ShoppingCart, User, Menu, X, Search, Heart } from 'lucide-react';
import { cn } from '@/lib/cn';
import { useCart } from '@/context/CartContext';
import MobileMenu from './MobileMenu';

const navLinks = [
  { href: '/', label: 'الرئيسية' },
  { href: '/products', label: 'المنتجات' },
  { href: '/categories', label: 'التصنيفات' },
  { href: '/offers', label: 'العروض' },
  { href: '/about', label: 'من نحن' },
];

export default function Header() {
  const [isScrolled, setIsScrolled] = useState(false);
  const [isMobileMenuOpen, setIsMobileMenuOpen] = useState(false);
  const pathname = usePathname();
  const { isSignedIn } = useAuth();
  const { user } = useUser();
  const { cart } = useCart();

  // تأثير التمرير
  useEffect(() => {
    const handleScroll = () => {
      setIsScrolled(window.scrollY > 20);
    };
    window.addEventListener('scroll', handleScroll);
    return () => window.removeEventListener('scroll', handleScroll);
  }, []);

  const cartItemsCount = cart?.items?.reduce((sum, item) => sum + item.quantity, 0) || 0;

  return (
    <header
      className={cn(
        "fixed top-0 left-0 right-0 z-50 transition-all duration-300",
        isScrolled
          ? "bg-gray-900/80 backdrop-blur-xl border-b border-white/10 shadow-xl"
          : "bg-transparent"
      )}
    >
      <div className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8">
        <div className="flex items-center justify-between h-16">
          
          {/* Logo */}
          <Link href="/" className="flex items-center gap-2">
            <div className="w-10 h-10 rounded-xl bg-gradient-to-br from-indigo-500 to-purple-500 flex items-center justify-center">
              <span className="text-white font-bold text-xl">M</span>
            </div>
            <span className="text-white font-bold text-xl hidden sm:block">Market</span>
          </Link>

          {/* Desktop Navigation */}
          <nav className="hidden md:flex items-center gap-1">
            {navLinks.map((link) => (
              <Link
                key={link.href}
                href={link.href}
                className={cn(
                  "px-4 py-2 rounded-lg text-sm font-medium transition-all duration-200",
                  pathname === link.href
                    ? "bg-white/10 text-white"
                    : "text-white/70 hover:text-white hover:bg-white/5"
                )}
              >
                {link.label}
              </Link>
            ))}
          </nav>

          {/* Actions */}
          <div className="flex items-center gap-2">
            {/* Search */}
            <button className="p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors">
              <Search className="w-5 h-5" />
            </button>

            {/* Wishlist */}
            <Link
              href="/wishlist"
              className="p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors"
            >
              <Heart className="w-5 h-5" />
            </Link>

            {/* Cart */}
            <Link
              href="/cart"
              className="relative p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors"
            >
              <ShoppingCart className="w-5 h-5" />
              {cartItemsCount > 0 && (
                <span className="absolute -top-1 -right-1 w-5 h-5 bg-indigo-500 rounded-full text-white text-xs flex items-center justify-center font-bold">
                  {cartItemsCount}
                </span>
              )}
            </Link>

            {/* User Menu */}
            {isSignedIn ? (
              <Link
                href="/account"
                className="flex items-center gap-2 p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors"
              >
                <div className="w-8 h-8 rounded-full bg-gradient-to-br from-indigo-500 to-purple-500 flex items-center justify-center">
                  <span className="text-white text-sm font-medium">
                    {user?.firstName?.[0] || "U"}
                  </span>
                </div>
              </Link>
            ) : (
              <Link
                href="/sign-in"
                className="px-4 py-2 rounded-lg bg-indigo-500 text-white text-sm font-medium hover:bg-indigo-600 transition-colors"
              >
                تسجيل الدخول
              </Link>
            )}

            {/* Mobile Menu Button */}
            <button
              onClick={() => setIsMobileMenuOpen(!isMobileMenuOpen)}
              className="md:hidden p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors"
            >
              {isMobileMenuOpen ? (
                <X className="w-5 h-5" />
              ) : (
                <Menu className="w-5 h-5" />
              )}
            </button>
          </div>
        </div>
      </div>

      {/* Mobile Menu */}
      <MobileMenu
        isOpen={isMobileMenuOpen}
        onClose={() => setIsMobileMenuOpen(false)}
        navLinks={navLinks}
        pathname={pathname}
      />
    </header>
  );
}
💡 شرح المكونات
  • usePathname(): لمعرفة المسار الحالي وتحديد الرابط النشط
  • useAuth() و useUser(): للتحقق من حالة تسجيل الدخول
  • useCart(): لجلب عدد عناصر السلة
  • useEffect: للكشف عن التمرير وتغيير شكل Header

شرح تفصيلي لأجزاء الـ Header

خلينا نفصص في كل جزء:

تأثير التمرير (Scroll Effect):

في السطور اللي بتستخدم useEffect، بنسمع لحدث التمرير (scroll event). لما المستخدم ينزل أكتر من 20 بكسل، بنغير قيمة isScrolled لـ true. وبما إن الـ className بتاع الـ Header بيعتمد على القيمة دي، بتتغير الشكل تلقائياً.

الروابط النشطة (Active Links):

بنستخدم usePathname() عشان نعرف الصفحة الحالية. وبعدين بنقارنها بكل رابط. لو متطابقة، بنضيف فئة "bg-white/10" عشان نبين إن الرابط ده هو النشط.

عداد السلة (Cart Badge):

بنحسب عدد العناصر في السلة باستخدام reduce. لو العدد أكبر من صفر، بنظهر Badge صغير فوق أيقونة السلة فيه العدد. لو صفر، مش بنظهر حاجة.

⚠️ أخطاء شائعة مع Header
  • الخطأ: استخدام position: fixed بدون padding-top للمحتوى — النتيجة: أول محتوى بيختفي تحت الـ Header
  • الخطأ: نسيان z-50 — النتيجة: عناصر تانية بتظهر فوق الـ Header
  • الخطأ: عدم التعامل مع window في Server Component — النتيجة: خطأ في Server Side Rendering
  • الحل: دايماً استخدم "use client" في المكونات اللي بتتفاعل مع المستخدم

📱 الخطوة 2: قائمة الموبايل

مكون MobileMenu للقائمة على الأجهزة المحمولة. في الأجهزة الصغيرة، الـ Navigation العادي مش بيكون مناسب، فبنعمل قائمة جانبية بتظهر لما المستخدم يضغط على أيقونة القائمة (Hamburger Menu).

المكون ده بيتكون من أجزاء:

  • Overlay: خلفية شافة سوداء بتظهر ورا القائمة
  • Panel: القائمة نفسها على الجانب الأيمن (لأننا شغالين RTL)
  • Navigation Links: الروابط الرئيسية
  • Auth Links: روابط تسجيل الدخول أو الحساب
components/layout/MobileMenu.jsx jsx 
"use client";

import Link from 'next/link';
import { useAuth } from '@clerk/nextjs';
import { cn } from '@/lib/cn';
import { X, User, LogIn } from 'lucide-react';

export default function MobileMenu({ isOpen, onClose, navLinks, pathname }) {
  const { isSignedIn } = useAuth();

  if (!isOpen) return null;

  return (
    <div className="md:hidden fixed inset-0 top-16 z-40">
      {/* Overlay */}
      <div
        className="absolute inset-0 bg-black/50 backdrop-blur-sm"
        onClick={onClose}
      />
      
      {/* Menu Content */}
      <div className="absolute right-0 top-0 w-72 h-full bg-gray-900/95 backdrop-blur-xl border-l border-white/10 overflow-y-auto">
        <div className="p-4">
          {/* Close Button */}
          <button
            onClick={onClose}
            className="absolute top-4 left-4 p-2 rounded-lg text-white/70 hover:text-white hover:bg-white/10 transition-colors"
          >
            <X className="w-5 h-5" />
          </button>

          {/* Navigation Links */}
          <nav className="mt-8 space-y-2">
            {navLinks.map((link) => (
              <Link
                key={link.href}
                href={link.href}
                onClick={onClose}
                className={cn(
                  "block px-4 py-3 rounded-xl text-base font-medium transition-all duration-200",
                  pathname === link.href
                    ? "bg-indigo-500/20 text-indigo-300"
                    : "text-white/70 hover:text-white hover:bg-white/5"
                )}
              >
                {link.label}
              </Link>
            ))}
          </nav>

          {/* Divider */}
          <div className="my-6 border-t border-white/10" />

          {/* Auth Links */}
          <div className="space-y-2">
            {isSignedIn ? (
              <>
                <Link
                  href="/account"
                  onClick={onClose}
                  className="flex items-center gap-3 px-4 py-3 rounded-xl text-white/70 hover:text-white hover:bg-white/5 transition-colors"
                >
                  <User className="w-5 h-5" />
                  <span>حسابي</span>
                </Link>
                <Link
                  href="/orders"
                  onClick={onClose}
                  className="flex items-center gap-3 px-4 py-3 rounded-xl text-white/70 hover:text-white hover:bg-white/5 transition-colors"
                >
                  <span>طلباتي</span>
                </Link>
              </>
            ) : (
              <Link
                href="/sign-in"
                onClick={onClose}
                className="flex items-center gap-3 px-4 py-3 rounded-xl bg-indigo-500 text-white font-medium"
              >
                <LogIn className="w-5 h-5" />
                <span>تسجيل الدخول</span>
              </Link>
            )}
          </div>
        </div>
      </div>
    </div>
  );
}

ملاحظة مهمة: لاحظ إننا بنستخدم md:hidden و md:block. ده معناه إن القائمة دي بتظهر بس على الشاشات الصغيرة (أقل من breakpoint md اللي هو 768px). على الشاشات الكبيرة، الـ Navigation العادي هو اللي بيظهر.

🦶 الخطوة 3: مكون Footer

الFooter هو الشريط السفلي في الصفحة. مهما طالت الصفحة، الـ Footer دايماً بيكون في الآخر. فيه روابط مهمة ومعلومات التواصل ووسائل الاجتماعية.

في المتجر بتاعنا، الـ Footer فيه 3 أقسام رئيسية:

  • المتجر: روابط للمنتجات والعروض والجديد
  • الدعم: الأسئلة الشائعة والشحن والتتبع
  • الشركة: من نحن والشروط وسياسة الخصوصية

💡 نصائح لتصميم Footer جيد

  • أضف روابط للشروط وسياسة الخصوصية — دي مهمة قانونياً
  • خلي الألوان متناسقة مع باقي الموقع

🏪 الخطوة 4: ShopLayout

التخطيط الرئيسي للمتجر. ده المكون اللي بيلف كل صفحات المتجر. ببساطة: Header في الأول، المحتوى في النص، والFooter في الآخر.

components/layout/ShopLayout.jsx jsx 
import Header from './Header';
import Footer from './Footer';

export default function ShopLayout({ children }) {
  return (
    <div className="min-h-screen flex flex-col bg-gray-950">
      {/* Header */}
      <Header />

      {/* Main Content */}
      <main className="flex-1 pt-16">
        {children}
      </main>

      {/* Footer */}
      <Footer />
    </div>
  );
}

ملاحظة مهمة: لاحظ pt-16 على الـ main. ده عشان الـ Header ثابت (fixed) في أعلى الشاشة، فمحتاجين مسافة فوق عشان المحتوى ما يختفيش تحته. بدون الـ padding ده، أول محتوى في الصفحة هيبان تحت الـ Header.

كمان لاحظ flex-1. ده بيخلي الـ main ياخد كل المساحة المتاحة. لو الصفحة طويلة، الـ Footer هينزل لتحت. لو الصفحة قصيرة، الـ Footer هيفضل في الآخر. الطريقة دي بتخلي الـ Footer دايماً في المكان الصح.

🔧 الخطوة 5: AdminLayout

التخطيط الخاص بلوحة التحكم. ده مختلف تماماً عن تخطيط المتجر. فيه شريط جانبي (Sidebar) فيه الروابط الإدارية، وشريط علوي (TopBar) فيه معلومات المستخدم.

الـ AdminLayout أهم من ShopLayout في بعض النواحي:

  • فيه شريط جانبي قابل للطي (Collapsible Sidebar)
  • محتاج Clerk Auth للتحقق من صلاحيات المستخدم
  • متجاوب مع الشاشات الصغيرة (Mobile Friendly)
  • فيه روابط خاصة بالإدارة زي إدارة المنتجات والطلبات والمستخدمين

ملاحظات مهمة على الكود:

  • isSidebarOpen: ده State بيحكم هل الشريط الجانبي مفتوح ولا مطوي. لما يكون مفتوح، العرض 64 (w-64). لما يكون مطوي، العرض 20 (w-20)
  • rotate-180: الأيقونة بتدور 180 درجة لما الشريط يتقلص. ده بيدي إحساس بالتفاعلية
  • ml-64 / ml-20: المحتوى الرئيسي بيتحرك يمين أو شمال حسب حالة الشريط الجانبي
  • flex-shrink-0: الأيقونات ما بتتقلصش لو الشريط اتضغط
  • SignOutButton: ده من Clerk، بيعمل تسجيل خروج من المستخدم
⚠️ ملاحظات هامة
  • تأكد من استخدام "use client" في المكونات التي تستخدم React Hooks
  • AdminLayout يحتاج Clerk Auth للتحقق من المستخدم
  • الشريط الجانبي متجاوب مع الشاشات الصغيرة
  • استخدم z-40 عشان الشريط الجانبي يظهر فوق المحتوى
  • استخدم transition-all duration-300 عشان الحركة تكون سلسة

📐 الخطوة 6: تخطيط الصفحة الرئيسية

الآن نحتاج نربط التخطيط بالصفحات في Next.js App Router. الطريقة سهلة: بنعمل ملف layout.js في كل مجلد، والـ Next.js بيطبقه تلقائياً على كل الصفحات جوا المجلد ده.

app/(shop)/layout.js jsx 
import ShopLayout from '@/components/layout/ShopLayout';

export const metadata = {
  title: 'Market - متجرك الإلكتروني',
  description: 'تسوّق أفضل المنتجات بأسعار منافسة',
};

export default function Layout({ children }) {
  return (
    <ShopLayout>
      {children}
    </ShopLayout>
  );
}
app/(admin)/layout.js jsx 
import AdminLayout from '@/components/layout/AdminLayout';

export const metadata = {
  title: 'Admin - لوحة التحكم',
};

export default function Layout({ children }) {
  return (
    <AdminLayout>
      {children}
    </AdminLayout>
  );
}

ملاحظة مهمة: لاحظ إننا بنستخدم (shop) و (admin) في المسارات. ده اسمه Route Groups في Next.js. الأقواس معناها إن المجلد ده مش جزء من المسار الفعلي. يعني المستخدم بيدخل /products مش /shop/products.

🚨 أخطاء شائعة يجب تجنبها

⚠️ الأخطاء اللي لازم تتجنبها
  • الخطأ 1: استخدام position: fixed للـ Header بدون padding للمحتوى — النتيجة: أول محتوى بيختفي
  • الخطأ 2: نسيان "use client" في المكونات اللي فيها useState — النتيجة: خطأ في الـ Server
  • الخطأ 3: عدم استخدام usePathname() لتحديد الرابط النشط — النتيجة: المستخدم مش ه يعرف هو فين
  • الخطأ 4: عدم التعامل مع الشاشات الصغيرة — النتيجة: الموقع مش هيشتغل على الموبايل
  • الخطأ 5: نسيان z-index — النتيجة: عناصر بتظهر فوق بعضها بالغلط
